# 統合プロジェクト - Integrated Projects

## 概要

このディレクトリには、複数のスキルを統合した**実務レベルのプロジェクト**が含まれています。

各プロジェクトは、基礎ガイドで学んだ知識を統合し、**実際のアプリケーション開発**を通じて実践的なスキルを身につけることを目的としています。

## 統合プロジェクトの特徴

### 実務レベルの開発体験

基礎ガイドと統合プロジェクトの違い：

| 項目 | 基礎ガイド | 統合プロジェクト |
|------|----------|----------------|
| **規模** | 単一機能・小規模 | 複数機能・中規模 |
| **技術数** | 1-2個 | 3-5個 |
| **アーキテクチャ** | シンプル | レイヤードアーキテクチャ |
| **認証** | なし | JWT認証システム |
| **データベース** | JSON/メモリ | PostgreSQL/本格的DB |
| **デプロイ** | なし | 本番デプロイ |
| **学習時間** | 1-3時間 | 15-20時間 |
| **成果物** | 学習用デモ | ポートフォリオ作品 |

### 就職・案件獲得に使える

✅ **ポートフォリオとして提示可能**
- 実務レベルのコード品質
- 本番環境で動作
- GitHub公開可能

✅ **技術面接でアピール可能**
- アーキテクチャ設計の説明
- 技術選定の理由
- 実装時の工夫

✅ **実務経験の代替**
- フルスタック開発経験
- チーム開発フロー（Git）
- 本番デプロイ経験

## 学習の進め方

### ステップ1：基礎ガイド完了（必須）

統合プロジェクトを始める前に、以下の基礎ガイドを完了してください：

**フルスタックタスク管理アプリの場合:**

1. **[React Development - 基礎ガイド](../react-development/SKILL.md)**
   - 01-07 全てのガイド（約6-8時間）

2. **[Node.js Development - 基礎ガイド](../nodejs-development/SKILL.md)**
   - 01-06 全てのガイド（約6-8時間）

3. **[Backend Development - 基礎ガイド](../backend-development/SKILL.md)**
   - 01-07 全てのガイド（約6-8時間）

**合計:** 約18-24時間

### ステップ2：統合プロジェクト実装（15-20時間）

基礎ガイド完了後、統合プロジェクトに取り組みます。

**推奨スケジュール（3週間）:**

```
Week 1:
- 環境構築
- データベースセットアップ
- 認証API実装
- タスクAPI実装

Week 2:
- 認証画面実装
- タスク管理画面実装
- ダッシュボード実装
- 統合テスト

Week 3:
- バックエンドデプロイ（Render）
- フロントエンドデプロイ（Vercel）
- 動作確認
- ポートフォリオ準備
```

### ステップ3：ポートフォリオ化（1-2時間）

プロジェクト完了後：

1. **README.md作成**
   - プロジェクト概要
   - 技術スタック
   - 実装機能
   - デモURL
   - セットアップ手順

2. **GitHubリポジトリ整理**
   - わかりやすいディレクトリ構成
   - コードコメント追加
   - .gitignore確認

3. **履歴書・職務経歴書に記載**
   - プロジェクト名
   - 使用技術
   - 担当範囲
   - 工夫した点

## よくある質問

### Q1: 基礎ガイドを飛ばして統合プロジェクトから始められますか？

**A:** 推奨しません。統合プロジェクトは基礎ガイドの知識を前提としています。基礎ガイドを飛ばすと：
- 理解が不十分になる
- エラー解決に時間がかかる
- 学習効率が下がる

まずは基礎ガイドを完了してください。

### Q2: どのくらいのレベルになれば就職できますか？

**A:** 統合プロジェクトを1-2個完成させ、ポートフォリオとして提示できるレベルになれば、ジュニアエンジニアとして就職可能です。

**目安:**
- フルスタックタスク管理アプリ完成 → ジュニアエンジニアレベル
- 2-3個の統合プロジェクト完成 → 中堅エンジニアレベル

### Q3: 実務経験がなくても大丈夫ですか？

**A:** はい。統合プロジェクトは実務経験の代替となります。

**面接での説明例:**
> 「実務経験はありませんが、フルスタックタスク管理アプリを個人開発しました。React + Node.js + PostgreSQLを使用し、JWT認証、CRUD操作、本番デプロイまで経験しました。GitHubで公開しており、実際に動作するデモもご覧いただけます。」

### Q4: どのプロジェクトから始めるべきですか？

**A:** **フルスタックタスク管理アプリ**から始めてください。理由：
- 最も基本的な機能を網羅
- 他のプロジェクトの基礎になる
- 就職面接で説明しやすい

### Q5: プロジェクトの改造は推奨されますか？

**A:** 強く推奨します。以下のような改造で学習効果が高まります：
- 機能追加（検索機能、タグ機能など）
- UI/UX改善（アニメーション、レスポンシブデザイン）
- 技術変更（Redux導入、GraphQL化など）
- パフォーマンス最適化
