# タスク2: パスワードハッシュ記述修正 (1時間)
位置: ios-security/guides/auth-implementation-complete.md:123

修正前:
「クライアント側でパスワードをハッシュ化してから送信」

修正後:
「**サーバー側のみ**でパスワードをハッシュ化

実装例:
```swift
// ❌ クライアント側でハッシュ化 (脆弱)
let hashedPassword = password.sha256()
api.login(username, hashedPassword)

// ✅ 正しい実装
// クライアント: HTTPS通信で暗号化された平文パスワードを送信
api.login(username, password)  // HTTPS必須

// サーバー (Node.js):
const bcrypt = require('bcrypt');
const saltRounds = 12;
const hash = await bcrypt.hash(password, saltRounds);
// DBに保存するのはhashのみ
```

理由: クライアント側ハッシュは中間者攻撃に脆弱 (OWASP A02:2021)
参考: https://cheatsheetseries.owasp.org/cheatsheets/Authentication_Cheat_Sheet.html
```
