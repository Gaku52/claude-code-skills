# MIT級品質基準書

> 全ガイドファイルがこの基準を満たすことで、MITレベルのレビューで95/100を取れる水準を保証する。

## 各項目の詳細基準

### 1. 正確性（20点）

**20点（完璧）:**
- 全ての技術的記述が公式ドキュメント・RFC・仕様書と一致
- バージョン情報が明記され、最新の安定版に基づいている
- コード例が全て動作確認済み
- 数値・統計データに出典が明記されている

**16点（合格）:**
- 主要な技術的記述が正確
- 致命的な誤りがない
- コード例の大半が動作する

**12点以下（不合格）:**
- 技術的な誤りが含まれる
- 古い情報が更新されていない
- コード例にバグがある

### 2. 深さ（20点）

**20点（MIT級）:**
- 内部実装の仕組みまで解説（例: V8エンジンの最適化、Linuxカーネルのスケジューリング）
- 「なぜこの設計になったか」の歴史的経緯・設計判断を説明
- 学術論文・RFC・仕様書レベルの知識を含む
- 計算量分析、メモリモデル等の理論的裏付け

**16点（合格）:**
- 概念の背景と理由が説明されている
- 単なるHOW-TOではなくWHYが含まれる
- 初心者向けの表面的な説明に留まらない

**12点以下（不合格）:**
- HOW-TOのみで理由がない
- 公式ドキュメントのコピーレベル
- 「おまじない」として扱っている

### 3. 実践性（20点）

**20点（完璧）:**
- 動作するコード例が5個以上
- 基礎→応用→発展の3段階の演習
- 実際のプロダクション環境を想定したユースケース
- コピー&ペーストで動くコード
- 実行結果が明示されている

**16点（合格）:**
- コード例が3個以上
- 少なくとも1つの演習
- 基本的なユースケースがカバーされている

**12点以下（不合格）:**
- コード例が断片的で動かない
- 演習がない
- 理論のみで実装例がない

### 4. 構造（15点）

**15点（教科書級）:**
- 前提知識 → 導入（なぜ必要か）→ 核心概念 → 実装 → アンチパターン → 演習 → 深掘り → FAQ → まとめ
- 各セクションが自然に次のセクションにつながる
- 見出し階層が3レベル以内で明確

**12点（合格）:**
- 論理的な流れがある
- 見出し構造が明確
- 迷わず読み進められる

**9点以下（不合格）:**
- 構成が混乱している
- 突然の話題転換がある
- 見出しと内容が対応していない

### 5. 網羅性（10点）

**10点（完全）:**
- エッジケースを3個以上分析
- 代替手法との比較表
- トレードオフの明示
- アンチパターン2個以上
- セキュリティ上の注意点

**8点（合格）:**
- 主要なエッジケースに言及
- 少なくとも1つの代替手法
- 基本的な注意点

**6点以下（不合格）:**
- ハッピーパスのみ
- 代替手法への言及なし
- 注意点が欠落

### 6. 可読性（10点）

**10点（出版品質）:**
- ASCII図解またはダイアグラム3個以上
- 比較表2個以上
- コード例のコメントが適切
- 日本語が明瞭で冗長でない
- 段落の長さが適切（1段落3-5文）

**8点（合格）:**
- 図解が1個以上
- 表が1個以上
- 読みやすい

**6点以下（不合格）:**
- テキストの壁
- 図解・表がない
- 専門用語が未説明

### 7. 相互参照（5点）

**5点（完全）:**
- 前提知識への参照リンク
- 関連Skillの具体的なファイルへのリンク
- 「次に読むべきガイド」が適切
- 学習パスの中での位置づけが明確

**4点（合格）:**
- 主要な関連Skillへの参照
- 次のガイドへのリンク

**2点以下（不合格）:**
- 相互参照がない
- リンクが壊れている

## ガイドファイル必須要素チェックリスト

各ガイドファイルは以下の要素を全て含むこと:

### 構造要素
- [ ] タイトル（H1）+ 1行概要
- [ ] 「この章で学ぶこと」（学習目標3個以上）
- [ ] 「前提知識」（参照リンク付き）
- [ ] 6つの主要セクション（導入/概念/実装/アンチパターン/演習/深掘り）
- [ ] FAQ（3個以上）
- [ ] まとめ表
- [ ] 「次に読むべきガイド」
- [ ] 参考文献（3個以上）

### コンテンツ要素
- [ ] コード例 5個以上（全て動作可能）
- [ ] 図解/ダイアグラム 3個以上
- [ ] 比較表 2個以上
- [ ] 実践演習 3個（基礎/応用/発展）
- [ ] アンチパターン 2個以上（NG/OKコード付き）
- [ ] エッジケース分析 2個以上

### 品質要素
- [ ] 文字数 40,000字以上（目標50,000字）
- [ ] 技術的正確性の自己検証済み
- [ ] 日本語の自然さ確認済み
- [ ] リンク有効性確認済み

## 品質レビュープロセス

### セルフレビュー（執筆直後）

1. 必須要素チェックリストを全項目確認
2. 採点基準で自己採点（80点以上で通過）
3. 不足箇所を補完

### クロスレビュー（Phase完了時）

1. ランダムに10ガイドを抽出
2. 品質基準で独立採点
3. 平均95/100以上を目標
