# ラムダ計算と計算モデル

> ラムダ計算はチューリングマシンと等価な計算モデルであり、関数型プログラミングの数学的基盤である。

## 1. ラムダ計算の基礎

```
ラムダ計算（Church, 1936）:

  3つの構文要素のみ:
  1. 変数:     x
  2. 抽象:     λx.M     （関数定義）
  3. 適用:     M N       （関数適用）

  例:
  恒等関数:    λx.x         → Python: lambda x: x
  定数関数:    λx.λy.x      → 常に第1引数を返す
  適用:        (λx.x) 5     → 5

  自然数（チャーチ数）:
  0 = λf.λx.x           → 「fを0回適用」
  1 = λf.λx.f x         → 「fを1回適用」
  2 = λf.λx.f (f x)     → 「fを2回適用」
  3 = λf.λx.f (f (f x)) → 「fを3回適用」

  加算: λm.λn.λf.λx.m f (n f x)
  → 「mの分fを適用した後、nの分fを適用」

  → たった3つの構文要素で全ての計算が表現可能！
```

## 2. 関数型プログラミングとの関係

```
ラムダ計算 → 関数型プログラミング:

  ラムダ計算           Haskell              Python/JS
  ──────────────────────────────────────────────────
  λx.x                \x -> x              lambda x: x
  λx.λy.x+y           \x y -> x + y        lambda x, y: x + y
  (λx.x*2) 5          (\x -> x*2) 5        (lambda x: x*2)(5)

  ラムダ計算から生まれた概念:
  - 高階関数: map, filter, reduce
  - クロージャ: 自由変数を捕捉する関数
  - カリー化: λx.λy.M  (多引数関数の1引数関数への分解)
  - 遅延評価: 必要になるまで計算しない
  - 参照透過性: 同じ入力→同じ出力（副作用なし）

  実務での影響:
  - React: 関数コンポーネント、イミュータブルな状態管理
  - Redux: 純粋関数としてのリデューサー
  - RxJS: 関数合成によるストリーム処理
  - Rust: 所有権 + 関数型スタイル
```

## 3. 他の計算モデル

```
チューリング等価な計算モデル:

  ┌──────────────────┬──────────────────┬─────────────────┐
  │ モデル            │ 年代             │ 影響を受けた言語 │
  ├──────────────────┼──────────────────┼─────────────────┤
  │ チューリングマシン │ 1936 (Turing)   │ 命令型言語全般   │
  │ ラムダ計算        │ 1936 (Church)   │ Lisp, Haskell   │
  │ 帰納的関数        │ 1936 (Gödel)    │ 数理論理学       │
  │ μ再帰関数        │ 1936 (Kleene)   │ 再帰の理論       │
  │ 組合せ論理        │ 1920s (Curry)   │ SK計算子         │
  │ セルオートマトン  │ 1970 (Conway)   │ Game of Life    │
  └──────────────────┴──────────────────┴─────────────────┘

  全て等価（同じ計算能力）= チャーチ=チューリングの提唱
```
