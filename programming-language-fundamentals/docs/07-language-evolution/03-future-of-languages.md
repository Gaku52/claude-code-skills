# プログラミング言語の未来

> AI がコードを書く時代に、プログラミング言語はどう進化するか。人間と AI の協調、新しいパラダイム、次世代の言語を展望する。

## この章で学ぶこと

- [ ] プログラミング言語の進化の方向性を予測する
- [ ] AI 時代における言語の役割の変化を理解する

---

## 1. 現在のトレンド

```
2024-2025 の明確なトレンド:

  1. 段階的型付けの普及
     JavaScript → TypeScript（事実上の標準化）
     Python → 型ヒント + mypy/Pyright
     PHP → 型宣言の強化
     → 「動的型付け言語に型を追加する」流れが加速

  2. メモリ安全性の要求
     米国政府: 「メモリ安全でない言語（C/C++）から移行せよ」
     Linux kernel: Rust の部分採用
     Android: Rust の採用拡大
     → Rust 的な所有権概念が他の言語にも波及

  3. Wasm（WebAssembly）の拡大
     ブラウザ外での実行: WASI, Cloudflare Workers
     多言語対応: Rust, C++, Go, Python → Wasm
     → 「一度書いてどこでも動く」の新しい形

  4. AI コード生成の日常化
     GitHub Copilot, Claude Code, Cursor
     → 言語の「書きやすさ」より「読みやすさ・検証しやすさ」が重要に
```

## 2. AI 時代の言語に求められるもの

```
AI がコードを書く時代に重要になる言語特性:

  1. 強い型システム
     → AI が生成したコードの正しさを型でチェック
     → 型注釈がAIへの仕様書になる

  2. 形式検証との統合
     → コードが仕様を満たすことを数学的に証明
     → 依存型（Dependent Types）の実用化

  3. 優れたエラーメッセージ
     → AI も人間もエラーから学ぶ
     → Rust, Elm のアプローチが標準に

  4. 宣言的な記述
     → 「何をするか」を書き、「どうやるか」はAI/コンパイラに任せる
     → SQL, React, Terraform のようなアプローチ

  5. 自然言語との融合
     → コメント/docstring が実行可能な仕様になる
     → テスト生成、コード生成のアンカーポイント
```

## 3. 注目の新言語・技術

```
Mojo (2023-):
  → Pythonの構文 + C/Rustの性能
  → AI/ML に特化（GPUネイティブ）
  → Python エコシステムと完全互換

Gleam (2024-):
  → Erlang VM(BEAM)上の型付き関数型
  → Elixir のエコシステムを活用
  → JavaScript にもコンパイル可能

Vale:
  → 所有権ベース + ジェネレーショナルリファレンス
  → Rust の借用チェッカーの複雑さを回避する試み

Zig:
  → C の現代的代替
  → コンパイル時実行（comptime）
  → 隠れた制御フローなし

Carbon (Google):
  → C++ の後継を目指す実験的言語
  → C++ との段階的相互運用

Verse (Epic Games):
  → ゲーム開発向け
  → 関数型ロジック + 並行処理
```

## 4. 長期的な展望

```
5年後（2030年頃）:
  - TypeScript が JavaScript を事実上置き換え
  - Rust がシステムプログラミングの主流に
  - AI がルーティンコードの大半を生成
  - Wasm がサーバーサイドでも一般化

10年後（2035年頃）:
  - 自然言語 → 形式仕様 → コード のパイプライン
  - 型レベルプログラミングの一般化
  - 量子コンピュータ向け言語の登場
  - プログラミングの民主化（非エンジニアもAIと共にコード）

不変の原則:
  - コンピュータサイエンスの基礎は変わらない
  - アルゴリズム・データ構造・型理論は永続的
  - 言語は変わっても、概念は残る
```

## 5. プログラマーの未来

```
AI時代に人間が担う役割:

  1. アーキテクチャ設計
     → システム全体の構造を決める
     → トレードオフを判断する

  2. 要件定義と仕様策定
     → 何を作るべきかを決める
     → AIへの的確な指示

  3. コードレビューと品質保証
     → AI生成コードの検証
     → セキュリティ・パフォーマンスの判断

  4. ドメイン知識の適用
     → ビジネスロジックの理解
     → ユーザー体験の設計

  結論: 「コードを書く能力」は自動化されるが
       「何を書くべきか判断する能力」は自動化されない
       → CS基礎の理解がますます重要になる
```

## まとめ

| トレンド | 方向性 | 影響 |
|---------|--------|------|
| 段階的型付け | 動的→静的への移行 | TypeScript, Python型ヒント |
| メモリ安全性 | C/C++ → Rust | 政府レベルの要求 |
| Wasm | ユニバーサルバイナリ | マルチプラットフォーム |
| AI協調 | コード生成の日常化 | 型と仕様の重要性増大 |
| 宣言的記述 | What > How | 抽象度の向上 |
