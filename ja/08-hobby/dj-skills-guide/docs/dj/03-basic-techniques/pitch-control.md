## ピッチコントロールとは

### 基本的な定義

ピッチコントロールとは、再生する楽曲のBPM（Beats Per Minute）と音程（ピッチ/キー）を調整する技術です。アナログレコードの時代から存在するDJの基本技術であり、デジタル時代になってさらに高度な制御が可能になりました。

```
ピッチコントロール = BPMと音程（キー）の調整

物理的な原理:
- レコード/CDの回転速度を変える = BPMと音程が同時に変化
- デジタル処理で分離 = BPMと音程を独立に制御可能

Tempo Fader:
- BPM を変更する物理的なスライダー
- 通常モード: キーも連動して変わる
- Master Tempo: キーを固定したままBPMだけ変更

Master Tempo（Key Lock）:
- BPMを変更しても元のキーを維持する機能
- DSP（デジタル信号処理）によるリアルタイム変換
- 音質への影響はアルゴリズムの品質に依存
```

### 歴史的背景

ピッチコントロールの歴史を理解することで、現在のテクノロジーの意義がより深く分かります。

**アナログ時代（1970年代〜）**

Technics SL-1200シリーズが1972年に登場し、DJプレイに革命をもたらしました。このターンテーブルには±8%のピッチコントロールが搭載されており、2枚のレコードのBPMを合わせるビートマッチングが可能になりました。当時はMaster Tempo機能は存在せず、BPMを変えれば必然的にキーも変化しました。

```
Technics SL-1200の進化:
- SL-1200 (1972): ±8% ピッチ範囲
- SL-1200MK2 (1979): DJスタンダードモデル
- SL-1200MK3 (1989): ピッチリセットボタン追加
- SL-1200MK5 (2002): ±8%/±16% 切替
- SL-1200MK7 (2019): ±8%/±16%/±50%

ピッチ精度:
- アナログ時代: 約0.1%刻み（感覚による）
- デジタル時代: 0.01%〜0.001%刻み（数値表示）
```

**CDJ時代（1990年代〜）**

Pioneer CDJ-500（1994年）の登場により、CDでのDJプレイが本格化しました。デジタル技術の進歩により、Master Tempo機能が実装され、BPMを変えてもキーを維持できるようになりました。

```
CDJ時代の革新:
- CDJ-500 (1994): CD DJプレイの幕開け
- CDJ-1000 (2001): Master Tempo搭載、業界標準に
- CDJ-2000 (2009): USB対応、高精度ピッチ
- CDJ-3000 (2020): 高品質Master Tempo搭載

Master Tempo品質の進化:
- 初期: 音質劣化が顕著（金属的な響き）
- 中期: 改善されるが、大きなBPM変更では劣化
- 現在: ±6%程度なら聴感上ほぼ無劣化
```

**デジタル/コントローラー時代（2000年代〜）**

PCDJソフトウェア（Traktor, Serato, rekordbox）の登場により、ピッチコントロールは完全にデジタル処理となり、さらに高度な機能が実現しました。キーの自動検出、ハーモニックミキシングの提案、リアルタイムキーシフトなどが可能になっています。

```
デジタル時代の機能拡張:
- 自動BPM検出: ±0.01 BPM精度
- 自動キー検出: Camelot/Open Key表記
- Key Shift: 半音単位でキーを上下
- BPM Sync: 自動ビートマッチング
- Key Sync: 自動ハーモニックマッチング
- Elastic Beatgrid: 変動BPMへの対応
```

### ピッチコントロールの物理的原理

アナログとデジタルのピッチコントロールの仕組みを理解することは、音質への影響を正しく把握する上で重要です。

**アナログ方式（回転速度変更）**

```
レコードの場合:
- 33 1/3 rpm → 回転速度そのものを変更
- 速度 +8% → 周波数（音程）も +8% 上昇
- 速度 -8% → 周波数（音程）も -8% 低下

周波数と半音の関係:
- 1半音 = 約5.946%の周波数変化
- +8% ≈ 約1.3半音の上昇
- -8% ≈ 約1.4半音の低下

具体例:
- 440Hz（A4）の音 +8% → 475.2Hz（A4とA#4の間）
- 440Hz（A4）の音 -8% → 404.8Hz（G#4とA4の間）
```

**デジタル方式（タイムストレッチ/ピッチシフト）**

```
Master Tempo の仕組み:
1. 音声信号をフレーム単位で分割
2. 各フレームの長さを変更（タイムストレッチ）
3. ピッチ（周波数）は元の値を維持
4. フレーム間を補間して滑らかに接続

主要なアルゴリズム:
- WSOLA (Waveform Similarity Overlap-Add)
  → 波形の類似部分を重ね合わせ
  → 比較的低負荷、良好な音質

- Phase Vocoder
  → FFTベースの周波数領域処理
  → 高品質だが計算負荷大

- Elastique（zplane社）
  → rekordbox / Traktor で使用
  → 業界最高品質のタイムストレッチ

- Rubberband
  → オープンソース
  → 多くのDJソフトで採用
```

## 1. Tempo Fader

### Tempo Faderの構造と設計

Tempo Fader（ピッチフェーダー/ピッチスライダーとも呼ばれる）は、DJコントローラーやCDJに搭載された縦方向のスライダーです。物理的な設計と操作感は、精密なBPM調整を可能にする上で非常に重要です。

```
一般的なTempo Faderの仕様:
- 方向: 縦方向（上下スライド）
- 長さ: 約60mm〜100mm（機種による）
- 抵抗: 適度な摩擦（意図しない移動を防止）
- 中央位置: クリック感あり（±0%を触感で確認）
- 分解能: デジタルでは1024〜4096段階

各機種のFader長:
- DDJ-FLX4: 約60mm（エントリーモデル）
- DDJ-1000: 約100mm（ミドルクラス）
- CDJ-3000: 約100mm（プロフェッショナル）
- SL-1200MK7: 約60mm（ターンテーブル）
```

**Fader長と精度の関係**

フェーダーが長いほど、同じBPM変更量に対してフェーダーの物理的な移動距離が大きくなるため、より精密な調整が可能です。

```
精度の比較（±8%範囲の場合）:
60mm Fader:
- 全範囲16% = 60mm
- 1% = 3.75mm
- 0.1% = 0.375mm（非常に繊細な操作が必要）

100mm Fader:
- 全範囲16% = 100mm
- 1% = 6.25mm
- 0.1% = 0.625mm（やや余裕がある）

プロの感覚:
- 0.1%の違い = 聴き分け可能（訓練されたDJ）
- 0.05%の違い = フロアでは気づかないレベル
- 0.5%の違い = 数秒でズレが明確に
```

### DDJ-FLX4のTempo Fader

DDJ-FLX4は入門者向けのコントローラーですが、しっかりしたTempo Faderを搭載しています。

```
DDJ-FLX4 Tempo Fader仕様:
位置: 各デッキの中央部分
方向:
  上方向: マイナス（遅く / BPM低下）
  中央: ±0%（原曲BPM / デフォルト）
  下方向: プラス（速く / BPM上昇）

注意: 一部のDJ機器では上下の方向が逆のものもある
      必ず自分の機材の方向を確認すること

範囲設定: ±6% / ±10% / ±16% / WIDE
デフォルト: ±8%相当

範囲切替方法（rekordbox）:
1. rekordboxの設定画面を開く
2. コントローラータブを選択
3. Tempo Slider Range を変更
4. または SHIFT + TEMPO FADER で切替

例（±8%範囲の場合）:
128 BPM × 1.08 = 138.24 BPM（最大値）
128 BPM × 1.00 = 128.00 BPM（中央値）
128 BPM × 0.92 = 117.76 BPM（最小値）

範囲の使い分け:
±6%  : 精密なビートマッチングに最適
±10% : 一般的なDJプレイ（推奨）
±16% : ジャンル混合プレイ
WIDE  : エクストリームなBPM変更
```

### 基本操作テクニック

ビートマッチングでTempo Faderを使う際の基本的な手順と、精密な操作のコツを解説します。

```
ビートマッチング時の基本手順:

Step 1: 目標BPMの確認
- 再生中のデッキ（デッキA）のBPMを確認
- 次にミックスする曲（デッキB）のBPMを確認
- 差分を計算（例: 128 BPM → 126 BPM = -2 BPM）

Step 2: 大まかなTempo Fader調整
- BPM表示を見ながらFaderを大まかに移動
- 目標BPMの±0.5 BPM以内に近づける
- この時点では視覚（BPM表示）に頼ってOK

Step 3: ヘッドフォンでの聴覚的微調整
- CUEボタンでデッキBをヘッドフォンモニター
- デッキAとBを同時に聴く（Split Cue）
- ビートのズレ方向を判断:
  → デッキBが速い = Faderを上（マイナス方向）に微調整
  → デッキBが遅い = Faderを下（プラス方向）に微調整

Step 4: 精密調整（0.1%単位）
- 指先の微細な動きでFaderを操作
- ビートが完全に重なるポイントを探す
- 数小節間ズレないことを確認

Step 5: ロック
- BPMが合った状態でFaderに触らない
- ビートが安定していることを数秒確認
- トランジションの準備に移行
```

**Tempo Faderの持ち方と操作のコツ**

```
持ち方のバリエーション:

1. ピンチグリップ（推奨）
   - 親指と人差し指でFaderノブをつまむ
   - 最も精密な操作が可能
   - プロDJの大半がこのスタイル

2. スライドグリップ
   - 人差し指の腹でFaderノブを押す
   - 大まかな調整に向く
   - 速い操作が可能

3. フィンガーティップ
   - 指先でFaderノブに軽く触れる
   - 超微細な調整に最適
   - 0.05%以下の調整に使う

操作のコツ:
- 急激な動きは避ける（聴衆にBPM変化がバレる）
- 微調整は「押す」のではなく「触る」感覚
- Faderノブが汗で滑る場合は指を拭く
- 中央位置のクリック感を基準点として覚える
- ライブ中は視覚（BPM表示）に頼りすぎない
```

### Tempo Faderの範囲設定と戦略

ピッチ範囲の設定は、プレイスタイルやジャンルによって最適な値が異なります。

```
範囲設定の選び方:

±6% が最適な場面:
- House / Techno のような同ジャンル内でのプレイ
- BPM差が小さい曲同士のミックス（±5 BPM以内）
- 精密なビートマッチングを重視する場合
- 長いトランジションを行う場合
→ メリット: 高精度、意図しない大きなBPM変更を防止
→ デメリット: BPM差が大きい曲に対応できない

±10% が最適な場面:
- 一般的なクラブDJプレイ
- 多ジャンルの曲をミックスする場合
- EDM / Pop / Hip-Hop の混合セット
→ メリット: 汎用性が高い、ほとんどの場面で対応可能
→ デメリット: 精度はやや低下

±16% が最適な場面:
- テクニカルDJ（スクラッチ、ターンテーブリズム）
- エクストリームなBPMトランジション
- ジャンルの大幅な切り替え
→ メリット: 非常に幅広いBPM範囲をカバー
→ デメリット: 精密調整が難しい

WIDE（±100%）が最適な場面:
- 実験的な音楽制作
- サウンドデザイン
- 通常のDJプレイでは使用しない
→ メリット: 完全な自由度
→ デメリット: DJプレイでの精度は極めて低い
```

### BPM範囲のリファレンス（ジャンル別）

各ジャンルの一般的なBPM範囲を把握しておくことで、Tempo Faderの範囲設定を適切に選べます。

```
ジャンル別BPM範囲:

Hip-Hop / R&B:      70-100 BPM
Reggaeton:          90-100 BPM
Deep House:         118-125 BPM
House:              120-130 BPM
Tech House:         124-130 BPM
Progressive House:  126-132 BPM
Techno:             125-145 BPM
Trance:             130-150 BPM
Drum & Bass:        160-180 BPM
Dubstep:            138-142 BPM（ハーフタイム: 69-71 BPM）
Trap:               130-170 BPM（ハーフタイム: 65-85 BPM）
EDM / Big Room:     126-132 BPM
Future Bass:        130-170 BPM
Hardstyle:          150-160 BPM
Hardcore:           160-200+ BPM
Ambient / Downtempo: 60-100 BPM

ジャンル間の互換性:
- House → Tech House: ±6%で十分
- House → Techno: ±10%推奨
- Hip-Hop → House: ±16%またはダブルタイム変換
- D&B → House: BPMを半分として扱う（170 BPM ≈ 85 BPM）
```

### Tempo Faderのキャリブレーション

長期間使用するとTempo Faderの精度が低下する場合があります。定期的なキャリブレーションが重要です。

```
キャリブレーション手順（DDJ-FLX4）:
1. rekordboxの環境設定を開く
2. コントローラー → DDJ-FLX4を選択
3. キャリブレーションオプションを確認
4. 画面の指示に従ってFaderを上下に動かす
5. 中央位置（±0%）が正しく認識されることを確認

確認方法:
- Faderを中央に合わせる
- BPM表示が原曲BPMと一致するか確認
- ±0.1 BPM以内なら正常
- それ以上ズレている場合はキャリブレーション実施

注意事項:
- ホコリや液体がFaderに入ると精度低下
- 定期的にエアダスターで清掃
- 接点復活剤は機種によって使用可否が異なる
- 修理が必要な場合はメーカーに相談
```

## 2. Master Tempo（Key Lock）

### 機能の詳細

Master Tempo（別名: Key Lock、キーロック）は、BPMを変更してもオリジナルのキー（音程）を維持する機能です。デジタル信号処理（DSP）によってリアルタイムに音声を変換します。

```
Master Tempo ON:
BPM変更 → キー変わらない（DSP処理で維持）
例: 128 BPM, Key Am → 135 BPM, Key Am（キー維持）

Master Tempo OFF:
BPM変更 → キーも変わる（物理的に自然な変化）
例: 128 BPM, Key Am → 135 BPM, Key A#m〜Bm程度に上昇

各ソフトウェアでの名称:
- rekordbox: Master Tempo
- Serato DJ: Key Lock
- Traktor: Key Lock
- Virtual DJ: Master Tempo
- djay: Key Lock
→ 機能は同一、名称が異なるだけ
```

### DSP処理の仕組みと音質

Master Tempoの内部処理を理解することで、音質への影響を正しく判断できます。

```
タイムストレッチの処理フロー:

1. 分析フェーズ
   - 入力音声を短いフレーム（10-50ms）に分割
   - 各フレームのピッチ周期を検出
   - トランジェント（アタック部分）を特定

2. 処理フェーズ
   BPMを上げる場合（例: +5%）:
   - フレームの持続時間を短縮（5%削除）
   - ピッチは元の周波数を維持
   - フレーム間の接続をクロスフェードで滑らかに

   BPMを下げる場合（例: -5%）:
   - フレームの持続時間を延長（5%追加）
   - 追加部分は既存の波形を繰り返し利用
   - ピッチは元の周波数を維持

3. 出力フェーズ
   - 処理済みフレームを再結合
   - 不自然なアーティファクトを低減
   - リアルタイムで出力

音質劣化のパターン:
- 小さなBPM変更（±3%以内）: ほぼ無劣化
- 中程度のBPM変更（±3-6%）: わずかな劣化（プロが注意深く聴けば分かる）
- 大きなBPM変更（±6-10%）: 明確な劣化（フランジング、金属感）
- 極端なBPM変更（±10%以上）: 著しい劣化（使用非推奨）
```

**音質劣化の具体的な症状**

```
Master Tempo使用時に発生しうる音質劣化:

1. フランジング（flanging）
   - 原因: フレーム接続部分の位相ずれ
   - 症状: 音が「揺れる」ような効果
   - 発生条件: BPM変更量が大きい場合

2. メタリックサウンド
   - 原因: 高周波数帯域のアーティファクト
   - 症状: 金属的な響きが付加される
   - 発生条件: 特にボーカル素材で顕著

3. トランジェントの劣化
   - 原因: アタック部分のタイミングずれ
   - 症状: キックやスネアのアタックが鈍る
   - 発生条件: パーカッシブな素材で顕著

4. プリエコー/ポストエコー
   - 原因: フレーム処理によるゴーストサウンド
   - 症状: アタックの前後に微細な音が発生
   - 発生条件: 静寂部分の直前/直後

5. 周波数マスキング
   - 原因: 処理による周波数特性の変化
   - 症状: 特定帯域が強調/減衰
   - 発生条件: 複雑な音色（オーケストラ等）
```

### DDJ-FLX4での操作

```
DDJ-FLX4 Master Tempo操作:

ボタン位置:
- 各デッキに1つずつ搭載
- Tempo Faderの近くに配置

操作方法:
ON:  ボタンを1回押す → ボタンが点灯
OFF: もう一度押す → ボタンが消灯

LED表示:
- 点灯: Master Tempo ON（キー固定）
- 消灯: Master Tempo OFF（通常モード）

rekordbox側の設定:
1. 環境設定 → オーディオ
2. Master Tempoの品質設定:
   - Normal: CPU負荷小、音質標準
   - High: CPU負荷中、音質良好
   - Best: CPU負荷大、音質最高
   推奨: High（バランスが良い）

ショートカット:
- DDJ-FLX4のMaster Tempoボタン
- rekordbox画面上のMaster Tempoアイコンをクリック
- キーボードショートカット（カスタマイズ可能）
```

### 使い分けの詳細ガイド

**Master Tempo OFF（通常モード）が適している場面**

```
推奨シーン:
1. BPM差が小さい場合（±3%以内）
   - キー変化が微小で聴感上問題ない
   - 音質劣化がゼロ
   - CPU負荷が低い

2. アナログ的な音の変化を楽しみたい場合
   - レコードDJの自然な音質変化
   - ビニール（レコード）の温かみ

## 3. キーチェンジの影響

### BPM変更とキーの関係（詳細）

音楽のキー（調性）は、再生速度の変更によって物理的に変化します。この関係を定量的に理解しておくことが重要です。

```
BPM変更とキー変化の数学的関係:

基本公式:
キー変化（セント） = 1200 × log2(変更後BPM / 元BPM)
※ 100セント = 1半音

計算例:
128 BPM → 130 BPM (+1.56%):
= 1200 × log2(130/128)
= 1200 × log2(1.0156)
= 1200 × 0.02237
= 約 26.8セント（約1/4半音）
→ ほぼ気にならないレベル

128 BPM → 135 BPM (+5.47%):
= 1200 × log2(135/128)
= 1200 × log2(1.0547)
= 1200 × 0.07687
= 約 92.2セント（ほぼ1半音）
→ 顕著なキー変化

128 BPM → 138 BPM (+7.81%):
= 1200 × log2(138/128)
= 1200 × log2(1.0781)
= 1200 × 0.1085
= 約 130.2セント（約1.3半音）
→ 明確なキー変化
```

**BPM変更とキー変化の早見表**

```
BPM変更%  セント  半音     聴感上の影響
────────  ──────  ────     ──────────────
±0.5%     ±9     微小     気づかない
±1.0%     ±17    微小     ほぼ気づかない
±1.5%     ±26    1/4半音  注意深く聴けば分かる
±2.0%     ±34    1/3半音  意識すれば分かる
±2.5%     ±43    約半半音 やや違和感
±3.0%     ±51    半半音   違和感あり（境界線）
±3.5%     ±60    半半音+  明確な違和感
±4.0%     ±68    2/3半音  かなりの変化
±5.0%     ±85    約1半音  大きな変化

### 聞こえ方の違い（詳細分析）

**キー変化（Master Tempo OFF）の聴感的影響**

```
BPMを上げた場合（Master Tempo OFF）:
- ピッチが上がる → 高く、明るく聞こえる
- テンポが速い → エネルギッシュに感じる
- ボーカルが高くなる → チップマンク効果（極端な場合）
- キックが軽くなる → 低域の存在感が減少
- 全体的に「浮いた」印象

BPMを下げた場合（Master Tempo OFF）:
- ピッチが下がる → 低く、暗く聞こえる
- テンポが遅い → ヘビーに感じる
- ボーカルが低くなる → 太く、重厚な印象
- キックが重くなる → 低域が強調
- 全体的に「沈んだ」印象

ジャンル別の許容範囲（Master Tempo OFF）:
- Techno / House: ±3-4%（インスト中心で許容範囲広い）
- Pop / Vocal: ±2%（ボーカルのキー変化が目立つ）
- Hip-Hop: ±2-3%（ラップは比較的許容）
- Trance: ±3%（メロディが目立つため注意）
- D&B: ±3-4%（高BPMで変化が相対的に小さい）
```

**キー固定（Master Tempo ON）の聴感的影響**

```
Master Tempo ON使用時の聴感:

BPMを上げた場合:
- キーは元のまま → ハーモニーが維持される
- テンポだけが速くなる → 自然に聞こえる
- 音質: わずかなタイムストレッチアーティファクト
  → 高域のシャリシャリ感
  → トランジェントのわずかな鈍り
  → ±3%以内ならほぼ気づかない

BPMを下げた場合:
- キーは元のまま → ハーモニーが維持される
- テンポだけが遅くなる → 自然に聞こえる
- 音質: タイムストレッチのアーティファクト
  → 音が若干「引き伸ばされた」感じ
  → リバーブ感が増したように聞こえることがある
  → ±3%以内ならほぼ気づかない

音質劣化を最小限にするコツ:
1. BPM変更を最小限にする
2. 高品質のMaster Tempoアルゴリズムを使用
3. ±6%を超える変更は避ける
4. ローパスフィルターで高域のアーティファクトを隠す
5. ミックスのトランジション中は他のエフェクトで音質変化を目立たなくする
```

## 4. 実践的な使い方

### ビートマッチング時のピッチコントロール

ビートマッチングにおけるピッチコントロールの実践的な手順を、初心者向けに詳しく解説します。

```
完全なビートマッチング手順:

準備段階:
1. デッキAで曲を再生中
2. デッキBに次の曲をロード
3. デッキBのBPMと現在のBPMの差を確認

例: デッキA = 128 BPM, デッキB = 125 BPM（差: 3 BPM / 2.4%）

Step 1: Master Tempoの判断
- BPM差が3%以内 → Master Tempo OFF推奨
- BPM差が3%以上 → Master Tempo ON推奨
- 今回は2.4%なのでOFFで良い

Step 2: Tempo Fader調整
a) BPM表示を見ながらFaderを下方向（プラス）に移動
b) デッキBのBPMが128.0と表示されるまで調整
c) この時点で視覚的に合っているはず

Step 3: CUEポイントの設定
a) デッキBの1拍目にCUEポイントを設定
b) ヘッドフォンでデッキBをモニター
c) デッキAの1拍目に合わせてCUEを叩く

Step 4: ヘッドフォンモニターでの聴覚確認
a) Split CueモードON
b) 左耳: デッキA / 右耳: デッキB（またはその逆）
c) ビートのズレを聴く:
   - 「ドッドッ」と2つに聞こえる = ズレている
   - 「ドン」と1つに聞こえる = 合っている

Step 5: 微調整
a) デッキBが速い場合:
   → Tempo Faderを上（マイナス）に微調整
   → 同時にジョグホイールで一時的に減速

b) デッキBが遅い場合:
   → Tempo Faderを下（プラス）に微調整
   → 同時にジョグホイールで一時的に加速

c) 0.1%単位で調整を繰り返す
d) 8〜16小節間ズレないことを確認

Step 6: トランジション開始
a) BPMが安定したことを確認
b) フェーダーを操作してトランジション開始
c) ミックス中もビートのズレを監視
d) 必要に応じてジョグホイールで微修正
```

### キー確認とハーモニックミキシングの実践

```
rekordbox でのキー表示と確認:

キー表示の設定:
1. rekordboxの表示設定
2. キー表記を選択:

   - Classic: Am, C 形式
   - Open Key: 1m, 1d 形式
3. カラー表示ON（キーごとに色分け）

曲A: 8A（Am）→ 曲B: 8A（Am）
→ 完璧なハーモニー

パターン2: 隣接キー（安全なミックス）

曲A: 8A（Am）→ 曲B: 8B（C）   ← 平行調

  a) 短いトランジション（カットミックス）
  b) メロディをEQでカット
  c) フィルターでメロディ帯域を抑制
  d) エフェクト（リバーブ、ディレイ）で境界をぼかす

## 5. ハーモニックミキシング基礎

### Camelot Wheelの完全ガイド

Camelot Wheelは、Mixed In Key社が開発したDJ向けのキー互換性システムです。時計の文字盤のように12の位置に配置され、隣接するキーが調和するように設計されています。

```
Camelot Wheel 完全マップ:

        12B(E)
    11B(A)  1B(B)
  10B(D)      2B(F#)
 9B(G)          3B(Db)
  8B(C)       4B(Ab)
    7B(F)   5B(Eb)
        6B(Bb)

        12A(C#m)
    11A(F#m) 1A(Abm)
  10A(Bm)      2A(Ebm)
 9A(Em)          3A(Bbm)
  8A(Am)       4A(Fm)
    7A(Dm)   5A(Cm)
        6A(Gm)

互換ルール:
1. 同じ番号: 完全互換
   例: 8A + 8A = 完璧

2. ±1番号（同じ列）: 良い互換性
   例: 8A + 7A = Dm（良い）
   例: 8A + 9A = Em（良い）

3. 同番号のA↔B: 良い互換性（平行調）
   例: 8A + 8B = Am + C（良い）

4. ±2番号: やや注意（短いトランジションなら可）
   例: 8A + 6A = 注意

5. それ以外: 避ける（メロディ重複時に不協和）
```

### ハーモニックミキシングの高度なテクニック

```
テクニック1: エナジーブースト
- 現在のキーから+7（Camelot番号）にジャンプ
- 例: 8A → 3A（半音上昇と同等の効果）
- ドロップやクライマックスで効果的
- 短いカットミックスで切り替える

テクニック2: メランコリックドロップ
- メジャーからマイナーへの切替
- 例: 8B(C) → 8A(Am) = 明るい → 暗い
- 感情的な効果が大きい
- ブレイクダウンで使うと効果的

テクニック3: ムードリフト
- マイナーからメジャーへの切替
- 例: 8A(Am) → 8B(C) = 暗い → 明るい
- ビルドアップ後のドロップで使うと爽快感
- 聴衆の気分を高揚させる

テクニック4: サブドミナントミックス
- -1（Camelot番号）への移動
- 例: 8A(Am) → 7A(Dm) = V → I の解決感
- 自然で落ち着いた進行
- チルアウトセクションに適切

テクニック5: ドミナントミックス
- +1（Camelot番号）への移動
- 例: 8A(Am) → 9A(Em) = I → V の浮遊感
- 期待感を生む進行
- ビルドアップに適切
```

### Master Tempo活用によるハーモニックミキシング

```
シナリオ: BPM差が大きい2曲をハーモニックにミックス

曲A: 125 BPM, 8A (Am)
曲B: 132 BPM, 8A (Am)
BPM差: 7 BPM (5.6%)

方法1: Master Tempo OFF
- 曲Bを125 BPMに下げる（-5.6%）
- キー変化: 約-97セント ≈ ほぼ1半音下がる
- 結果: 曲Bのキーが 8A(Am) → 7A(Dm)寄りに
- ハーモニーがずれる
- メロディが重なると違和感あり

方法2: Master Tempo ON（推奨）
- 曲Bを125 BPMに下げる（-5.6%）
- キー: 8A(Am) のまま維持
- 結果: 完璧なハーモニックミックス
- 音質: わずかな劣化あり（-5.6%なので許容範囲）
- DSP処理によるアーティファクトは最小限

方法3: Master Tempo ON + キーシフト
- 曲Bを125 BPMに下げる
- さらにキーを+1半音シフト（8A → 9A に変更）
- エナジーブースト効果を追加
- 高度なテクニックだが効果的

判断基準:
BPM差 ≤ 3%: Master Tempo OFF推奨
BPM差 3-6%: Master Tempo ON推奨
BPM差 6-10%: Master Tempo ON必須
BPM差 > 10%: ミックスの方法自体を再検討
```

上達の目安: 1週間でON/OFFの違いを即座に聴き分けられるようになる
```

### 練習3: ハーモニックミキシングの実践

```
目標: Camelot Wheelに基づいたミックスを実践する

準備:
- rekordboxでキー表示をCamelotに設定
- 同じCamelot番号の曲を5-10曲集める
  例: 8A(Am)の曲を集める
- BPMは近い曲を選ぶ（±5 BPM以内）

ステップ:
1. 曲AをデッキAにロードして再生
2. 曲B（同じCamelot番号）をデッキBにロード
3. BPMを合わせる（Master Tempo状態は任意）
4. メロディが重なる部分でミックスを開始
5. 2曲のメロディがハーモニーを奏でるか確認
6. 不協和音がないかチェック

応用:
- 隣接Camelot番号の曲でも試す（±1）
- A↔B（平行調）のミックスも試す
- エナジーブースト（+7）のミックスも試す

評価基準:
- メロディの重なりが美しいか
- 不協和音が発生していないか
- トランジションが自然か
- キーの変化が気にならないか

練習時間: 各回30分、週2回以上
上達の目安: 1ヶ月でCamelot Wheelに基づいた
自然なハーモニックミキシングが行えるようになる
```

### 練習4: BPMの段階的変更

```
目標: セット全体のBPMを自然にコントロールする

準備:
- 異なるBPMの曲を10曲用意
  例: 122, 124, 124, 126, 126, 128, 128, 130, 128, 126 BPM
- BPMの昇順/降順に並べる

ステップ:
1. 最初の曲（122 BPM）を再生
2. 次の曲（124 BPM）をデッキBにロード
3. デッキBのBPMを122に合わせてミックスイン
4. ミックス完了後、デッキBのBPMを徐々に124に戻す
   → Faderを少しずつ（0.5%ずつ）中央に戻す
   → 8-16小節かけてゆっくり変更
5. 次の曲（126 BPM）をデッキAにロードして同様に

ポイント:
- 1曲あたりのBPM変更は2 BPM以内
- 変更は8小節以上かけて行う
- 聴衆が気づかないように段階的に
- ドロップやブレイクのタイミングで変更すると目立たない

練習時間: 各回45分、週1回以上
上達の目安: 2週間で自然なBPMトランジションが身につく
```

### 練習5: 耳だけでのピッチマッチング

```
目標: BPM表示に頼らずに聴覚だけでBPMを合わせる

準備:
- rekordboxのBPM表示を非表示にする
  （設定 → 表示 → BPM非表示）
- 同ジャンル、近いBPMの曲を用意
- ヘッドフォンを装着

ステップ:
1. デッキAで曲を再生
2. デッキBに曲をロード
3. BPM表示を見ずにヘッドフォンモニター
4. デッキBのキックを聴きながらデッキAと比較
5. Tempo Faderで調整:
   a) 大まかな調整: キックの速さの違いを感じ取る
   b) 微調整: 重なり具合を聴いて合わせる
6. ビートが完全に同期するまで繰り返す

聴覚的な判断基準:
- 「ドンドン」→ 2つのキックが分離 = 大きなズレ
- 「ドッドン」→ フラム（微小なズレ） = あと少し
- 「ドン」→ 1つに聞こえる = ほぼ完璧
- 数秒経ってもズレない = 完全同期

上級者テクニック:
- キック以外の要素（ハイハット、クラップ）でも判断
- フェーズ（位相）のズレを聴き取る
- ビートの「うねり」を感じ取って方向を判断

練習時間: 各回30分、週3回以上
上達の目安: 1ヶ月で耳だけで±0.2%精度のマッチングが可能に
         3ヶ月で±0.1%精度に到達
```

## 7. ジャンル別ピッチコントロール戦略

### House / Tech House

```
特徴:
- BPM範囲: 120-130 BPM
- ジャンル内のBPM差が小さい
- 4つ打ちキックが基本
- メロディよりリズム重視

推奨設定:
- ピッチ範囲: ±6%
- Master Tempo: OFF推奨（BPM差が小さいため）
- ミックスポイント: ブレイクの前後

ピッチコントロール戦略:
1. ほとんどの場合、±2-3%以内の調整で済む
2. Master Tempoはボーカル曲のみON
3. トランジションは16-32小節の長めで
4. キックの重なりを重視

プロのTips:
- Tech Houseは125-128 BPMに収束する傾向
- 128 BPMを基準に±3 BPM以内で選曲すると安全
- ハイハットのスウィング感はBPM変更で変わらない
- フィルターを使ったトランジションとの併用が効果的
```

### Techno

```
特徴:
- BPM範囲: 125-145 BPM（サブジャンルで大きく異なる）
- インストゥルメンタル中心
- 反復的なパターン
- 長いミックスが一般的

推奨設定:
- ピッチ範囲: ±10%（サブジャンル間の移動に対応）
- Master Tempo: 基本OFF（インスト中心のため許容範囲広い）
- ミックスポイント: 16-64小節の長いトランジション

ピッチコントロール戦略:
1. Minimal Techno (125-130): ±6%で十分
2. Peak Time Techno (130-140): ±10%推奨
3. Industrial Techno (135-145): ±10%推奨
4. セット内でBPMを段階的に上げる戦略が効果的

プロのTips:
- BPMの段階的な上昇でエネルギーを高める
  例: 128 → 130 → 132 → 134 → 136
- ブレイクダウン中にBPMを変更すると気づかれにくい
- ドラムパターンの類似性を利用して長いミックスを実現
- リバーブやディレイでトランジションを滑らかに
```

### Trance / Progressive

```
特徴:
- BPM範囲: 128-150 BPM
- メロディ重視
- 感情的な盛り上がり/下がり
- ブレイクダウンが重要

推奨設定:
- ピッチ範囲: ±10%
- Master Tempo: ON推奨（メロディの調和が重要）
- ミックスポイント: ブレイクダウン/ビルドアップ

ピッチコントロール戦略:
1. ハーモニックミキシング必須（メロディが重なるため）
2. Master Tempo ONでキーを維持
3. ブレイクダウンでBPM変更を行う
4. ビルドアップでエネルギーを維持/上昇させる

プロのTips:
- Tranceではキーの一致が最も重要
- ブレイクダウン中にBPMを変えて次の曲に合わせる
- ボーカルトランスではMaster Tempo必須
- キー変更を避けるため、セットリストをキーベースで構築
- Progressive Tranceは128-132 BPM周辺が多い
```

### Hip-Hop / R&B

```
特徴:
- BPM範囲: 70-100 BPM（非常に幅広い）
- ボーカル/ラップが中心
- ハーフタイム/ダブルタイムの概念
- バウンス感が重要

推奨設定:
- ピッチ範囲: ±16%（BPM差が大きい場合に対応）
- Master Tempo: ON推奨（ラップの音程変化を防止）
- ミックスポイント: アウトロ/イントロ、8小節単位

ピッチコントロール戦略:
1. BPM差が大きい場合はダブルタイム変換を活用
   例: 75 BPM → 150 BPM として扱う
2. Master Tempo ONでボーカルのキーを維持
3. ブレイク部分でBPMを変更
4. フェードアウト/フェードインの短いトランジション

プロのTips:
- ラッパーの声のキーは変えないこと（すぐバレる）
- ドラムブレイクを利用してBPMを切り替える
- スクラッチングではMaster Tempo OFFの方が自然
- バウンス感（グルーヴ）はBPM変更で失われやすいので注意
- EQでキックを切り替える「スワップミックス」が効果的
```

### Drum & Bass / Jungle

```
特徴:
- BPM範囲: 160-180 BPM
- 高速ブレイクビーツ
- ベースラインが重要
- ハーフタイム（80-90 BPM）の解釈も

推奨設定:
- ピッチ範囲: ±6%（同ジャンル内）/ ±16%（他ジャンルとの混合）
- Master Tempo: 場合による
- ミックスポイント: 16小節単位、ドロップ前後

ピッチコントロール戦略:
1. D&B内のミックス: ±6%で十分
   （170-174 BPM が最も多い帯域）
2. 他ジャンルとの接続: ハーフタイム変換
   例: D&B 174 BPM → House 87 BPM
3. ベースラインの音程を意識（Master Tempo推奨）
4. ブレイクビーツのグルーヴを維持

プロのTips:
- D&Bは174 BPMが「標準」BPM
- Liquid D&Bは170-174、Neurofunk/Jump Upは172-176が多い
- ハーフタイムとして他ジャンルと接続する場合:
  → 174 BPM ÷ 2 = 87 BPM（Hip-Hop/R&Bとの親和性）
- ローリングベースラインのキーマッチングが重要
- アーメンブレイクのピッチ変更は歴史的に一般的
```

## 8. よくある間違いとトラブルシューティング

### 初心者がやりがちなピッチコントロールの間違い

```
間違い1: Tempo Faderを大きく動かしすぎる
症状: BPMが目標値を大幅に超えて行ったり来たり
原因: フェーダーの感度に慣れていない
対策:
- まず小さな動きから始める
- BPM表示を見ながら少しずつ調整
- ±6%の範囲設定で精密さを確保
- 指先の微細な動きを練習する

間違い2: Master Tempoを常にONにする
症状: 音質が不必要に劣化している
原因: Master Tempoの副作用を理解していない
対策:
- BPM差が±3%以内ならOFFで十分
- 音質を比較して判断する
- インスト曲ではOFFの方が音質が良い
- 必要な時だけONにする

間違い3: キーを完全に無視する
症状: ミックスのハーモニーがぐちゃぐちゃ
原因: キーの概念を理解していない
対策:
- rekordboxのCamelotキー表示を有効にする
- 同じまたは隣接するCamelot番号を選ぶ
- メロディが重なる部分で不協和音がないか確認
- ハーモニックミキシングの基礎を学ぶ

間違い4: BPM表示を過信する
症状: BPM表示は合っているのにビートがズレる
原因: BPM検出の精度限界/ビートグリッドのズレ
対策:
- 必ず聴覚で確認する（最終判断は耳）
- ビートグリッドが正しいか事前にチェック
- 生録音楽はBPMが変動する場合がある
- 数値より聴感を優先する

間違い5: Tempo Faderを触ったまま手を離さない
症状: 意図しないBPM変動が起こる
原因: 手の震えやFaderのドリフト
対策:
- BPMが合ったらFaderから手を離す
- ジョグホイールでの微修正に切り替える
- Faderに触れる時間を最小限にする
- 安定したら即座に手を離す習慣を身につける
```

### 高度なトラブルシューティング

```
問題1: Master Tempoで音が「金属的」になる
原因: タイムストレッチアルゴリズムの限界
解決策:
a) BPM変更量を小さくする
b) rekordboxのMaster Tempo品質をHiFi/Perfectに上げる
c) Serato DJの場合はPitch 'n Time DJプラグインを導入
d) ミックス中にローパスフィルターで高域を抑制
e) BPM差が大きすぎる曲の組み合わせを避ける

問題2: ビートグリッドのズレ
原因: BPM検出の誤差/変動BPMの曲
解決策:
a) rekordboxでビートグリッドを手動修正
b) 曲の先頭だけでなく中盤も確認
c) 変動BPM曲にはElastic Beatgridを使用
d) ライブ演奏のレコーディングは特に注意
e) 検出BPMが実際の半分/倍になっていないか確認
   例: 70 BPMが140 BPMと検出される場合

問題3: Tempo Faderの物理的な問題
原因: ハードウェアの劣化/汚れ
解決策:
a) エアダスターでホコリを除去
b) 接点復活剤（メーカー推奨品のみ）を使用
c) Faderの動きが硬い場合はメンテナンス
d) デッドゾーン（反応しない範囲）がある場合は修理
e) 保証期間内ならメーカーに相談

問題4: Syncボタンとの併用で混乱
原因: SyncモードとTempo Faderの競合
解決策:
a) Syncボタンの動作モードを理解する
   - Beat Sync: ビート同期のみ
   - Tempo Sync: BPMも同期
b) SyncをOFFにしてからTempo Faderで手動調整
c) SyncとTempo Faderを同時に使わない
d) Syncの便利さに頼りすぎない（手動の技術も維持）

問題5: 曲の途中でBPMがズレ始める
原因: Tempo Faderの微妙なズレ/曲自体のBPM変動
解決策:
a) ジョグホイールでリアルタイム修正
b) 長いミックス中は定期的にビートの同期を確認
c) 変動BPMの曲はBeatgridを確認/修正
d) クォンタイズ機能を活用（rekordbox）
e) ミックスが短い場合は多少のズレは許容
```

### Sync機能とピッチコントロールの関係

```
Sync機能の種類と動作:

1. Beat Sync（ビートシンク）
   - ビートの位相を合わせるだけ
   - BPMは変更しない
   - Tempo Faderの位置は変わらない
   - 手動ビートマッチングの補助として使える

2. Tempo Sync（テンポシンク）
   - BPMを自動的に合わせる
   - Tempo Faderの位置も連動して変わる
   - Master Tempoの設定に従う
   - 完全自動のBPMマッチング

3. Master/Slave モード
   - 1台のデッキをMasterに設定
   - Slaveデッキが自動的にMasterのBPMに追従
   - Master側のBPM変更にSlaveが連動

Syncを使う場面:
- マルチデッキ（3-4デッキ）でのミックス
- ライブリミックスやサンプリング中
- 複雑なエフェクト操作に集中したい場合
- 素早いDJプレイが求められるバトル/コンテスト

Syncを使わない場面:
- 手動ビートマッチングのスキルを磨きたい場合
- アナログ機材（CDJ/ターンテーブル）でのプレイ
- より「人間的」なグルーヴを出したい場合
- プロDJとしての技術を証明したい場面

バランスの取り方:
- 練習時: Sync OFF（技術向上のため）
- 本番時: 状況に応じてSyncを使い分ける
- 業界の見方: Syncの使用は個人の選択（技術の一部）
- 重要なのは最終的な音楽の質
```

## 9. 高度なピッチコントロールテクニック

### ピッチライディング

ピッチライディングとは、Tempo Faderをリアルタイムで操作しながらBPMを微調整し続けるテクニックです。主にアナログDJ（レコード/CDJ）で使用されます。

```
ピッチライディングの基本:

目的:
- BPMのわずかなズレを継続的に修正
- ビートが完全に同期した状態を維持
- 変動BPMの曲に対応

テクニック:
1. Tempo Faderに常に指を置いておく
2. ビートのズレを聴き取った瞬間に微修正
3. 修正量は0.05-0.1%程度の極めて小さい動き
4. ズレを感じたら即座に対応
5. 大きく動かすのではなく「揺らす」イメージ

ピッチライディングのパターン:
パターンA: 一定方向のドリフト
- 原因: Tempo Faderの位置が微妙にズレている
- 対策: Faderを目標方向にわずかに移動して固定
- 根本解決: Faderの位置を正しく設定し直す

パターンB: ランダムなズレ
- 原因: 変動BPMの曲/Faderのジッター
- 対策: リアルタイムでの継続的な微修正
- コツ: 全体の流れを聴いて「傾向」を把握

パターンC: 周期的なズレ
- 原因: BPMの端数（128.3 vs 128.0 BPM等）
- 対策: 周期を予測して先回りで修正
- 上級: ジョグホイールとの併用
```

### ピッチベンド（ジョグホイールとの連携）

```
ピッチベンドの概念:
- Tempo Fader: 恒久的なBPM変更（固定）
- ジョグホイール: 一時的な速度変更（ピッチベンド）
- 両者の使い分けが重要

ジョグホイールでのピッチベンド:

前方に回す（時計回り）:
→ 一時的に再生速度が上がる
→ ビートが「前に進む」
→ 遅れているビートを合わせるために使用

手前に回す（反時計回り）:
→ 一時的に再生速度が下がる
→ ビートが「後ろに戻る」
→ 進みすぎたビートを合わせるために使用

DDJ-FLX4でのジョグホイール操作:
- 天面（トッププレート）を触る: 一時停止/スクラッチ
- 側面を回す: ピッチベンド
- 回す速度で変化量を調整

Tempo Faderとの組み合わせ:

ケース1: BPMは合っているが位相がズレている
→ Tempo Faderは動かさない
→ ジョグホイールで位相を合わせる
→ 一回のヌッジ（軽く触る）で解決

ケース2: BPMがわずかにズレている
→ Tempo Faderで恒久的に修正
→ 修正中のズレはジョグホイールで補正
→ BPMが合ったらジョグホイールは使わない

ケース3: ミックス中にズレが発生
→ まずジョグホイールで応急処置
→ その後Tempo Faderで根本修正
→ 聴衆に気づかれないうちに対処
```

### デジタルキーシフトの活用

```
Key Shift機能:
- BPMを変えずにキーだけを半音単位で変更
- rekordbox / Serato / Traktor いずれも対応
- ハーモニックミキシングの強力なツール

rekordboxでのKey Shift:
1. デッキ詳細パネルを開く
2. Key Shiftの上下ボタンで調整
3. 半音単位で±6半音まで変更可能
4. 元のキーに対する変更量が表示される

活用シナリオ:

シナリオ1: キーが合わない曲を合わせる
曲A: 8A (Am), 128 BPM
曲B: 6A (Gm), 128 BPM（Camelotで2ステップ離れている）
→ 曲BのKey Shiftを+2半音 → 8A (Am) に変更
→ 完璧なハーモニックミックスが可能に
→ BPMは変更不要

シナリオ2: エナジーブースト
曲A: 8A (Am)
曲B: 8A (Am)（同じキー）
→ 曲BのKey Shiftを+1半音 → 8A → 3A相当
→ 半音上がることでエネルギーが上昇
→ ドロップで切り替えると効果的

シナリオ3: ムード変更
曲A: 8A (Am)（マイナー = 暗い）
→ Key Shiftで+3半音 → 8B (C)（メジャー = 明るい）
→ 同じ曲なのに全く違う雰囲気に
→ 実験的だが面白い効果

注意点:
- 大きなKey Shift（±3半音以上）は音質劣化が大きい
- ボーカル曲では不自然に聞こえることがある
- Key Shift + Master Tempoは二重の処理で音質劣化が加算
- 事前にヘッドフォンで確認してから使用する
```

### リアルタイムBPMトランジション

```
テクニック: ミックス中のBPM漸進的変更

目的:
- 2曲のミックス中にBPMを段階的に変更
- 聴衆が気づかないレベルでのBPM移行
- ジャンル切り替え時の自然なトランジション

方法:

ステップ1: ミックスイン
- 曲Aを再生中（例: 128 BPM）
- 曲BをデッキBにロード（例: 元は132 BPM）
- 曲Bを128 BPMに合わせてミックスイン

        self._max_size = max_size
        self._created_at = datetime.now()

    def add(self, key: str, value: any) -> bool:
        """アイテムの追加（サイズ制限付き）"""
        if len(self._items) >= self._max_size:
            return False
        self._items.append({
            'key': key,
            'value': value,
            'timestamp': datetime.now().isoformat()
        })
        return True

    def find(self, key: str) -> Optional[Dict]:
        """キーによる検索"""
        for item in reversed(self._items):
            if item['key'] == key:
                return item
        return None

    def remove(self, key: str) -> bool:
        """キーによる削除"""
        for i, item in enumerate(self._items):
            if item['key'] == key:
                self._items.pop(i)
                return True
        return False

        """統計情報"""
        return {
            'total_items': len(self._items),
            'max_size': self._max_size,
            'usage_percent': len(self._items) / self._max_size * 100,

        }

def slow_search(data: list, target: int) -> int:
    """非効率な検索"""
    for i in range(len(data)):
        for j in range(i + 1, len(data)):
            if data[i] + data[j] == target:
                return (i, j)
    return (-1, -1)

# 最適化後（O(n)）
def fast_search(data: list, target: int) -> tuple:
    """ハッシュマップを使った効率的な検索"""
    seen = {}
    for i, num in enumerate(data):
        complement = target - num
        if complement in seen:
            return (seen[complement], i)
        seen[num] = i
    return (-1, -1)

            raise
    return wrapper

### トレードオフの分析

技術的な判断には必ずトレードオフが伴います。以下の観点で分析を行いましょう:

**1. 短期 vs 長期のコスト**
- 短期的に速い方法が長期的には技術的負債になることがある
- 逆に、過剰な設計は短期的なコストが高く、プロジェクトの遅延を招く

**2. 一貫性 vs 柔軟性**
- 統一された技術スタックは学習コストが低い
- 多様な技術の採用は適材適所が可能だが、運用コストが増加

**3. 抽象化のレベル**
- 高い抽象化は再利用性が高いが、デバッグが困難になる場合がある
- 低い抽象化は直感的だが、コードの重複が発生しやすい

# 設計判断の記録テンプレート
class ArchitectureDecisionRecord:
    """ADR (Architecture Decision Record) の作成"""

    def __init__(self, title: str):
        self.title = title
        self.context = ""
        self.decision = ""
        self.consequences = []
        self.alternatives = []

    def set_context(self, context: str):
        """背景と課題の記述"""
        self.context = context
        return self

    def set_decision(self, decision: str):
        """決定内容の記述"""
        self.decision = decision
        return self

    def add_consequence(self, consequence: str, positive: bool = True):
        """結果の追加"""
        self.consequences.append({
            'description': consequence,
            'type': 'positive' if positive else 'negative'
        })
        return self

    def add_alternative(self, name: str, reason_rejected: str):
        """却下した代替案の追加"""
        self.alternatives.append({
            'name': name,
            'reason_rejected': reason_rejected
        })
        return self

    def to_markdown(self) -> str:
        """Markdown形式で出力"""
        md = f"# ADR: {self.title}\n\n"
        md += f"## 背景\n{self.context}\n\n"
        md += f"## 決定\n{self.decision}\n\n"
        md += "## 結果\n"
        for c in self.consequences:
            icon = "✅" if c['type'] == 'positive' else "⚠️"
            md += f"- {icon} {c['description']}\n"
        md += "\n## 却下した代替案\n"
        for a in self.alternatives:
            md += f"- **{a['name']}**: {a['reason_rejected']}\n"
        return md
```

### ピッチコントロールの要点

```
基本原則:
1. Tempo Fader = BPM調整の基本ツール
2. Master Tempo = キー固定のDSP機能
3. 両者の使い分けが音質とハーモニーの鍵

判断フローチャート:

BPM差を確認
├── ±3%以内 → Master Tempo OFF
│   └── 理由: 音質劣化なし、キー変化も微小
├── ±3-6% → Master Tempo ON推奨
│   └── 理由: キー変化が聴感的に影響
├── ±6-10% → Master Tempo ON必須
│   └── 理由: キー変化が大きい
└── ±10%以上 → ミックス方法を再検討
    └── 選択肢: カットミックス、ダブルタイム変換等

キーの確認:
├── 同じCamelot番号 → 完璧
├── ±1番号 → 良好
├── A↔B → 良好（平行調）
├── ±2番号 → 短いトランジションなら可
└── それ以外 → 避ける or エフェクトで対処
```

### スキルレベル別の到達目標

```
初級者（学習開始〜3ヶ月）:
- Tempo Faderの基本操作ができる
- BPM表示を見ながら±1 BPM以内に合わせられる
- Master TempoのON/OFFの違いを理解している
- Syncボタンでビートを合わせられる

中級者（3ヶ月〜1年）:
- 耳だけで±0.5 BPM以内にビートマッチングできる
- Master Tempoの使い分けが適切にできる
- Camelot Wheelに基づいたハーモニックミキシングができる
- ジョグホイールとTempo Faderの使い分けができる
- 複数ジャンルのBPMに対応できる

上級者（1年以上）:
- 耳だけで±0.1 BPM以内の精密なマッチングが可能
- ピッチライディングができる
- リアルタイムBPMトランジションができる
- Key Shiftを活用した高度なハーモニックミキシングが可能
- 任意のジャンル間のスムーズなBPMトランジションが可能
- 音質の微細な変化を聴き分けられる
```

- **Tempo Fader**: BPM調整、機種によって±6%〜WIDEの範囲設定
- **Master Tempo OFF**: 自然な音質、キー変化あり、±3%以内で使用
- **Master Tempo ON**: キー固定、わずかな音質劣化、±3%以上で推奨
- **使い分け**: BPM差と音楽のジャンル/内容で判断
- **ハーモニック**: Camelot Wheelで互換キーを確認
- **DDJ-FLX4**: Master Tempoボタン、rekordboxとの連携設定
- **練習**: 段階的に難易度を上げて継続的に行う
- **プロのTips**: プリプランニング、ダイナミックBPM、ダブルタイム変換
