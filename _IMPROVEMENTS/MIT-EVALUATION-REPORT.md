### 🚨 セキュリティリスク

**CRITICAL: 本番環境用.envファイルの公開**
```bash
位置: api-cost-skill/.env
内容: 実際のAPIキーが平文で記載
リスク: APIキー漏洩、不正利用の可能性

必須対応:
1. 即座にGit履歴から削除 (git filter-branch)
2. 漏洩したAPIキーの無効化
3. .gitignoreへの追加
4. GitHub Secrets scanningの有効化
```

**HIGH: 誤ったパスワードハッシュ推奨**
```
位置: ios-security/guides/auth-implementation-complete.md:123
内容: 「クライアント側でパスワードをハッシュ化してから送信」

問題点:
- ハッシュ化はサーバー側でのみ実施すべき
- クライアント側ハッシュは中間者攻撃に脆弱
- bcrypt/Argon2はサーバー専用

修正:
クライアント側: HTTPS通信で平文パスワード送信
サーバー側: bcrypt/Argon2でハッシュ化 + salt
```
