## Reporting a Vulnerability

While this repository primarily contains documentation and best practices,
if you discover a security concern, please follow these guidelines:

### What to Report

- Sensitive information accidentally committed (API keys, credentials, etc.)
- Security vulnerabilities in example code
- Misleading security recommendations in Skills

### How to Report

1. **Do NOT open a public issue** for security concerns
2. Send an email to the repository owner (see GitHub profile for contact)
3. Or use GitHub's private vulnerability reporting:
   - Navigate to the repository
   - Click on "Security" tab
   - Click "Report a vulnerability"

### What to Include

- Description of the issue
- Location (file path and line numbers)
- Potential impact
- Suggested fix (if applicable)

### Response Timeline

- Initial response: Within 72 hours
- Status update: Within 1 week
- Resolution timeline: Depends on severity
