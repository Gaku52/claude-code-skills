## 転職活動の全体像

### エンジニア転職の特徴

一般的な転職活動と異なり、エンジニアの転職では**技術力の証明**が最も重要です。

**重視される要素：**

| 要素 | 重要度 | 証明方法 |
|------|--------|----------|
| 技術力 | ★★★★★ | ポートフォリオ、コーディング面接 |
| 学習意欲 | ★★★★☆ | GitHub活動、技術記事 |
| コミュニケーション能力 | ★★★★☆ | 面接、ドキュメント |
| 学歴 | ★★☆☆☆ | 履歴書 |
| 職歴 | ★★★☆☆ | 職務経歴書 |

### 未経験者の転職難易度

**現実的な難易度：**

```
年齢別の難易度：
- 20代前半：★★☆☆☆（比較的容易）
- 20代後半：★★★☆☆（標準的）
- 30代前半：★★★★☆（やや困難）
- 30代後半以降：★★★★★（困難）

ポートフォリオの有無：
- ポートフォリオあり：★★★☆☆
- ポートフォリオなし：★★★★★
```

**ポイント：**
- 年齢が若いほど有利
- ポートフォリオがあれば難易度が大幅に下がる
- 学習意欲と成長性をアピールできれば年齢のハンデは覆せる

### 求人の種類

#### 1. 未経験歓迎の求人

**特徴：**
- 研修制度が充実
- 給与は低め（年収300〜400万円）
- SES（客先常駐）が多い

**メリット：**
- 実務経験を積める
- 研修でスキルアップできる
- 内定が取りやすい

**デメリット：**
- 給与が低い
- 常駐先によって環境が変わる
- スキルアップのスピードが遅い場合がある

#### 2. ポテンシャル採用

**特徴：**
- 実務経験は不問だが、技術力を重視
- 自社開発企業が多い
- 給与は標準〜やや高め（年収350〜500万円）

**メリット：**
- 自社開発で働ける
- 成長環境が整っている
- 給与が比較的高い

**デメリット：**
- 選考ハードルが高い
- ポートフォリオが必須
- コーディング面接がある

#### 3. スタートアップ

**特徴：**
- 少数精鋭
- 裁量が大きい
- 給与は幅広い（年収300〜600万円）

**メリット：**
- 幅広い経験を積める
- 意思決定に関われる
- 成長スピードが速い

**デメリット：**
- 安定性が低い
- 労働時間が長い場合がある
- 教育体制が整っていない

### 求人の探し方

**主な求人媒体：**

| 媒体 | 特徴 | 推奨度 |
|------|------|--------|
| **Green** | IT業界特化、カジュアル面談可能 | ★★★★★ |
| **Wantedly** | スタートアップ多数、カジュアル面談 | ★★★★☆ |
| **リクルートエージェント** | 求人数が多い | ★★★☆☆ |
| **レバテックキャリア** | エンジニア特化、高年収求人多数 | ★★★★☆ |
| **Findy** | GitHubと連携、スキルマッチング | ★★★★☆ |
| **Paiza** | コーディングテストで企業にアピール | ★★★☆☆ |

**おすすめの使い分け：**

1. **まず登録すべき：Green、Wantedly**
   - カジュアル面談で企業の雰囲気を掴む
   - 未経験歓迎の求人も多い

2. **ポートフォリオが完成したら：Findy**
   - GitHubと連携してスキルをアピール
   - 企業からスカウトが来る

3. **本格的に活動開始：レバテックキャリア**
   - エージェントのサポートを受ける
   - 面接対策、書類添削が充実

## 技術面接の準備

### 技術面接でよくある質問 20選

#### 基礎知識編

**1. HTML、CSS、JavaScriptの役割の違いを説明してください**

```
回答例：
HTML：Webページの構造を定義（骨組み）
CSS：Webページの見た目を定義（装飾）
JavaScript：Webページに動的な機能を追加（動き）

例えば、HTMLでボタンを作り、CSSで色や形をデザインし、
JavaScriptでクリック時の処理を実装します。
```

**2. 変数宣言（var、let、const）の違いは？**

```
回答例：
var：関数スコープ、再宣言・再代入可能（非推奨）
let：ブロックスコープ、再代入可能
const：ブロックスコープ、再代入不可（定数）

実務では、基本的にconstを使い、再代入が必要な場合のみletを使用します。
varは使いません。
```

**3. == と === の違いは？**

```
回答例：
==：型変換した上で比較（暗黙的な型変換）
===：型も含めて厳密に比較

例：
'5' == 5   // true（型変換されて比較）
'5' === 5  // false（型が異なるため）

実務では、予期しないバグを防ぐため、常に===を使用します。
```

#### React編

**4. Reactのコンポーネントとは何ですか？**

```
回答例：
UIの部品を表す独立した再利用可能な単位です。

例えば、ボタン、ヘッダー、カードなどをコンポーネントとして作成し、
複数の場所で再利用できます。
これにより、コードの保守性と再利用性が向上します。
```

**5. useStateとuseEffectの使い分けを説明してください**

```
回答例：
useState：コンポーネントの状態を管理するHook
useEffect：副作用（データ取得、DOM操作など）を実行するHook

例：
useState：フォームの入力値、モーダルの開閉状態などを管理
useEffect：API からデータを取得、タイトルの変更、
           イベントリスナーの登録など
```

**6. propsとstateの違いは？**

```
回答例：
props：親コンポーネントから受け取る読み取り専用のデータ
state：コンポーネント内部で管理する可変のデータ

例：
props：親から渡された「ユーザー名」を表示
state：コンポーネント内の「モーダルの開閉状態」を管理
```

#### TypeScript編

**7. TypeScriptを使うメリットは何ですか？**

```
回答例：
1. 型安全性：コンパイル時にエラーを検出できる
2. 開発効率：エディタの補完が効く
3. 可読性：型定義がドキュメントの役割を果たす
4. リファクタリング：型があるため安全に変更できる

実務では、バグの早期発見とチーム開発の効率化に大きく貢献します。
```

**8. anyとunknownの違いは？**

```
回答例：
any：型チェックを完全に無効化（型安全性を失う）
unknown：型が不明だが、使用前に型を確認する必要がある（より安全）

実務では、anyは極力使わず、unknownを使用します。
```

```
ネットワーク負荷が大きく、パフォーマンスが悪化しました。

そこで、Supabase Realtime を導入し、WebSocketによる

結果、ネットワーク負荷を大幅に削減し、リアルタイムで

この経験を通じて、パフォーマンスを考慮した技術選定の重要性を学びました。
```

### 頻出問題のパターン

#### 1. 配列操作

**問題例：配列から重複を削除**

```javascript
// 入力: [1, 2, 2, 3, 4, 4, 5]
// 出力: [1, 2, 3, 4, 5]

function removeDuplicates(arr) {
  return [...new Set(arr)];
}

// または
function removeDuplicates(arr) {
  return arr.filter((item, index) => arr.indexOf(item) === index);
}
```

#### 2. 文字列操作

**問題例：文字列を反転**

```javascript
// 入力: "hello"
// 出力: "olleh"

function reverseString(str) {
  return str.split('').reverse().join('');
}
```

#### 3. オブジェクト操作

**問題例：配列からオブジェクトを作成**

```javascript
// 入力: ['a', 'b', 'c'], [1, 2, 3]
// 出力: {a: 1, b: 2, c: 3}

function arrayToObject(keys, values) {
  return Object.fromEntries(keys.map((key, i) => [key, values[i]]));
}
```

#### 4. FizzBuzz（定番）

```javascript
// 1から100まで
// 3の倍数: "Fizz"
// 5の倍数: "Buzz"
// 15の倍数: "FizzBuzz"
// それ以外: 数値

function fizzBuzz(n) {
  for (let i = 1; i <= n; i++) {
    if (i % 15 === 0) {
      console.log('FizzBuzz');
    } else if (i % 3 === 0) {
      console.log('Fizz');
    } else if (i % 5 === 0) {
      console.log('Buzz');
    } else {
      console.log(i);
    }
  }
}
```
