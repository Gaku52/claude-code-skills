## なぜ音源が重要なのか

**音源 = 音楽制作の心臓:**

```
制作の構成要素:

音源 (Instrument): 40%
最も重要
音色が全て

エフェクト: 30%
音色を磨く

ミキシング: 20%
バランス

マスタリング: 10%
最終調整

音源なし:

MIDI ノート → 音源 → 音が鳴る
     ↑        ×
   あっても   ない
   意味なし   = 無音

音源あり:

MIDI → Wavetable → ベース音
MIDI → Operator → リード音
MIDI → Drum Rack → キック音

= 完成

プロの選択:

Ableton付属音源:
70-80%はこれだけ

理由:
十分すぎる品質
動作が軽い
統合が完璧

外部音源:
20-30%
特殊な音色のみ
```

### 音源選択が楽曲クオリティを決める理由

音楽制作において、音源の選択は最終的な楽曲の印象を大きく左右します。同じメロディライン、同じコード進行でも、使用する音源によって全く異なるトラックになります。これは特にエレクトロニックミュージックにおいて顕著です。

```
音源の品質が影響する要素:

周波数特性:
- 豊富な倍音を持つ音源 → 存在感のあるサウンド
- 倍音が少ない音源 → クリーンだが薄い印象
- 倍音構成の違いが「プロっぽさ」を決める

ダイナミクス:
- ベロシティレスポンスの自然さ
- エンベロープの滑らかさ
- 人間的な揺らぎの再現

空間表現:
- ステレオフィールドの広がり
- モノラルでの芯の強さ
- 周波数帯域ごとのステレオ感

時間変化:
- アタックの質感（パーカッシブ vs スムース）
- サステイン中の微細な変化
- リリースの自然な減衰

プロが付属音源を好む理由:

1. CPU効率が良い
   → 同時に多くのトラックを走らせられる
   → リアルタイムパフォーマンスで安定

2. Ableton内で完結
   → プリセットの互換性が保証される
   → プロジェクトの共有が容易

3. アップデートで継続的に改善
   → Live 11でWavetableが大幅強化
   → 新しいプリセットが追加され続ける

4. Pushとの統合
   → Pushコントローラーとの連携が完璧
   → パラメーターの自動マッピング
```

```
特徴: 壮大で広がりのあるサウンド

主力: Wavetable (テクスチャー)

Lo-fi / Chillhop:
主力: Simpler (ビンテージサンプル)
コード: Analog (温かいキーボード)
ドラム: Drum Rack (ローファイドラム)
特徴: ビットクラッシュされた質感
```

## 音源の種類と使い分け

**シンセサイザーの3大方式:**

### 1. 減算式（Subtractive）

```
原理:

豊富な倍音 → フィルターで削る = 音色

代表:
Analog

特徴:
シンプル
温かい
アナログ的

用途:
パッド
ビンテージサウンド
```

#### 減算式シンセの詳細解説

減算式シンセシスは最も歴史が古く、直感的に理解しやすい合成方式です。Moog、Roland Jupiter、Sequential Prophet-5などの伝説的なシンセサイザーはすべてこの方式を採用しています。

```
減算式の信号フロー詳細:

Step 1: オシレーターで波形生成
┌─────────────────────────────────┐
│ Saw Wave (ノコギリ波)            │
│ → 全ての整数倍音を含む           │
│ → 最も倍音が豊富                 │
│ → ブラス、ストリングスの基本      │
│                                  │
│ Square Wave (矩形波)             │
│ → 奇数倍音のみ                   │
│ → クラリネット的な響き           │
│ → 中空的なサウンド               │
│                                  │
│ Triangle Wave (三角波)            │
│ → わずかな奇数倍音               │
│ → フルート的な響き               │
│ → Sub Bassに最適                 │
│                                  │
│ Noise (ノイズ)                   │
│ → 全周波数を含む                 │
│ → パーカッション、ウインド        │
│ → アタック成分の追加             │
└─────────────────────────────────┘

Step 2: フィルターで不要な倍音をカット
┌─────────────────────────────────┐
│ Low Pass Filter (LPF)            │
│ → 高域をカット                   │
│ → 最も使用頻度が高い             │
│ → 温かみを出す                   │
│                                  │
│ High Pass Filter (HPF)           │
│ → 低域をカット                   │
│ → 薄い、ブライトな音             │
│ → ライザー効果                   │
│                                  │
│ Band Pass Filter (BPF)           │
│ → 特定の帯域のみ通す             │
│ → 電話的なサウンド               │
│ → ボーカル的な質感               │
│                                  │
│ Notch Filter                     │
│ → 特定の帯域をカット             │
│ → フェイザー的な効果             │
└─────────────────────────────────┘

Step 3: フィルターの重要パラメーター
┌─────────────────────────────────┐
│ Cutoff (カットオフ)              │
│ → フィルターが効き始める周波数    │
│ → 最も重要なパラメーター         │
│ → オートメーションで動かすと効果的│
│                                  │
│ Resonance (レゾナンス)           │
│ → カットオフ付近の強調           │
│ → 上げると「ビー」という音に     │
│ → 最大で自己発振する             │
│                                  │
│ Slope (スロープ)                 │
│ → フィルターの急峻さ             │
│ → 12dB/oct → 穏やかなカット     │
│ → 24dB/oct → 急激なカット       │
│ → Technoでは24dBが定番           │
└─────────────────────────────────┘

Analogでの実践例 - 温かいパッド:

Osc 1: Saw Wave, -12 semi (1オクターブ下)
Osc 2: Saw Wave, +7 semi (5度上), Detune +10
Filter: LPF 24dB, Cutoff 800Hz, Res 20%
Filter Env: Attack 500ms, Decay 2s, Sustain 60%
Amp Env: Attack 200ms, Decay 0, Sustain 100%, Release 3s
LFO: Cutoff → +15%, Rate 0.5Hz

結果: ゆっくり開くフィルターが特徴の
温かいアナログパッドサウンド
```

#### FM合成の詳細解説

FM合成はYamaha DX7で1983年に商業化され、80年代のポップミュージックを定義したサウンドです。原理は数学的ですが、理解すれば非常にパワフルな音作りが可能になります。

```
FM合成の数学的原理（簡略化）:

基本: キャリア周波数をモジュレーターで変調

キャリア（C）: 聞こえる音の基本周波数
モジュレーター（M）: キャリアの周波数を変調する

結果の倍音: C ± n×M （nは整数）

例: C=440Hz, M=440Hz の場合
→ 440Hz, 880Hz, 1320Hz, 1760Hz...
→ 整数倍音 → 明るく澄んだ音

例: C=440Hz, M=550Hz の場合
→ 非整数倍音が発生
→ 金属的、ベル的な音色

Operatorでの用語:

Carrier (キャリア):
→ 実際に音として出力されるオシレーター
→ 必ず最低1つ必要

Modulator (モジュレーター):
→ キャリアの周波数を変調するオシレーター
→ 直接音として聞こえない
→ 変調の深さ（Amount）で倍音量を制御

Algorithm (アルゴリズム):
→ オシレーター間の接続パターン
→ Operatorでは11種類
→ どれがキャリアでどれがモジュレーターかを決定

Ratio (レシオ):
→ 各オシレーターの周波数比
→ 整数比 → 調和的な音（ブラス、オルガン）
→ 非整数比 → 非調和音（ベル、金属音）

Operatorのアルゴリズム選択ガイド:

Algorithm 1: [D→C→B→A]
→ 直列接続（最も複雑な変調）
→ 金属的なサウンド、ノイズ的なテクスチャー
→ 用途: パーカッション、効果音

Algorithm 2: [D→C]→B→A
→ 2つのモジュレーターが1つのキャリアを変調
→ バランスの取れた複雑さ
→ 用途: エレピ、ベル

Algorithm 7: A+B+C+D（全て並列）
→ 全てキャリア（加算合成）
→ オルガン的なサウンド
→ 用途: オルガン、クリーンなトーン

Algorithm 4: [D→C]+[B→A]
→ 2つの独立したFMペア
→ 2つの音色レイヤー
→ 用途: レイヤードサウンド

実践: FMベル音色の作り方:

Algorithm: 2（BがAを変調、DがCを変調）
Osc A (Carrier): Sine, Ratio 1.00
Osc B (Mod): Sine, Ratio 3.50 ← 非整数比がポイント
Osc B Level: 70%
Osc A Env: Attack 0ms, Decay 3s, Sustain 0%
Osc B Env: Attack 0ms, Decay 1.5s, Sustain 0%
→ モジュレーターが先に減衰 = ベルの倍音変化を再現
```

#### ウェーブテーブル合成の詳細解説

ウェーブテーブル合成はPPG WaveやWaldorf系シンセサイザーで発展し、現代のソフトウェアシンセサイザーで最も人気のある方式の一つです。Serumの大成功以降、多くのDAWがウェーブテーブルシンセを標準搭載するようになりました。

```
ウェーブテーブル合成の原理:

従来のシンセ:
1つの波形（Saw, Square等）を繰り返し再生
→ 静的な倍音構成

ウェーブテーブルシンセ:
複数の波形を並べたテーブルを用意
→ テーブル内の位置を変えると波形が変化
→ 位置をLFOやエンベロープで自動的に動かす
→ 時間とともに音色が変化する

波形テーブルの構造:
┌──────────────────────────────────┐
│ Position 0   → Sine Wave        │
│ Position 25  → Saw に近い波形    │
│ Position 50  → 複雑な波形        │
│ Position 75  → ノイズに近い波形   │
│ Position 100 → Digital な波形    │
└──────────────────────────────────┘
↑ この位置（Position）を動かすことで
  1つのオシレーターから多彩な音色変化を得る

Wavetableの強み:

1. 波形の多様性
   → 数百種類のウェーブテーブルを内蔵
   → カテゴリ: Basic, Complex, Vintage, etc.
   → カスタム波形のインポートも可能

2. モーフィングの滑らかさ
   → 波形間の補間が非常に滑らか
   → 不自然なクリックノイズが発生しない

3. 2オシレーター構成
   → Osc 1 と Osc 2 を独立設定
   → 異なるウェーブテーブルを組み合わせ
   → Sub Oscillator も装備

4. 強力なモジュレーション
   → マトリクス方式で自由にルーティング
   → Envelope, LFO, Velocity, Key Track等
   → 複雑な音色変化をプログラム可能

実践: モダンなTechnoベースの作り方:

Osc 1: Wavetable "Aggressive", Position 35%
Osc 2: Wavetable "Basic Shapes", Position 60%
Sub Osc: ON, Sine Wave, -24dB
Filter 1: LPF 24dB, Cutoff 450Hz, Res 30%
Filter Env: Attack 0, Decay 300ms, Sustain 20%
Amp Env: Attack 0, Decay 0, Sustain 100%, Release 50ms
LFO 1 → Filter Cutoff: Amount 25%, Rate 1/4 note

結果: うねりのある太いテクノベース
→ Osc Positionのオートメーションで展開を追加
```

#### サンプリング方式の詳細

サンプリングは「現実の音を録音して楽器として再生する」方式です。シンセシスでは作れないリアルな音色や、既存の楽曲からの素材取り込みなど、幅広い用途があります。

```
サンプリングの基本概念:

マルチサンプリング:
→ 1つの楽器を複数のピッチ・ベロシティで録音
→ キーボードの各鍵盤に適切なサンプルを割り当て
→ ピッチシフトのアーティファクトを最小化
→ リアルな楽器再現に不可欠

ゾーンマッピング:
→ どのMIDIノートでどのサンプルを再生するか
→ Key Zone（音程の範囲）
→ Velocity Zone（強さの範囲）
→ Samplerで詳細に設定可能

ループポイント:
→ サンプルの一部を繰り返し再生
→ 短いサンプルでも長い音が出せる
→ ループの品質が自然さを決定

Sampler vs Simpler の使い分け:

Sampler:
→ マルチサンプルに対応
→ 最大128ゾーン
→ 3つのフィルター
→ 複雑なモジュレーション
→ 本格的なサンプル楽器の構築

Simpler:
→ 1サンプルに特化
→ 3つのモード（Classic/One-Shot/Slice）
→ ドラッグ＆ドロップで即使用
→ Drum Rack内のパッドとして最適
→ スライス機能でブレイクビーツ作成

Simplerの3モード:

Classic モード:
→ 通常のサンプル再生
→ ループ可能
→ 鍵盤に沿ってピッチが変化
→ 用途: メロディックな楽器音

One-Shot モード:
→ MIDIノートオンで1回再生
→ ノートオフで停止しない
→ ピッチ変化あり
→ 用途: ワンショットサンプル、効果音

Slice モード:
→ サンプルを自動分割
→ 各スライスを個別のノートに割り当て
→ Warp マーカーまたはTransientで分割
→ 用途: ブレイクビーツ、ボーカルチョップ
```

```

温かみが欲しい時 → Analog
→ アナログ回路のモデリングによる自然な倍音

モダンな音が欲しい時 → Wavetable

クラブミュージック → Wavetable
→ 現代的なテクノ、ハウス
→ EDM、ベースミュージック

Analog の隠れた強み:
→ CPU負荷が非常に低い
→ パッドを複数レイヤーしても軽い
→ Freeze不要で多数のトラックに使える
```

```
│ ┌─────┬─────┬─────┬─────┐           │
│ │ Pad │ Pad │ Pad │ Pad │ × 4行     │
│ │  1  │  2  │  3  │  4  │ = 16 Pad  │

│ 各パッド = 独立したインストゥルメント  │
│ → Simpler / サンプル / シンセ        │
│ → 個別のエフェクトチェイン           │

パッドの推奨配置（テクノ/ハウス標準）:

Row 1 (C1-D#1): キック系
  C1:  Main Kick
  C#1: Sub Kick / Layered Kick
  D1:  Kick Variation
  D#1: Kick FX / Processed

Row 2 (E1-G1): スネア/クラップ
  E1:  Main Snare / Clap
  F1:  Snare Variation
  F#1: Rim Shot
  G1:  Clap Variation

Row 3 (G#1-B1): ハイハット
  G#1: Closed HH (Tight)
  A1:  Closed HH (Loose)
  A#1: Open HH
  B1:  Pedal HH

Row 4 (C2-D#2): パーカッション
  C2:  Shaker
  C#2: Tambourine
  D2:  Conga / Bongo
  D#2: Ride / Crash

Choke Group の設定:

目的:
→ 実際のハイハットのように
  Open HH を Closed HH でミュートする

設定方法:
1. Open HH パッドを選択
2. Choke Group を「1」に設定
3. Closed HH パッドを選択
4. Choke Group を同じ「1」に設定
5. → Closed HH を叩くと Open HH が止まる

応用:
→ 複数のシンバルを同じ Choke Group に
→ コンガの Open/Mute も同様に設定可能
→ 最大16グループまで作成可能

テクノキック加工のチェイン例:

Kick Sample
  → EQ Eight (30Hz以下カット, 60Hzブースト)
  → Saturator (Drive 5-10dB, Warm Tube)
  → Compressor (Ratio 4:1, Fast Attack)
  → Utility (Mono, Gain調整)

結果: パンチがあり、ローエンドが制御されたキック
```

## 重要な概念

### シンセサイザーの基本構造

```
信号の流れ:

Oscillator (オシレーター):
音の元
波形生成

↓

Filter (フィルター):
音色調整
明るさ・暗さ

↓

Amplifier (アンプ):
音量調整

↓

Effects (エフェクト):
Reverb, Delay等

制御:

Envelope (エンベロープ):
時間変化
ADSR

LFO (Low Frequency Oscillator):
周期的変化
ビブラート、トレモロ

Modulation (モジュレーション):
パラメーター間の接続
```

### エンベロープ（ADSR）完全ガイド

```
ADSR の各パラメーター詳細:

A = Attack（アタック）:
→ ノートオンから最大値に達するまでの時間
→ 0ms: 即座に最大（パーカッシブ）
→ 10-50ms: わずかな立ち上がり（自然）
→ 100-500ms: ゆっくり立ち上がる（パッド）
→ 1000ms+: 非常にゆっくり（アンビエント）

D = Decay（ディケイ）:
→ 最大値からSustainレベルに落ちるまでの時間
→ 0ms: Sustainレベルに即移行
→ 50-200ms: プラック音のキャラクター
→ 500-2000ms: ピアノ的な減衰
→ 数秒: ベル的な響き

S = Sustain（サステイン）:
→ キーを押している間の音量レベル（%）
→ 0%: 音が完全に消える（プラック、ベル）
→ 30-70%: 適度に減衰（ピアノ、エレピ）
→ 100%: 音量が落ちない（オルガン、パッド）

R = Release（リリース）:
→ ノートオフから音が消えるまでの時間
→ 0-20ms: 即停止（タイト）
→ 50-200ms: 自然な消え方
→ 500-2000ms: 余韻のある消え方
→ 5000ms+: 長い残響（アンビエント）

音色別の ADSR テンプレート:

Bass (タイトなベース):
A: 0ms  D: 200ms  S: 80%  R: 50ms
→ 即座に立ち上がり、リリースも短く制御

Pluck (プラック):
A: 0ms  D: 300ms  S: 0%  R: 100ms
→ アタック後すぐに消える、弦を弾いた音

Pad (パッド):
A: 500ms  D: 0ms  S: 100%  R: 2000ms
→ ゆっくり立ち上がり、長く持続、長い余韻

Bell (ベル):
A: 0ms  D: 3000ms  S: 0%  R: 2000ms
→ 即座に鳴り、ゆっくり減衰

Brass (ブラス):
A: 50ms  D: 500ms  S: 70%  R: 200ms
→ わずかな立ち上がり、適度な減衰

String (ストリングス):
A: 200ms  D: 0ms  S: 100%  R: 500ms
→ 自然な立ち上がり、持続、穏やかな消え方

エンベロープの重要な応用:

1. Filter Envelope:
→ フィルターの Cutoff を時間制御
→ 「ワウ」「ビョン」といった音色変化
→ 短い Decay → プラック的な質感
→ 長い Attack → スウェル効果

2. Pitch Envelope:
→ ピッチ（音程）を時間制御
→ キックドラムの「ドン」感
→ レーザー的な効果音
→ 微かな Pitch Decay → アナログ感

3. Mod Envelope:
→ 任意のパラメーターを時間制御
→ Wave Position の変化
→ LFO Amount の変化
→ エフェクトパラメーターの変化
```

### LFO（Low Frequency Oscillator）完全ガイド

```
LFO の基本:

LFO = 低い周波数のオシレーター
→ 音として聞こえない（0.01-20Hz程度）
→ 他のパラメーターを周期的に変化させる
→ 「揺れ」「うねり」を生み出す

LFO 波形の種類と効果:

Sine（サイン波）:
→ 滑らかな上下動
→ ビブラート、トレモロ
→ 最も自然な揺れ

Triangle（三角波）:
→ サインに似ているが角がある
→ わずかにシャープな揺れ

Saw Up（ノコギリ波 上昇）:
→ ゆっくり上がって急に下がる
→ フィルタースイープ効果
→ ライザー的な使い方

Saw Down（ノコギリ波 下降）:
→ 急に上がってゆっくり下がる
→ 減衰的な動き

Square（矩形波）:
→ ON/OFF のデジタル的な切り替え
→ トレモロ、ゲート効果
→ リズミカルな音量変化

Random（ランダム）:
→ ステップ状のランダム変化
→ サンプル＆ホールド
→ レトロなSF的サウンド

LFO の主な接続先と効果:

LFO → Filter Cutoff:
Rate 0.1-0.5Hz: ゆっくりしたフィルタースウィープ
Rate 1-4Hz: ワウワウ効果
Rate テンポ同期: リズミックフィルター

LFO → Pitch:
Amount 小さめ (5-15 cent): ビブラート
Amount 大きめ (半音以上): 特殊効果
Rate 5-7Hz: 自然なビブラート

LFO → Volume:
Rate ゆっくり: トレモロ
Rate テンポ同期: ゲーティング（トランス系）
Square波: 完全なON/OFF切り替え

LFO → Wave Position (Wavetable):
→ 音色が周期的に変化
→ Evolving な音色を簡単に作成
→ Amount と Rate のバランスが重要

LFO → Pan:
→ オートパン効果
→ ステレオフィールドの動き
→ Rate ゆっくり: 広がり感
→ Rate 速め: 空間的な揺らぎ

テンポ同期 LFO のリズミック活用:

1/1 (1小節):
→ ゆっくりした変化
→ セクション間の変化に

1/2 (2拍):
→ ハーフノートの揺れ

1/4 (4分音符):
→ ビートに同期した変化

1/8 (8分音符):
→ よく使われるリズミック効果

1/16 (16分音符):
→ 速いゲーティング
→ トランスゲート的効果

1/4T (三連符):
→ グルーヴ感のある揺れ

1/8D (付点8分):
→ ディレイ的なリズム感
```

### モジュレーション・マトリクスの活用

```
モジュレーションマトリクスとは:

定義:
→ ソース（変調元）とデスティネーション（変調先）を
  自由に接続するシステム
→ Wavetable の中核機能の一つ
→ 複雑な音色変化をプログラム可能

ソース（変調元）の種類:

内部ソース:
- Envelope 1/2/3: 時間変化
- LFO 1/2: 周期的変化
- Key Track: 鍵盤位置による変化
- Velocity: 打鍵の強さ
- Aftertouch: 押し込みの強さ
- Mod Wheel: モジュレーションホイール
- Random: ノートごとのランダム値

デスティネーション（変調先）の例:

オシレーター:
- Wave Position
- Transpose (ピッチ)
- Gain (音量)
- Pan

フィルター:
- Cutoff
- Resonance
- Drive

エンベロープ:
- Attack Time
- Decay Time
- Sustain Level

LFO:
- Rate
- Amount

エフェクト:
- 各エフェクトのパラメーター

実践的なモジュレーションの組み合わせ:

組み合わせ 1: ベロシティ → Filter Cutoff
→ 強く弾くと明るく、弱く弾くと暗く
→ 表情豊かな演奏が可能
→ Amount: 30-50%

組み合わせ 2: Key Track → Filter Cutoff
→ 高い音ほどフィルターが開く
→ 低音は暗く、高音は明るく
→ 自然な楽器の特性を再現

組み合わせ 3: LFO → Wave Position
→ 音色が周期的に変化
→ Evolving なサウンド
→ Rate: 0.1-1Hz, Amount: 20-40%

組み合わせ 4: Envelope → Wave Position
→ ノートの時間経過で音色が変化
→ アタック時に明るく → 徐々に暗く
→ 生きた音色の表現

組み合わせ 5: Mod Wheel → LFO Amount
→ Mod Wheel で揺れの深さをリアルタイム制御
→ パフォーマンス向け
→ 静的な音 → Wheel操作 → ビブラートが増す
```
