# EQ・コンプレッサー

ミックスの基礎となる最重要エフェクト。EQ Eightとコンプレッサーを完全マスターし、プロの音質を実現します。EQとコンプレッサーは、あらゆる音楽制作において最も頻繁に使用されるツールであり、これらを正確に操作できるかどうかがミックスの品質を左右します。本章では、Ableton Live付属のEQ EightおよびCompressorを中心に、基礎理論から実践的なテクニックまでを網羅的に解説します。

## この章で学ぶこと

- EQ Eightの8バンドパラメトリックEQの全機能
- 各バンドタイプ（Peak、Shelf、High/Low Pass）の使い分け
- Compressorの全パラメーターの理解と操作
- Attack・Releaseの調整による音色コントロール
- サイドチェイン圧縮の理論と実践
- トラック別EQ・コンプ設定の詳細ガイド
- よくある失敗とその対処法
- プロフェッショナルなミックスワークフロー
- ダイナミックEQとマルチバンドコンプレッションの基礎
- A/B比較テクニックと耳の訓練方法

## なぜEQ・コンプが最重要なのか

**ミックスの90%:**

EQとコンプレッサーは、ミックスにおいて最も根本的な役割を担うエフェクトです。どんなに高価なプラグインを購入しても、EQとコンプレッサーの使い方が間違っていれば、プロフェッショナルな音質には到達できません。逆に言えば、この2つだけで驚くほど高品質なミックスを作ることが可能です。

```
プロの音質:

1位: EQ Eight (50%)
周波数バランスの整理
不要帯域の除去
各楽器の住み分け

2位: Compressor (40%)
ダイナミクス制御
音量の安定化
パンチと存在感の付与

3位: その他 (10%)
空間系エフェクト
モジュレーション
サチュレーション

使用頻度:

EQ Eight:
全トラック100%使用
必須中の必須

Compressor:
Kick・Bass・Vocal: 100%必須
Synth Lead: 80%
Pad: 50%
その他: 70%

プロとアマチュアの差:

アマチュア:
- EQでブーストばかり
- コンプのAttack/Releaseが不適切
- A/B比較をしない
- 耳ではなく目で判断

プロフェッショナル:
- EQはカット優先（引き算の美学）
- コンプのAttack/Releaseを楽曲に合わせて微調整
- 常にA/B比較で効果を確認
- 耳を信じ、メーターは参考程度

真実:
「良いミックス」= EQ・コンプが完璧
音色・楽曲の質はその次の問題
プロとアマの差はこの2つのスキルに集約される
```

### EQとコンプの歴史的背景

EQとコンプレッサーは、録音技術の歴史とともに発展してきました。初期のレコーディングスタジオでは、ハードウェアのEQとコンプレッサーが使用され、その音色特性がジャンルごとのサウンドを定義してきました。

```
歴史的名機:

EQ:
- Pultec EQP-1A: 1950年代、真空管EQ
  特徴: 独特の温かみとエアー感
  現在: プラグインで再現

- Neve 1073: 1970年代、トランジスタEQ
  特徴: 太く力強い中域
  現在: Universal Audio等で再現

- SSL 4000 Channel EQ: 1980年代
  特徴: クリーンでパンチのある音
  現在: Waves SSL等

Compressor:
- Teletronix LA-2A: オプトコンプ
  特徴: 滑らかで自然な圧縮
  用途: Vocal・Bass

- UREI 1176: FETコンプ
  特徴: 速いアタック、パンチ
  用途: Drums・Vocal

- SSL G-Bus Compressor: VCAコンプ
  特徴: 全体をまとめるグルー感
  用途: バスコンプ・マスタリング

- Fairchild 670: 真空管コンプ
  特徴: 温かく太い音
  用途: 伝説的なマスタリング用

現代のDAW環境:
- これらの名機をプラグインで再現可能
- Ableton付属のEQ Eight/Compressorは
  クリーンで汎用性が高い
- まずは付属プラグインをマスターすることが重要
```

## EQ Eight 完全ガイド

**8バンドパラメトリックEQ:**

EQ Eightは、Ableton Liveに付属する高品質なパラメトリックEQです。8つの独立したバンドを持ち、各バンドを自由にカスタマイズできます。視覚的な周波数カーブ表示により、直感的な操作が可能です。

### インターフェイス

```
┌─────────────────────────────────┐
│  EQ Eight                       │
├─────────────────────────────────┤
│                                 │
│      周波数カーブ表示            │
│         (視覚的)                │
│   20Hz ──────────────── 20kHz   │
│   -12dB ────────────── +12dB    │
│                                 │
│  Band 1  [●]───────────────     │
│    Freq: 100 Hz                │
│    Gain: +3.0 dB               │
│    Q: 1.0                      │
│    Type: Peak                  │
│                                 │
│  [1][2][3][4][5][6][7][8]      │
│   バンド選択                    │
│                                 │
│  [Adaptive Q] [Scale: 0 dB]    │
│  [Audition] [Oversampling]     │
└─────────────────────────────────┘

8つのバンド:
それぞれ独立して調整可能
任意のバンドをOn/Off切替

バンドタイプ:
- Low Cut (High Pass Filter)
- Low Shelf
- Peak (Bell)
- Notch
- High Shelf
- High Cut (Low Pass Filter)

追加機能:
- Adaptive Q: ゲイン変化でQ値自動調整
- Scale: 全バンドのGainを一括スケーリング
- Audition: 選択バンドの周波数のみ聴取
- Oversampling: 高域での精度向上（CPU増加）
```

### Adaptive Qの詳細

```
Adaptive Q:

機能:
Gainの変化量に応じてQ値を自動調整

On:
- Gainを上げるとQ値自動で広がる
- Gainを下げるとQ値自動で狭まる
- より自然なEQカーブ

Off:
- Q値は常に固定
- 精密なコントロール

推奨:
- 初心者: On（自然な結果が得やすい）
- 中級者以上: Off（精密制御のため）
- サージカルEQ: Off推奨
- ブロードEQ: On推奨
```

### Scale機能の活用

```
Scale:

機能:
全バンドのGainを一括で増減

活用方法:

1. 通常通りEQ設定
2. Scale: 0 dB（標準）

A/B比較:
Scale: -12 dB → 0 dB
EQ効果を段階的に確認

過剰防止:
Scale: 50%に設定
→ 全Gainが半分に
→ 控えめなEQ効果

推奨:
まず好きなだけEQ設定
→ Scale 50%に減らす
→ 過剰処理の防止
```

### バンドタイプ詳細

```
1. Low Cut (High Pass Filter):

機能:
設定周波数以下をカット
低域を段階的に除去

用途:
不要な低域除去
マイクの近接効果除去
ランブルノイズ除去
全トラックの必須処理

設定:

Frequency:
カット開始点（-3dBポイント）

Slope:
6 dB/oct: 緩やか、自然
12 dB/oct: 標準的なカット
18 dB/oct: やや急峻
24 dB/oct: 急峻、明確なカット
48 dB/oct: 非常に急峻、ブリックウォール的

推奨:

Kick: 30 Hz (24 dB/oct)
理由: サブベース以下の不要振動を除去
     30Hz以下は体感も聴覚もほぼ不可能
     スピーカーの保護にも重要

Bass: 40 Hz (24 dB/oct)
理由: Kickとの低域分離
     40Hz以下はBassには不要
     明確なローエンドを実現

Lead: 200 Hz (12 dB/oct)
理由: Leadに低域は不要
     12dB/octで自然なカット
     低域をKick・Bassに譲る

Vocal: 80 Hz (18 dB/oct)
理由: マイク近接効果の除去
     ポップノイズの軽減
     明瞭度の向上

Pad: 300 Hz (12 dB/oct)
理由: 低域は完全にKick・Bassに
     Padは中高域で空間を埋める
     低域カットで他トラックとの干渉防止

Hi-Hat: 400 Hz (24 dB/oct)
理由: Hi-Hatに低域は一切不要
     ブリードの除去
     クリーンな高域

注意点:
- High Passは必ず全トラックに適用
- 「不要かも」と思ったら必ずカット
- カット後にA/Bで確認
- 位相回転に注意（特に急峻なSlope）

位相回転について:
全てのフィルターは位相を回転させる
急峻なSlope = 大きな位相回転
対策: Linear Phase EQの使用（CPU重い）
通常: ミニマルフェーズで十分


2. Low Shelf:

機能:
設定周波数以下の全体を均一に増減

用途:
温かみの追加（ブースト）
低域過多の削減（カット）
全体の重心調整

設定:

Frequency:
変化が始まる周波数ポイント

Gain:
増減量 (-12 〜 +12 dB)
通常は ±3 dB以内が安全

Q (Resonance):
Shelf形状の急峻さ
低いQ: 緩やかな変化
高いQ: 急峻な変化（共鳴ピーク発生）

推奨:

Bass太くしたい:
80 Hz, +2 dB
理由: 80Hz中心でBassの基音を強化
     わずかなブーストで十分な効果

Vocal軽くしたい:
120 Hz, -2 dB
理由: 120Hz以下をわずかにカット
     近接効果の軽減
     明瞭度の向上

Kick強調:
60 Hz, +3 dB
理由: Kickの基音を強調
     サブベース感の追加

全体を温かく:
100 Hz, +1 dB
理由: Master EQで全体の温かみ追加
     わずかなブーストで十分

注意点:
- Low ShelfとHigh Passを併用する場合
  周波数が近すぎないよう注意
- ブースト量は控えめに
- ヘッドルームの消費に注意


3. Peak (Bell):

機能:
特定周波数を中心にベルカーブで増減

用途:
問題周波数の除去（ナローバンドカット）
良い周波数の強調（ワイドバンドブースト）
共鳴除去
キャラクター付与

設定:

Frequency:
中心周波数（影響の最大点）

Gain:
増減量（カットまたはブースト）

Q (Quality Factor):
帯域幅の制御
0.5 = 非常に広い（約2.5オクターブ）
1.0 = 広い（約1.4オクターブ）
2.0 = やや狭い（約0.7オクターブ）
5.0 = 狭い（約0.3オクターブ）
10.0 = 非常に狭い（ノッチ的）

推奨:

問題周波数除去:
Q: 3-5 (狭い)
Gain: -3 〜 -6 dB
手法: まずQ高く、Gain上げてスイープ
     → 問題周波数特定
     → Gain下げてカット

強調:
Q: 1-2 (広い)
Gain: +2 〜 +4 dB
手法: 広いQでブースト
     → 自然な強調

サージカルEQ:
Q: 8-12 (非常に狭い)
Gain: -6 〜 -12 dB
用途: 共鳴除去、フィードバック除去

Sweeping EQ（問題周波数探し）:
1. Peakバンドを1つ選択
2. Q: 5.0 (狭め)
3. Gain: +8 dB (大きくブースト)
4. Frequency: ゆっくりスイープ
5. 不快に聞こえるポイントを特定
6. Gain: -3 〜 -6 dB (カット)
7. Q: 2-4に調整
8. A/B比較で確認

注意点:
- ブーストは最大+4 dBを推奨
- カットは-6 dBまでが一般的
- それ以上のカットが必要 = 音源に問題あり
- Q値は目的に応じて使い分け


4. Notch:

機能:
非常に狭い帯域を深くカット

用途:
特定周波数の完全除去
ハウリング防止
電源ハム除去（50/60Hz）
特定共鳴の除去

設定:
Frequency: 問題周波数
Gain: -12 〜 -∞ dB
Q: 10-20 (非常に狭い)

推奨:
電源ハム: 50 Hz (日本) / 60 Hz (米国)
倍音: 100, 150, 200, 250 Hz...
ハウリング: 特定周波数

注意点:
- 通常のミックスではあまり使わない
- ライブ/PA向けの機能
- 使用する場合は最小限に


5. High Shelf:

機能:
設定周波数以上の全体を均一に増減

用途:
明るさの追加（ブースト）
刺さる高域の軽減（カット）
エアー感の付与
全体のトーン調整

設定:

Frequency:
変化が始まる周波数ポイント

Gain:
増減量

推奨:

明るくしたい:
8 kHz, +2 dB
理由: 8kHz以上全体を持ち上げ
     エアー感とプレゼンスの追加

暗くしたい:
6 kHz, -3 dB
理由: 高域全体を抑制
     暗い音色に
     デジタル感の軽減

エアー感:
10 kHz, +1.5 dB
理由: 超高域のみ持ち上げ
     空気感と高級感

De-Harsh:
5 kHz, -2 dB
理由: 耳に痛い帯域を抑制
     長時間聴きやすい音に

注意点:
- High Shelfでの大きなブーストは
  ノイズも持ち上げるため注意
- デジタル音源はShelf +1.5 dBで十分
- アナログ機材通過音はもう少し必要な場合あり


6. High Cut (Low Pass Filter):

機能:
設定周波数以上をカット
高域を段階的に除去

用途:
暗い音色の作成
不要な高域ノイズ除去
KickやBassから高域成分を除去
ビンテージ感の付与

設定:

Frequency:
カット開始点

Slope:
6/12/18/24/48 dB/oct

推奨:

Kick: 8 kHz (12 dB/oct)
理由: Kickに超高域は不要
     クリックノイズの除去
     他トラックとの干渉防止

Bass: 5 kHz (18 dB/oct)
理由: Bassに高域は不要
     フレットノイズの軽減
     Hi-Hat帯域との分離

Pad (暗い雰囲気): 10 kHz (12 dB/oct)
理由: 暗いPadの作成
     Techno的サウンド
     高域をHi-Hatに譲る

Sub Bass: 200 Hz (24 dB/oct)
理由: サブベースは低域のみ
     倍音は別レイヤーで
     超クリーンな低域

注意点:
- High CutはLow Passフィルターと同義
- 急峻なSlopeは位相回転が大きい
- カットし過ぎるとこもった音に
- A/B比較で適切な設定を確認
```

### 周波数帯域の特徴

各周波数帯域には固有の特徴があり、これを理解することがEQ操作の基礎となります。人間の可聴範囲は20Hz〜20kHzですが、年齢や聴力によって個人差があります。

```
20-60 Hz (Sub Bass):

特徴:
体で感じる周波数帯域
スピーカーの振動として体感
多くのイヤフォンでは再生困難
クラブのサウンドシステムで真価を発揮

楽器:
Kick（基音の下部）
Sub Bass（シンセサイザー）

処理:
High Pass 30-40 Hz で不要な超低域をカット
DCオフセットの除去
スピーカー保護のためにも必須

問題:
過剰: スピーカー損傷のリスク
不足: パワー感の欠如
共鳴: 部屋の定在波との干渉

モニタリング:
スペクトラムアナライザーで視覚確認
ヘッドフォンでの確認推奨
車のサブウーファーでの最終チェック


60-250 Hz (Low):

特徴:
パワーと温かみの帯域
楽曲の基盤となる周波数
太さの源泉

楽器:
Kick（基音: 60-100 Hz）
Bass（基音: 60-200 Hz）
Tom
ピアノ左手

処理:
濁りやすい帯域のため注意が必要
複数楽器が重なると即座にマスキング発生
KickとBassの周波数分離が最重要課題

問題:
過剰: ブーミー、こもった音
不足: スカスカ、軽い音
マスキング: KickとBassの共存問題

テクニック:
Kick中心: 60-80 Hz → Bassは100 Hz以上に
Bass中心: 80-120 Hz → Kickは60 Hz以下に
サイドチェインで動的分離


250-500 Hz (Low Mid):

特徴:
最も濁りやすい帯域
「ボックス感」「ダンボール感」の原因
全楽器がこの帯域にエネルギーを持つ

楽器:
全楽器が集中する問題帯域
特にSnare、Guitar、Vocal、Piano

処理:
-2 〜 -4 dBカットが定番
スペースを作るための「引き算」
広いQ (1-2)でカット

問題:
過剰: ボックス感、こもり、不明瞭
不足: 薄い、力のない音

テクニック:
スイープEQで最も不快な周波数を特定
通常350-400 Hz付近が問題になりやすい
各トラックで-2 dB程度カットするだけで劇的改善
「引き算のEQ」の代表的な帯域


500 Hz-2 kHz (Mid):

特徴:
存在感と前に出る力の帯域
人間の声の基本周波数帯域
耳が最も敏感な範囲に近い

楽器:
Vocal（基音: 300 Hz-1 kHz）
Snare（ボディ: 500 Hz-1 kHz）
Guitar（主要帯域）
シンセリード

処理:
+2 〜 +4 dBブーストで存在感追加
広いQ (1-2)でのブースト推奨
過剰ブーストは「鼻声」的になる

問題:
過剰: 鼻声的、疲れる音
不足: 存在感なし、引っ込む

テクニック:
Vocalは800 Hz-1.5 kHz付近で存在感確保
Snareは600-800 Hzでボディ感
Leadシンセは1-2 kHzで前に出す


2-5 kHz (Upper Mid):

特徴:
明瞭度とアタック感の帯域
人間の耳が最も敏感な周波数域
聴覚の等ラウドネス曲線のピーク付近
言葉の子音が集中する帯域

楽器:
Vocal（明瞭度: 2-4 kHz）
Snare（スナップ/クラック: 3-5 kHz）
Guitar（プレゼンス: 3-5 kHz）
シンベのアタック

処理:
+2 〜 +3 dBブーストでVocal明瞭度向上
Snareのスナップ感追加
過剰ブーストは耳に刺さる

問題:
過剰: 耳に痛い、歯擦音強調、疲れる
不足: 不明瞭、こもった印象

テクニック:
De-essing: 3-6 kHzの歯擦音を動的にカット
2-3 kHzの「プレゼンス・ディップ」テクニック
競合するトラック間で周波数をずらす


5-10 kHz (Presence / Brilliance):

特徴:
空気感と明るさの帯域
ハーモニクス（倍音）が豊富
シンバルのシミー感
エッジとディテール

楽器:
Hi-Hat（主要帯域）
Cymbals
シンセの倍音
Vocalのブレス

処理:
+1 〜 +2 dBブーストでわずかに明るく
過剰注意：刺さる音の原因
De-essing対象帯域

問題:
過剰: 刺さる、きつい、疲れる
不足: 暗い、こもった印象

テクニック:
High Shelfではなく narrow Peakで精密制御
Hi-Hatとの競合に注意
この帯域はモニタリング環境に大きく依存


10-20 kHz (Air):

特徴:
空気感と高級感の帯域
「存在していることすら気づかない」繊細な帯域
年齢とともに聴力が低下する帯域
ハイレゾ音源で差が出やすい

楽器:
シンバルのハーモニクス
ブレスノイズ
テープヒス
空気感

処理:
+0.5 〜 +1.5 dBでHigh Shelfブースト
Master EQでのわずかな追加が効果的
ノイズも持ち上がるため注意

問題:
過剰: ノイズ増加、不自然なシャリシャリ感
不足: 密閉感、息苦しさ

テクニック:
マスタリングでのエアー追加が定番
12 kHz付近のShelfブーストが最も効果的
Spectrum Analyzerでノイズレベルを確認
```

## EQ実践例

**トラック別設定:**

### Kick

```
目標:
パワフルで明確なKick
ミックスの中心として機能
Bassとの明確な分離

Band 1 (High Pass):
Type: Low Cut
Freq: 30 Hz
Slope: 24 dB/oct
理由: 不要な超低域をカット
     スピーカー保護
     DCオフセット除去
     クリーンなローエンド実現

Band 2 (Low Shelf):
Freq: 60 Hz
Gain: +3 dB
理由: サブベース域のパワー追加
     クラブでの体感を強化
     Kickの「腹」に来る感覚

Band 3 (Peak - カット):
Freq: 250 Hz
Gain: -3 dB
Q: 2.0
理由: ボックス感の除去
     ダンボール感の軽減
     Low-Mid域の濁りを削減

Band 4 (Peak - ブースト):
Freq: 3-5 kHz
Gain: +2 dB
Q: 1.5
理由: クリック感/アタックの強調
     ミックスでの存在感確保
     パンチの追加

Band 5 (High Cut):
Type: Low Pass
Freq: 10 kHz
Slope: 12 dB/oct
理由: 不要な高域をカット
     Hi-Hat帯域との干渉防止
     クリーンなKick音

Band 6 (Peak - オプション):
Freq: 100-120 Hz
Gain: +1 dB
Q: 1.5
理由: Kickの「パンチ」追加
     中低域の存在感
     Bassとの適切な分離

結果:
太く、明確で存在感のあるKick

A/B比較ポイント:
- バイパスして元の音と比較
- 低域のパワー感は適切か
- 250Hz付近のこもりは除去されたか
- アタックは明確に聴こえるか
```

### Bass

```
目標:
太くクリアなBass
Kickとの明確な分離
楽曲のグルーヴを支える

Band 1 (High Pass):
Freq: 40 Hz
Slope: 24 dB/oct
理由: サブベース以下の不要成分除去
     Kickのサブベース域を侵食しない

Band 2 (Low Shelf):
Freq: 80 Hz
Gain: +2 dB
理由: Bassの基音を強化
     太さと温かみの追加
     Kickとの共存（周波数をずらす）

Band 3 (Peak - カット):
Freq: 200-300 Hz
Gain: -2 dB
Q: 2.0
理由: Kickとの周波数分離
     ボックス感の除去
     Low-Mid域のクリーンアップ

Band 4 (Peak - ブースト):
Freq: 800-1200 Hz
Gain: +2 dB
Q: 1.5
理由: 小さいスピーカーでの存在感確保
     Bassの「抜け」を向上
     ミックス内での明瞭度

Band 5 (High Cut):
Freq: 5 kHz
Slope: 18 dB/oct
理由: Bassに高域は不要
     フレットノイズの除去
     Lead/Vocal帯域との干渉防止

Band 6 (Peak - オプション):
Freq: 500-600 Hz
Gain: -1.5 dB
Q: 2.5
理由: 中域の濁り防止
     Vocalとの住み分け

結果:
Kickと分離された明確なBass

テクニック補足:

KickとBassの周波数分離:

方法1: 周波数棲み分け
Kick: 60 Hz中心
Bass: 100 Hz中心
→ 互いの帯域を少しずつカット

方法2: サイドチェインEQ
Bass側でKickの基音帯域のみダッキング
→ 動的な分離

方法3: EQカーブの相補性
Kickでブーストした帯域 = Bassでカット
Bassでブーストした帯域 = Kickでカット
→ パズルのピースのように組み合わせる
```

```
明瞭で前に出るVocal
ミックスの主役として機能

     高級感のある仕上がり

Freq: 1-1.5 kHz
Gain: +1 dB
Q: 1.0

明瞭で自然なVocal

Dynamic EQ for De-essing:
静的なEQではなく、ダイナミックEQを使用

Vocal EQの順序:
1. High Pass（必須）
2. 濁り除去（200-400 Hz）

4. De-ess（6-8 kHz）

```

## Compressor 完全ガイド

**ダイナミクス制御:**

コンプレッサーは、音量差（ダイナミクスレンジ）を制御するエフェクトです。大きい音を抑え、全体の音量を均一化することで、安定したミックスを実現します。しかし、単に音を潰すだけのツールではなく、音色を形成し、パンチや存在感を付与する創造的なツールでもあります。

### コンプレッサーの基本原理

```
コンプレッサーの動作原理:

入力信号 → 検出回路 → 判定 → ゲインリダクション → 出力

1. 入力信号がThresholdを超える
2. Ratioに基づいてゲインを削減
3. Attackで圧縮開始速度を決定
4. Releaseで圧縮解除速度を決定
5. Make-up Gainで音量を補正

視覚的イメージ:

入力:    ████████████████████████
                    ↑ Threshold
圧縮後:  ████████████████████
              圧縮部分 ↓
              ■■■■■■■ (Gain Reduction)

Make-up: ██████████████████████████
         全体を持ち上げ
         → 小さい音が相対的に大きくなる
         → ダイナミクスが揃う
```

### パラメーター詳細

```
1. Threshold (スレッショルド):

機能:
圧縮が開始される音量レベル
このレベルを超えた信号のみが圧縮される

単位:
dB (デシベル)

設定:

高い (-5 dB):
ピーク部分のみ圧縮
少しだけ圧縮
自然な結果

低い (-30 dB):
ほぼ全ての信号を圧縮
多く圧縮
劇的な効果

推奨:

Kick: -12 dB
理由: パンチ維持しつつ安定化
     GR -4〜-6 dB目標

Bass: -15 dB
理由: しっかり安定化
     GR -6〜-9 dB目標

Vocal: -18 dB
理由: 全体的に安定
     小さい声も拾う
     GR -3〜-5 dB目標

Master: -6 dB
理由: ピーク部分のみ軽く圧縮
     ダイナミクス維持
     GR -2〜-4 dB目標

Pad: -10 dB
理由: 軽い圧縮で安定化
     GR -2〜-3 dB

Snare: -12 dB
理由: パンチを維持
     GR -4〜-6 dB

GR目標:
一般的に -3 〜 -6 dB が適切
それ以上は過剰圧縮のリスク


2. Ratio (レシオ):

機能:
圧縮の強さを決定する比率
ThresholdをXdB超えた信号をYdBに圧縮

例:
4:1の場合:
Thresholdを4 dB超えた信号
→ 1 dBに圧縮される

Thresholdを8 dB超えた信号
→ 2 dBに圧縮される

設定:

1:1 = 圧縮なし（バイパスと同等）
1.5:1 = 非常に軽い圧縮
2:1 = 軽い圧縮（マスタリング向け）
3:1 = 軽中程度（Vocal向け）
4:1 = 標準的な圧縮
6:1 = やや強い圧縮（Bass向け）
8:1 = 強い圧縮
10:1 = 非常に強い圧縮
20:1 = ほぼLimiter
∞:1 = 完全なLimiter（Ceiling超え不可）

推奨:

Kick: 4:1
理由: パンチを維持しつつ安定化
     アタックのダイナミクスを保存

Bass: 6:1 (強め)
理由: Bassは音量安定が最重要
     グルーヴの維持
     強めの圧縮でもBassは許容

Vocal: 3:1 (軽め)
理由: 自然さを維持
     過剰圧縮で不自然にしない
     2段コンプで分散

Drums全体: 3-4:1
理由: パンチ維持
     グルー感

Master: 2:1 (軽め)
理由: ダイナミクス維持が最優先
     わずかな接着効果

Pad: 2-3:1
理由: 軽い安定化で十分


3. Attack (アタック):

機能:
Thresholdを超えてから圧縮が最大になるまでの時間
圧縮の「反応速度」

単位:
ms (ミリ秒)

重要性:
★★★★★ (最重要パラメーター)
音色を最も大きく変化させる

設定:

速い (0.01-5 ms):
効果: トランジェント（アタック）を潰す
結果: タイトで制御された音
     パンチは減少
用途: Vocal安定化、De-essing

中間 (5-20 ms):
効果: トランジェントを通した後に圧縮
結果: パンチを維持しつつ安定化
     最もバランスの良い設定
用途: Kick、Snare、多くの楽器

遅い (20-100 ms):
効果: アタック部分は完全に通過
結果: パンチが強調される
     サスティン部分のみ圧縮
用途: Bass（グルーヴ維持）、実験的

非常に遅い (100+ ms):
効果: ほぼトランジェント全体を通過
結果: 最もパンチのある音
     音量制御としての効果は低下
用途: 特殊効果、パラレルコンプレッション

推奨:

Kick: 10 ms
理由: アタック（ビーター音）を通過させる
     パンチ維持が最重要
     5ms以下はアタック潰れに注意

Bass: 30 ms
理由: 指弾き/スラップのアタック保存
     グルーヴ感の維持
     ノートの立ち上がりを自然に

Vocal: 5 ms (速い)
理由: 音量の安定化が目的
     自然な印象を維持
     子音の突出を防止

Snare: 10-15 ms
理由: スティックのアタック保存
     パンチとスナップの維持

Drums Bus: 10 ms
理由: 全ドラムのパンチを維持
     グルー感の追加

Master: 10-30 ms
理由: トランジェント保存
     音楽的なダイナミクス維持

重要な注意:
Attack設定はA/B比較が必須
1-2msの違いで音が大きく変わる
「パンチが欲しい」= Attack遅く
「安定させたい」= Attack速く


4. Release (リリース):

機能:
信号がThreshold以下になった後、
圧縮が解除されるまでの時間

単位:
ms (ミリ秒)

重要性:
グルーヴ感を決定する重要パラメーター

設定:

速い (20-80 ms):
効果: 素早く元の音量に復帰
結果: リズミカルな「呼吸」感
     ポンピング効果（意図的なら○）
問題: 歪みの原因になることがある

中間 (80-200 ms):
効果: 適度な復帰速度
結果: 自然なダイナミクス制御
     最もバランスの良い設定

遅い (200-1000 ms):
効果: ゆっくり元に戻る
結果: 滑らかで自然な圧縮
     ダイナミクスの変化が穏やか
問題: 連続したピークで圧縮が戻らない

Auto:
効果: プログラムに応じて自動調整
結果: 多くの場合良好な結果
推奨: 初心者はAutoから始める

推奨:

Kick: 80 ms
理由: 次のKickまでに復帰
     タイトなグルーヴ
     BPM 128: 468ms/beat → 80msで十分

Bass: 100 ms
理由: ノート間でスムーズに復帰
     グルーヴ維持

Vocal: 40-60 ms
理由: 自然な音量追従
     文節の切れ目で復帰

Snare: 100-150 ms
理由: テールが自然に減衰
     次のスネアまでに復帰

Master: Auto
理由: 楽曲全体のテンポに追従
     最も安全な選択

Drums Bus: 100-200 ms
理由: グルー感の付与
     パンピング回避

Release Time計算目安:
60000 / BPM / 4 = 1/16音符 (ms)
BPM 128: 60000/128/4 = 117ms
→ 1/16音符ごとに復帰
→ 一般的に80-120msが適切


5. Knee (ニー):

機能:
Threshold付近での圧縮の始まり方

設定:

0 dB (Hard Knee):
効果: Thresholdを超えた瞬間から全力圧縮
結果: 明確で正確な圧縮
     アグレッシブな効果
用途: Drums、パーカッシブな素材

3-6 dB (Medium):
効果: Threshold付近で徐々に圧縮開始
結果: 自然な遷移
用途: 多くの楽器、バランス重視

6-12 dB (Soft Knee):
効果: Thresholdのかなり前から緩やかに圧縮開始
結果: 非常に自然で滑らかな圧縮
     圧縮していることが分かりにくい
用途: Vocal、マスタリング

推奨:

Kick: 0 dB (Hard)
理由: 明確なパンチ制御

Snare: 0-3 dB
理由: パンチ維持しつつやや自然に

Bass: 3-6 dB
理由: 自然なグルーヴ

Vocal: 6 dB (Soft)
理由: 最も自然な圧縮
     聴取者に圧縮を感じさせない

Master: 3-6 dB
理由: 全体を滑らかにまとめる

Pad: 6 dB (Soft)
理由: 緩やかな安定化


6. Gain (Make-up Gain):

機能:
圧縮によって下がった音量を補正

理由:
圧縮 = 音量が下がる
→ Make-up Gainで補正
→ 元の音量感に戻す
→ 「小さい音が相対的に大きくなる」効果

計算:

Auto:
自動でGR分を補正
手軽で便利

Manual:
GR -6 dB → Gain +6 dB
GR分を手動で補正
より精密な制御

推奨:
初心者: Auto推奨
中級者: Manual推奨

重要な注意:
Make-up Gainは音量を揃えるためのもの
「音が良くなった」と感じるのは
単に音量が大きくなっただけの可能性

A/B比較方法:
1. コンプ On: 音を聴く
2. コンプ Off: 音を聴く
3. 音量差がある場合は不公平な比較
4. Output Gainで音量を揃えてから比較
5. 音量が同じ状態で本当に良くなったか確認
```

### Peak vs RMS

```
Peak Mode:

検出方式:
信号のピーク値（瞬間最大値）を検出

特徴:
速い反応速度
トランジェントに正確に反応
短いピークも逃さない

用途:
Drums全般（Kick、Snare、Hi-Hat）
パーカッシブな素材
アタックの制御が重要な場合

メリット:
ピーク制御が正確
クリッピング防止に効果的

デメリット:
過剰反応することがある
音楽的でない場合がある


RMS Mode:

検出方式:
信号の実効値（平均的な音量）を検出

特徴:
滑らかな反応
全体的な音量に基づく
人間の聴覚に近い検出

用途:
Vocal
Bass
Pad
Master/Bus処理

メリット:
音楽的な圧縮
人間の聴感に近い
自然な結果

デメリット:
短いピークを見逃すことがある
ピーク制御としては不正確


推奨まとめ:

Kick: Peak
理由: アタックの正確な制御

Snare: Peak
理由: アタックの制御

Hi-Hat: Peak
理由: トランジェント制御

Bass: RMS
理由: 全体的な音量安定化

Vocal: RMS
理由: 自然な音量制御

Pad: RMS
理由: 滑らかな安定化

Master: RMS
理由: 全体的な音量制御

Drums Bus: Peak
理由: トランジェント制御


FF1 / FF2 (Feed-Forward):
Ableton Compressorの検出モード

FF2: より透明な検出

入力ではなく出力を検出に使用

自然で音楽的な圧縮
```

## サイドチェイン圧縮

**Kickで他をダッキング:**

サイドチェインコンプレッションは、EDM/ダンスミュージック制作において最も重要なテクニックの一つです。特にTechno/Houseでは、このテクニック無しにプロレベルのミックスは不可能と言っても過言ではありません。

### 基本概念

```
通常のCompressor:

入力信号 → 検出回路 → ゲインリダクション
（自分自身の信号で自分を圧縮）

Sidechain Compressor:

外部信号 (Kick) → 検出回路
自分の信号 (Bass) → ゲインリダクション
（Kickの信号でBassを圧縮）

効果:
Kickが鳴る瞬間 → Bassの音量が下がる
Kickが止む → Bassの音量が戻る
→ Kickのスペースが常に確保される

視覚的イメージ:

Kick:   █ . . . █ . . . █ . . . █ . . .
Bass:   ▁▅█████▁▅█████▁▅█████▁▅█████
         ↑       ↑       ↑       ↑
      Kickの瞬間Bassが下がる

結果:
「ポンポン」というリズミカルな効果
Kickが常に明確に聴こえる
低域の濁りが解消
グルーヴ感の強化

用途:
Techno: 必須テクニック
House: 必須テクニック
EDM全般: 非常に多用
Pop: 軽めに使用
```

### 設定方法

```
Ableton Liveでの設定手順:

1. Bass TrackにCompressor挿入

2. Compressor下部の三角マーク (Sidechain)
   → クリックして展開

3. Sidechain: On (有効化)

4. Audio From:
   → Kick Track を選択
   (ドロップダウンメニューから)

5. パラメーター設定:

Threshold: -24 dB (低め)
理由: Kickの全てのヒットで確実にダッキング

Ratio: 8:1 (強め)
理由: 明確なダッキング効果
     6:1〜10:1の範囲で調整

Attack: 0.1 ms (最速)
理由: Kickの瞬間に即座に反応
     遅いと効果が不十分

Release: 80-150 ms (テンポ依存)
理由: 次のKickまでに復帰
     短すぎ: 不自然なクリック音
     長すぎ: Bassが消えすぎ

     BPM別推奨Release:
     BPM 120: 120-150 ms
     BPM 128: 100-130 ms
     BPM 135: 80-110 ms
     BPM 140: 70-100 ms

Gain: +6-8 dB
理由: ダッキングで下がった分を補正

Knee: 0 dB (Hard)
理由: 明確なダッキング

6. GR確認:

Kick鳴る時:
GR -6 〜 -10 dB (明確にダッキング)

Kick無い時:
GR 0 dB (圧縮なし)

確認方法:
- GRメーターがKickに合わせて動くか
- Bassを聴いてポンピング効果があるか
- Kickが明確に聴こえるか

結果:
Kickのスペースが常に確保される
低域がクリーンになる
グルーヴ感が向上する
```

### サイドチェインEQ

```
機能:
サイドチェイン入力信号に対するEQ
特定の周波数帯域のみで検出させる

用途:
Kickの低域（基音）のみでダッキング
高域のKickクリック音には反応させない

設定:

1. Sidechain: On

2. EQ: On（Sidechainパネル内のEQボタン）

3. Sidechain EQ設定:
   High Pass: 30 Hz
   Low Pass: 150 Hz
   → Kickの基音帯域（60-100 Hz）のみを検出

効果:
低域のマスキングのみを解消
高域成分（キックのクリック）には反応しない
より自然なダッキング効果

推奨:
Bass: High Pass 30 Hz, Low Pass 200 Hz
Pad: High Pass 30 Hz, Low Pass 150 Hz
Lead: 通常のサイドチェインでOK

上級テクニック:

Mid/Side Sidechain:
低域（Mid）のみダッキング
高域（Side）はそのまま
→ ステレオ幅を維持しつつ低域分離

Multiband Sidechain:
帯域ごとに異なるサイドチェイン設定
→ より精密なダッキング
```

### サイドチェイン応用テクニック

```
1. Pad Ducking:

設定:
Kick → Pad (Sidechain)
Threshold: -20 dB
Ratio: 6:1
Attack: 0.5 ms
Release: 100-200 ms
GR: -4 〜 -6 dB

効果:
Padがリズミカルに呼吸する
Kickスペース確保
Trance/Progressive必須

2. Vocal Ducking:

設定:
Music → Vocal (Reverse Sidechain)
Vocal → Music (Sidechain)

効果:
Vocalが入る時、楽曲が少し下がる
Vocalの明瞭度が向上
ラジオDJスタイル

3. Reverb Ducking:

設定:
Dry Signal → Reverb Return (Sidechain)

効果:
音が鳴っている間: Reverb下がる
音が止んだ後: Reverb出現
→ 明瞭度維持 + 空間感
→ 非常に効果的なプロテクニック

4. Ghost Sidechain:

設定:
無音のKickトラック（Mute済み）
→ サイドチェインソースとして使用

効果:
実際のKickとは独立したダッキングパターン
より自由な設定が可能
KickパターンとSCパターンを分離
```

## よくある失敗と対処法

### 1. High Pass不足

```
問題:
ミックス全体が濁る
低域がモヤモヤ
パワー感がない（逆説的）

原因:
Low-Mid (200-400 Hz)に全トラックが混雑
High Passフィルターの未使用

解決:

全トラックにHigh Passを必ず適用:

Kick: 30 Hz (24 dB/oct)
Bass: 40 Hz (24 dB/oct)
Snare: 100 Hz (18 dB/oct)
Lead: 200 Hz (12 dB/oct)
Pad: 300 Hz (12 dB/oct)
Vocal: 80 Hz (18 dB/oct)
Hi-Hat: 400 Hz (24 dB/oct)
FX: 200-500 Hz (12 dB/oct)

効果:
劇的にクリアなミックスに
低域のパワー感が増す（逆説的だが真実）
各楽器の分離が向上

なぜ低域カットで音が良くなるのか:
- 不要な低域がスピーカーの出力を消費していた
- 低域のマスキングが解消される
- ヘッドルームが確保される
- Kick・Bassの低域がより明確に
```

### 2. コンプのAttack速すぎ

```
問題:
Kickのパンチがない
全体が平坦で退屈
ドラムが機械的

原因:
Attack: 0.1 ms
アタック（トランジェント）が完全に潰れている

解決:

Attack: 10 ms以上に設定
アタックを通過させる

Kick: 10 ms
Snare: 10-15 ms
Bass: 30 ms

A/B比較:
1. Attack: 0.1 ms で聴く
2. Attack: 10 ms で聴く
3. アタックの違いを確認
4. パンチの有無を確認

実験:
Attack: 0.1ms → 5ms → 10ms → 20ms → 50ms
段階的に変化させて最適値を見つける
```

### 3. EQブースト過剰

```
問題:
不自然な音
特定帯域が刺さる
全体のバランスが崩れる

原因:
+6 dB以上の過剰ブースト

解決:

ブースト上限:
最大 +3 〜 +4 dB

カット優先の哲学（引き算EQ）:
他のトラックをカット
→ 相対的にブーストと同じ効果
→より自然な結果

例:
Vocalを明るくしたい場合:

NG: Vocal High Shelf +6 dB
→ 不自然、ノイズ増加

OK: 他トラックの3-5 kHz を -2 dB
→ 相対的にVocalが明るく聴こえる
→ 自然で透明

推奨:
「引き算のEQ」を基本戦略とする
ブーストは最後の手段
カットで解決できないか常に考える
```

### 4. GR多すぎ

```
問題:
音が完全に潰れている
ダイナミクスがゼロ
「壁のような」平坦な音
小さい音と大きい音の差がない

原因:
GR -10 dB以上
Thresholdが低すぎ / Ratioが高すぎ

解決:

GR目標:
個別トラック: -3 〜 -6 dB
Bus: -2 〜 -4 dB
Master: -2 〜 -4 dB

ルール:
「少し足りない」くらいが正解
圧縮は控えめに、段階的に

パラレルコンプレッション:
強い圧縮をしたい場合は
Parallel（Dry/Wet）で原音とブレンド
→ 圧縮の効果 + ダイナミクス維持
```

### 5. A/B比較をしない

```
問題:
EQ/コンプの効果を正確に判断できない
「良くなった」と思い込む（音量差のため）

原因:
A/B比較を怠る
音量を揃えずに比較

解決:

正しいA/B比較手順:

1. EQ/コンプの設定を行う
2. Output Gainを調整して
   ON/OFFで音量差をなくす
3. ONで聴く（2-3秒）
4. OFFで聴く（2-3秒）
5. 本当に良くなっているか判断

重要:
人間は「音量が大きい」=「音が良い」と感じる
1-2 dBの音量差でも「良くなった」と錯覚
必ず音量を揃えてから比較すること

頻度:
EQ/コンプ設定のたびにA/B比較
最低でも5分に1回
```

## EQ・コンプの順序

**どちらが先？**

```
標準的な順序: EQ → Compressor

理由:
1. EQで不要帯域をカット（High Passなど）
2. コンプレッサーがクリーンな信号を処理
3. コンプが不要な低域に反応しない
4. より効率的なゲインリダクション

具体例:
1. EQ Eight: High Pass 80 Hz (Vocal)
2. EQ Eight: -3 dB @ 300 Hz (こもり除去)
3. Compressor: 3:1, -18 dB Threshold
→ コンプはクリーンな信号のみ処理


例外: Compressor → EQ

用途:
問題周波数がコンプで強調される場合
コンプ後にEQで最終調整

手順:
1. Compressor先に挿入
2. 圧縮後に問題周波数が目立つ
3. EQ後段で問題周波数をカット

具体例:
Vocal:
1. Compressor: 声の安定化
2. EQ: De-ess (6-8 kHz カット)
→ コンプで強調された歯擦音をEQで処理


上級テクニック: EQ → Comp → EQ

プロのVocalチェーン:
1. EQ Eight (1st): High Pass + こもり除去
2. Compressor: 音量安定化
3. EQ Eight (2nd): 明瞭度ブースト + エアー追加
→ 最も柔軟で精密な処理

推奨:
初心者: EQ → Comp（まずはこれ）
中級者: 両方試して耳で判断
上級者: EQ → Comp → EQ
```

## ダイナミックEQ

**コンプとEQのハイブリッド:**

```
ダイナミックEQとは:

通常のEQ:
常に一定のカット/ブースト

ダイナミックEQ:
信号レベルに応じてカット/ブーストが変化

例:
歯擦音（6 kHz）が閾値を超えた時のみカット
→ 通常時は6 kHzに影響なし
→ 歯擦音の瞬間のみ-6 dBカット

用途:

De-essing:
歯擦音の瞬間のみカット
通常のEQより自然

問題共鳴の制御:
特定ノートでのみ発生する共鳴
ダイナミックEQで動的に対処

マスタリング:
低域が過剰な瞬間のみカット
通常時は影響なし

Ableton Live:
ダイナミックEQ非搭載
サードパーティ推奨:
- FabFilter Pro-Q 3 (最推奨)
- TDR Nova (無料)

代替手法:
Multiband Dynamics使用
→ 帯域ごとのダイナミクス制御
→ 近い効果を実現可能
```

## マルチバンドコンプレッション基礎

```
マルチバンドコンプレッサーとは:

通常のコンプ:
全帯域を一括で圧縮

マルチバンドコンプ:
帯域を分割して独立に圧縮

Ableton Live: Multiband Dynamics

帯域分割例:
Band 1: 20-120 Hz (Low)
Band 2: 120-1000 Hz (Low-Mid)
Band 3: 1000-6000 Hz (Mid-High)
Band 4: 6000-20000 Hz (High)

用途:

マスタリング:
各帯域を独立制御
低域タイト + 高域スムーズ

問題解決:
特定帯域だけが突出する場合
その帯域のみ圧縮

Bass制御:
低域のみ強めに圧縮
中高域は自然に

注意:
初心者は通常のCompressorをまずマスター
マルチバンドは中級者以上の道具
設定が複雑で誤用のリスクが高い
```

## プロのEQ・コンプ戦略

```
トップダウンミキシング:

1. Master EQで全体のトーン設定
2. 各バスのEQ・コンプ設定
3. 個別トラックのEQ・コンプ設定

ボトムアップミキシング:

1. 各トラックのEQ・コンプ設定
2. バスの処理
3. マスター処理

推奨:
ハイブリッドアプローチ
1. マスターEQで大まかなトーン

4. その他の楽器
5. バス処理
6. マスター仕上げ

重要原則:
- カット優先（引き算のEQ）
- コンプは控えめに

- リファレンストラックを常に参照
- 耳を信じ、メーターは参考
- 定期的に休憩（耳の疲労防止）
- 複数の再生環境で確認
```

## まとめ

### EQ Eight

```
□ 全トラックにHigh Pass必須（最重要）
□ Low-Mid (250-500 Hz)カット優先
□ ブースト最大 +3 〜 +4 dB
□ Q値: カットは狭く正確に、ブーストは広く自然に
□ Spectrumで視覚確認しながら
□ 引き算EQを基本戦略とする
□ A/B比較を必ず行う
□ Sweeping EQで問題周波数を特定
```

### Compressor

```
□ GR: -3 〜 -6 dB（個別トラック）
□ Attack: アタック残す (10 ms+)が基本
□ Release: グルーヴ維持、テンポに連動
□ Ratio: 3-4:1が標準、目的に応じて調整
□ Peak/RMS: 素材に応じて選択
□ Knee: 素材の性質に合わせて
□ A/B比較必須（音量揃えて）
□ 2段コンプでVocal処理（プロテクニック）
```

### サイドチェイン

```
□ Kick → Bass (必須、Techno/House)
□ Kick → Pad (推奨)
□ Ratio: 8:1, Attack: 0.1 ms
□ GR: -6 〜 -10 dB
□ Release: テンポに合わせて調整
□ Sidechain EQ活用で自然な効果
□ Ghost Sidechainで独立制御
□ Reverb Duckingも試す
```

### 重要原則

```
□ Less is More（控えめが正解）
□ 引き算EQ優先（カット > ブースト）
□ コンプは控えめに（GR -3〜-6 dB）
□ 全てA/B比較（音量揃えて）
□ Spectrumで視覚確認
□ リファレンストラックと常に比較
□ 耳を信じる（メーターは参考）
□ 定期的に休憩を取る
□ 複数環境で確認（ヘッドフォン、スピーカー、車）
□ 毎日15分の耳トレーニング
```
