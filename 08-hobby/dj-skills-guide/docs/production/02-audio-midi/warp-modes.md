## Warpとは

**BPM同期の魔法:**

### 基本概念

```
Warpの仕組み:

元のサンプル:
140 BPM

プロジェクト:
128 BPM

Warp処理:
140 BPM → 128 BPM
タイムストレッチ

結果:
テンポは変わる
ピッチは変わらない (デフォルト)

従来の方法:

ピッチ変更:
BPM変わる = ピッチも変わる
140 BPM → 128 BPM = 音が低くなる

Warp:
BPM だけ変わる
ピッチは保持
```

### タイムストレッチの原理

```
グラニュラー合成:

原理:
音声を小さな粒（Grain）に分割
各粒を再配置
→ テンポだけ変更

テンポを遅くする場合:
粒と粒の間隔を広げる
→ 長い再生時間
→ ピッチは各粒が保持

テンポを速くする場合:
粒と粒の間隔を狭める
→ 短い再生時間
→ ピッチは各粒が保持

粒のサイズ (Grain Size):

大きい粒:
音質が良い
タイミング精度が低い
パッド・テクスチャ向き

小さい粒:
タイミング精度が高い
アーティファクトが出やすい
リズム素材向き

位相ボコーダー (Phase Vocoder):

Complex/Complex Proで使用:
周波数領域で処理
より高音質
CPU負荷高い

仕組み:
FFT (高速フーリエ変換)
周波数成分を分析
時間を変更
逆FFTで再合成
```

### Warp On/Off

```
Clip View > Warp:

☑ Warp (On)
プロジェクトBPMに同期

☐ Warp (Off)
元のテンポで再生

Auto-Warp:

Preferences > Record Warp Launch:
Auto-Warp Long Samples: ☑

効果:
長いサンプル (30秒以上) を
自動でWarp

短いサンプル (Kick等):
自動Warpなし
手動でOnにする
```

### Warpの基本パラメーター

```
Clip View の Warp関連パラメーター:

1. Warp On/Off:
   ☑ / ☐

2. Warp Mode:
   Beats / Tones / Texture /
   Re-Pitch / Complex / Complex Pro

3. Seg. BPM:
   サンプルの元BPM
   自動検出 or 手動入力

4. :2 / ×2 ボタン:
   BPMを半分 / 倍に
   間違った検出の修正に

5. Transpose:
   ピッチ変更 (半音単位)
   -48 〜 +48 st

6. Detune:
   微細ピッチ変更 (セント単位)
   -50 〜 +50 ct

7. Gain:
   Clip全体のゲイン
   -inf 〜 +6 dB

8. Start / End:
   再生範囲
   Warp Markerとは別

9. Loop:
   ループ On/Off
   ループ範囲
```

## Warp Mode 6種類

**音源別の使い分け:**

### 1. Beats Mode

```
用途:
ドラムループ
パーカッション
リズム重視

特徴:
トランジェント (アタック) を保持
グリッドに吸着

設定:

Preserve:
Transients (推奨)
または
Transients + Envelope

Granulation Resolution:
1/8 (デフォルト)

音質:
ドラムに最適
クリアなアタック

使用例:

Kick Loop: Beats
Hi-Hat Loop: Beats
Percussion: Beats
Drum Break: Beats

NG:

ボーカル: × (不自然)
ベース: △ (ケースバイケース)
```

### Beats Modeの詳細パラメーター

```
Preserve:

Transients:
トランジェントのみ保持
最も一般的
推奨設定

1/4:
4分音符ごとに保持
大きなグリッド

1/8:
8分音符ごとに保持
デフォルト

1/16:
16分音符ごとに保持
細かいリズム

1/32:
32分音符ごとに保持
非常に細かい

Transients + Loop:
テンポを遅くした場合
各セグメントをループ
→ リズムの隙間を埋める

Transients + Loop + Crossfade:
ループのつなぎ目をクロスフェード
より自然な音

テンポ変更時の動作:

テンポを速く:
セグメントが短くなる
→ 自然な音

テンポを遅く:
セグメントが長くなる
→ 隙間が発生

隙間の処理:
Off: 無音
Loop: セグメントをループ
Loop + CF: ループ + クロスフェード

大幅なテンポ変更 (例: 140→100 BPM):

Beats Modeの弱点:
アーティファクトが目立つ
→ Complexの方が良い場合あり

小さなテンポ変更 (±10 BPM):
Beats Modeで十分
最もクリアな音
```

### 2. Tones Mode

```
用途:
メロディ
ハーモニー
音程重視

特徴:
ピッチを保持
音楽的

設定:

Grain Size:
30-60 (デフォルト 44)

音質:
メロディに最適
自然な音程

使用例:

ボーカルメロディ: Tones
ストリングス: Tones
ブラス: Tones
ピアノ: Tones

NG:

ドラム: × (アタック鈍る)
```

### Tones Modeの詳細

```
Grain Sizeの影響:

小さい値 (10-30):
粒が小さい
タイミング精度高い
音質はやや劣る
速いフレーズに

大きい値 (50-100):
粒が大きい
音質が良い
タイミング精度低い
ゆっくりしたフレーズに

推奨値:

ボーカルメロディ: 40-60
ベースライン: 50-70
ストリングス: 60-80
ピアノ: 40-50
リード: 30-50

調整方法:

1. デフォルト値 (44) で聴く
2. アーティファクトがあれば:
   値を上下させて試す
3. ベストな値を見つける

Tones Mode vs Complex:

Tones:
CPU軽い
単純な音源に最適
モノフォニックに強い

Complex:
CPU重い
複雑な音源に最適
ポリフォニックに強い
```

### 3. Texture Mode

```
用途:
アンビエント
パッド
テクスチャ

特徴:
滑らかな変化
グラニュラー

設定:

Grain Size:
20-100

Flux:
0-100 (ランダム性)

音質:
柔らかい
エフェクト的

使用例:

Pad: Texture
アンビエント: Texture
SFX: Texture
ドローン: Texture

NG:

ドラム: × (ぼやける)
ボーカル: △ (特殊効果なら○)
```

### Texture Modeの詳細

```
Grain Size:

小さい値 (5-20):
細かい粒
テクスチャ感が強い
エフェクト的

中間値 (30-60):
バランス良い
一般的な使用

大きい値 (70-100):
大きい粒
元の音に近い
微細なテクスチャ

Flux (ランダム性):

0:
ランダム性なし
規則的なグラニュラー

50:
適度なランダム性
自然な変化

100:
最大のランダム性
予測不能な変化
クリエイティブ向け

クリエイティブな活用:

1. パッドの進化:
   Flux: 70-100
   → 常に変化するパッド
   → アンビエントに最適

2. テクスチャ生成:
   短い音源をTexture Mode
   テンポを大幅に遅く
   → 新しいテクスチャ

3. サウンドデザイン:
   Grain Size + Flux を自動化
   → 動的に変化するサウンド

4. グリッチエフェクト:
   Grain Size: 1-5
   → グリッチ・グラニュラー効果
```

### 4. Re-Pitch Mode

```
用途:
ターンテーブル的

特徴:
BPM変更 = ピッチ変更
Warpなし状態

設定:
なし (シンプル)

音質:
アナログ的
自然

使用例:

ヴィンテージ効果: Re-Pitch
DJ的なピッチ変更: Re-Pitch

NG:

通常の制作: × (BPM同期しない)
```

### Re-Pitch Modeの詳細

```
動作原理:

レコードプレーヤーと同じ:
速度を上げる → ピッチが上がる
速度を下げる → ピッチが下がる

計算:
元: 140 BPM
プロジェクト: 128 BPM
ピッチ変化: 128/140 = 0.914
→ 約 -1.5 半音

メリット:

1. CPU負荷ゼロ:
   タイムストレッチなし
   単純な速度変更

2. 音質劣化なし:
   アルゴリズム処理なし
   完全に自然な音

3. アナログ感:
   レコード/テープの質感
   温かみがある

デメリット:

1. ピッチが変わる:
   テンポ変更 = ピッチ変更
   キーが合わなくなる

2. 小さなテンポ変更のみ:
   ±5%程度が限界
   大幅な変更は不自然

活用シーン:

1. ヴァイナルDJ:
   レコードのピッチファインダーと同じ効果
   ±3%程度の微調整

2. テープエフェクト:
   テープの速度変化をシミュレート
   ヴィンテージ感

3. チョップドサンプル:
   短い単音サンプル
   ピッチ変更が味になる

4. Lo-Fi制作:
   意図的なピッチ変動
   テープワウフラッター風
```

```
用途:
複雑な音源
マルチトラック
ミックスダウン

特徴:
高音質
CPU負荷高い

設定:

なし (自動)

音質:
最高品質
万能

使用例:

フルミックス: Complex
ステムファイル: Complex
複雑なループ: Complex

推奨:

迷ったら Complex:
高音質保証

Complexが最強
```

### Complex Modeの詳細

```
アルゴリズム:

位相ボコーダー (Phase Vocoder):
FFT分析
周波数成分の保持
高品質な再合成

特徴:

1. ポリフォニック対応:
   複数の音が混在してもOK
   和音・ミックスに最適

2. 高い音質:
   アーティファクトが少ない
   自然なサウンド

3. 幅広いテンポ変更:
   ±50%でも高品質
   大幅な変更に強い

CPU負荷:

Beats: 低
Tones: 低
Texture: 中
Re-Pitch: 最低
Complex: 高
Complex Pro: 最高

対策:
1. 必要なトラックのみComplex
2. 確定したらFreeze/Flatten
3. ドラムはBeatsで十分

使い分け:

Complex が必要な場面:
- フルミックスのWarp
- ステムファイル
- 複雑なハーモニー
- 大幅なテンポ変更

Complex が不要な場面:
- 単純なドラムループ → Beats
- 単一メロディ → Tones
- パッド → Texture
```

```
用途:
Complexの上位版
ピッチ変更に対応

特徴:
最高音質
Formant保持可能

設定:

Formants:
100 (デフォルト)

Envelope:
100 (デフォルト)

```

### Complex Pro Modeの詳細

```
Formants パラメーター:

Formant（フォルマント）とは:
声の音色を決める共鳴特性
母音の識別に関わる

「あ」「い」「う」「え」「お」:
→ 異なるフォルマント特性

ピッチ変更時の問題:

通常のピッチシフト:
ピッチ上げ → フォルマントも上がる
→ チップマンク効果（声が小さくなる）

ピッチ下げ → フォルマントも下がる
→ 声が太く不自然に

Complex Pro:
ピッチ変更してもフォルマント保持
→ 自然な声質を維持

Formants値:

0:
フォルマント補正なし
= 通常のピッチシフト

100 (デフォルト):
完全なフォルマント保持
ボーカルに最適

中間値 (30-70):
部分的な補正
クリエイティブな効果

Envelope パラメーター:

エンベロープの追従性:

低い値:
スペクトルエンベロープが平坦
→ ぼやけた音

高い値:
スペクトルエンベロープが忠実
→ 鮮明な音

推奨: 100-128

Transpose との組み合わせ:

例1: ボーカルを5半音上げ
Mode: Complex Pro
Formants: 100
Transpose: +5

結果:
ピッチは上がるが声質は保持
チップマンク効果なし

例2: ボーカルを3半音下げ
Mode: Complex Pro
Formants: 100
Transpose: -3

結果:
ピッチは下がるが声質は保持
自然な低い声

例3: 性別変換効果
Formants: 0 にして
Transpose: +12 → 女性的
Transpose: -12 → 男性的
```

## Warp Marker

**細かいタイミング調整:**

### Warp Markerとは

```
定義:
波形上の基準点

用途:
タイミング修正
手動グリッド合わせ

表示:

Clip View > Waveform:

     ↓ ← Warp Marker
~~~~~|~~~~~/\~~~~~~
     1.1.1

黄色い縦線:
グリッド位置

操作:

ダブルクリック:
Warp Marker 作成

ドラッグ:
位置調整

Cmd+クリック:
削除
```

### Warp Markerの種類

```
1. Start Marker (灰色):
   クリップの再生開始位置
   固定

2. Auto Warp Marker (灰色):
   自動生成されたマーカー
   Abletonが自動配置

3. User Warp Marker (黄色):
   ユーザーが手動配置
   最も重要

4. Transient Marker (灰色・小):
   トランジェント位置
   自動検出

マーカーの優先度:
User > Auto > Transient

User Markerを配置すると:
その位置が確定
周囲のAutoMarkerが調整される
```

### Warp Marker配置

```
基本フロー:

1. Warp: On

2. 最初のビートを探す

3. その位置で ダブルクリック
   → Warp Marker 作成

4. その Marker を 1.1.1 にドラッグ

5. 次のビート (1.2.1) を探す

6. ダブルクリック → ドラッグ

7. 繰り返し

自動配置:

多くの場合:
Abletonが自動で配置

手動調整:
ズレている箇所のみ修正

複数Marker:

ドラムループ:
4-8箇所

ボーカル:
フレーズごと
```

### 効率的なWarp Marker配置テクニック

```
テクニック1: アンカーポイント戦略

手順:
1. まず曲の先頭 (1.1.1) を固定
2. 曲の中間点を固定
3. 曲の終わり近くを固定
4. 各セクション間を細分化

理由:
大きな区間から固定
→ 全体のズレを防ぐ
→ 細部を詰める

テクニック2: ダウンビート優先

手順:
1. 各小節の1拍目にMarker
2. 再生して確認
3. ズレている拍にMarker追加

理由:
1拍目が合えば全体のグルーヴが保たれる
細かいズレは後から修正

テクニック3: 右クリックメニュー活用

Set 1.1.1 Here:
選択位置を1.1.1に設定
曲の先頭を正確に合わせる

Warp From Here:
選択位置から自動Warp
BPM再検出

Warp From Here (Straight):
選択位置から一定BPMでWarp
テンポ一定の素材に

Warp From Here (Start At...):
選択位置を指定拍に設定

## Warpの品質を最大化する

### 高品質Warpのための設定

```
Hi-Quality Mode:

設定:
Clip View > HiQ ボタン

効果:
より高品質なアルゴリズム
CPU負荷増加

推奨:
重要なサンプル: On
大量のサンプル: Off (必要な時だけ)

RAM Mode:

設定:
Clip View > RAM ボタン

効果:
サンプルをRAMに読み込み
ディスクアクセスなし
より安定した再生

推奨:
重要なサンプル: On
ディスク速度が遅い場合: On
```

### テンポ変更幅とWarp品質の関係

```
テンポ変更の目安:

±5% (例: 128 → 134 BPM):
どのモードでも高品質
違いがほぼわからない

±10% (例: 128 → 140 BPM):
適切なモード選択で高品質
Complex/Complex Proが安全

±20% (例: 128 → 154 BPM):
Complex/Complex Pro推奨
他モードはアーティファクト

±30%以上 (例: 128 → 170 BPM):
Complex Pro必須
それでもアーティファクトあり
再録音を検討

±50%以上:
極端な変更
サウンドデザイン的
Texture Modeも面白い

品質を保つためのTips:

1. 元サンプルの品質:
   24bit/48kHz以上が理想
   MP3は避ける

2. モノ vs ステレオ:
   モノの方がWarp品質高い
   不要なステレオ情報を排除

3. Normalize:
   Warp前に適切なレベル
   小さすぎる音はノイズが目立つ

4. ドライ音:
   リバーブ等が少ない方が
   Warp品質が高い
```

## ワープマーカー微調整の上級テクニック

### サブビート精度でのマーカー配置

```
通常のマーカー配置:
  拍の頭（ダウンビート）に配置
  → 大まかな同期

上級テクニック:
  裏拍・ゴーストノートにも配置
  → 極めて精密な同期

手順:

1. ズームレベルを最大にする:
   Cmd + '+' を何度も押す
   波形の細部が見える状態に

2. 波形のトランジェントを目視:
   アタック部分の立ち上がり
   = 鋭いピーク

3. トランジェントの「直前」にマーカーを配置:
   音が立ち上がる0.5-2ms前
   → 再生時にアタックが正確に鳴る

4. グリッドラインとの関係:
   16分音符グリッド（Cmd+1で細かく）
   マーカーをグリッドに吸着

注意:
  細かすぎるマーカー配置は逆効果
  → 音がブツブツ切れる
  → 必要な箇所だけに限定する

目安:
  4小節ループの場合
  基本: 各拍頭（16箇所）
  追加: 問題のあるビート間のみ（+4-8箇所）
  合計: 20-24箇所が上限
```

### スウィング・グルーヴの保持

```
問題:
  スウィング感のある素材を
  ストレートグリッドにWarpすると
  グルーヴが失われる

解決策1: スウィンググリッドの使用

  手順:
  1. グリッドを16分音符に設定
  2. スウィング量を確認:
     裏拍がどれだけ遅れているか
  3. Warp Markerを裏拍の「実際の位置」に
  4. グリッドではなく聴感で位置を決定

解決策2: Groove Pool の活用

  手順:
  1. Warpした素材をグリッドに合わせる
  2. Groove Poolからグルーヴテンプレート適用
  3. Groove量を調整（50-100%）
  4. → グリッドに合いつつスウィング感を復元

解決策3: 部分的Warp

  手順:
  1. 拍の頭のみWarp Marker配置
  2. 拍間のタイミングは触らない
  3. → 大枠はグリッド同期
  4. → 拍間のグルーヴは元のまま

プロのアドバイス:
  「完璧にグリッドに合わせる」のが
  必ずしも正解ではない

  人間の演奏のグルーヴこそが魅力
  微妙な揺れを残すことで生命感が出る
```

### 複数トラック間のWarp整合性

```
問題:
  マルチトラック録音をWarpする際
  各トラックのWarp Markerがバラバラだと
  位相がズレて音が痩せる

例:
  ドラムのマルチトラック録音:
  キックマイク / スネアマイク / OH（左右）
  → 各トラックを個別にWarpすると位相崩壊

解決策:

方法1: グループWarp
  1. 全トラックを選択
  2. 1つのトラックにWarp Marker配置
  3. そのマーカー位置を他トラックにコピー
  → 全トラックが同じタイミングで伸縮

方法2: バスにまとめてからWarp
  1. マルチトラックをバウンス（ミックスダウン）
  2. 1つのステレオファイルとしてWarp
  → 位相問題を回避

方法3: Edit > Paste Time
  1. 基準トラックのWarp Marker位置をコピー
  2. 他トラックにペースト
  3. 位置関係が保持される

推奨:
  ドラムマルチトラック → 方法2が最も安全
  ステム分離した素材 → 方法1で対応
  ライブ録音 → 方法1+手動微調整
```

## 複雑なテンポ変化への対応

### アッチェレランド / リタルダンドのある素材

```
アッチェレランド（加速）:
  テンポが徐々に速くなる演奏

リタルダンド（減速）:
  テンポが徐々に遅くなる演奏

対処法:

1. セクション分割アプローチ:

   テンポが変化する区間を特定
   → 4-8小節ごとにWarp Marker配置
   → 各区間の実際のテンポに合わせる

   例:
   小節1-4: BPM 120
   小節5-8: BPM 123（少し加速）
   小節9-12: BPM 126（さらに加速）
   小節13-16: BPM 120（元に戻る）

2. 密なマーカー配置:

   各小節の1拍目に必ずマーカー
   加速/減速が急な箇所は
   各拍にもマーカー配置

   作業量は増えるが最も精密

3. Warp From Here の活用:

   テンポが変わる境目で
   右クリック > Warp From Here
   → その地点から新しいテンポで再計算

   複数回使用可能:
   区間A → Warp From Here
   区間B → Warp From Here
   区間C → Warp From Here
```

### 変拍子素材のWarp

```
変拍子とは:
  4/4拍子以外の拍子
  例: 3/4, 5/4, 7/8, 6/8

問題:
  Abletonのグリッドは4/4基準
  変拍子素材のBPM検出が狂いやすい

対処法:

1. 拍子設定の変更:
   Scene/Song Time Signatureを変更
   例: 3/4 に設定
   → グリッドが3拍子に

2. 手動BPM計算:

   3/4拍子の場合:
   1小節 = 3拍
   4/4換算BPM = 実際BPM × (3/4)

   例: 3/4で120 BPM = 4/4換算で90 BPM

   Seg. BPMに手動入力

3. 混合拍子（例: 7/8）:

   7/8 = 4+3 or 3+4 の組み合わせ
   Warp Markerで各拍を手動配置

   グリッドに完全に合わせるのは困難
   → セクション単位で大まかに合わせる

4. ポリリズム素材:

   3連符と4連符が混在
   → 基準となるリズムを決める
   → そのリズムに合わせてWarp
   → 他のリズムは自然にズレる（味）

実践Tips:
  変拍子素材はWarpせず
  プロジェクトのテンポを素材に合わせる
  という選択肢も常に検討する
```

## トラブルシューティング詳細

### 問題別の解決フローチャート

```
問題1: Warp後に音がカチカチ・プチプチする

原因:
  a) Warp Markerが近すぎる
  b) Grain Sizeが小さすぎる（Tones/Texture）
  c) テンポ変更幅が大きすぎる

解決:
  a) 不要なMarkerを削除（Cmd+クリック）
  b) Grain Sizeを大きく（+10-20）
  c) Complex / Complex Pro に変更

問題2: 音が「水中にいるような」感じになる

原因:
  位相ボコーダーのアーティファクト
  Complex / Complex Pro で発生しやすい

解決:
  1. HiQモードをOnにする
  2. テンポ変更幅を確認（大きすぎないか）
  3. Tones Modeを試す（単純な音源の場合）
  4. Envelope値を上げる（Complex Proの場合）

問題3: ドラムのアタックがぼやける

原因:
  Complex / Tones / Texture を使用中
  これらはトランジェント保持が弱い

解決:
  Beats Modeに変更
  Preserve: Transients

問題4: テンポが2倍/半分で検出される

原因:
  Abletonのビート検出アルゴリズムの限界
  キックが弱い曲やシンコペーションで発生

解決:
  :2 / ×2 ボタンで修正
  または Seg. BPM に正しい値を手動入力

問題5: Warp後にステレオ感が失われる

原因:
  位相ボコーダーによる左右チャンネルの
  微妙なズレの消失

解決:
  1. Complex Pro を使用
  2. Envelope値を120-128に上げる
  3. Warp後にステレオイメージャーで補正
  4. 元素材のステレオ幅が十分か確認

問題6: 長い楽曲で後半になるほどズレが大きくなる

原因:
  元素材のBPMが微妙に揺れている
  または Seg. BPM が0.1-0.5 BPMずれている

解決:
  1. Seg. BPMを小数点以下で微調整
     例: 128.00 → 127.85
  2. 曲の後半にもWarp Marker追加
  3. 30秒〜1分ごとにアンカーポイント配置
  4. Warp From Here で区間ごとに再計算
```

### CPU負荷の最適化テクニック

```
症状:
  多数のWarpトラックでCPUが限界
  ドロップアウト・ノイズ発生

段階的な対処:

Level 1（軽度）:
  1. Audio Buffer Size を上げる:
     128 → 256 → 512 → 1024
  2. 不要なトラックをMute

Level 2（中度）:
  3. Complex Pro → Complex に変更（可能な箇所）
  4. Complex → Tones/Beats に変更（適切な箇所）
  5. HiQモードをOffにする（最終ミックス時のみOn）

Level 3（重度）:
  6. 確定したトラックをFreeze:
     右クリック > Freeze Track
     CPU負荷がゼロに
  7. さらにFlatten:
     右クリック > Flatten
     Warp情報が消えて純粋なオーディオに

Level 4（最終手段）:
  8. Export して再読み込み:
     Warpした音をバウンス
     新しいオーディオファイルとして読み込み
  9. サンプルレートを下げる（非推奨だが効果大）

予防策:
  プロジェクト開始時にテンプレート作成
  よく使うサンプルは事前にFlatten済みを用意
  1プロジェクト内のComplex Proは5-8トラック以下に
```

## Warp機能の内部処理と最適化の深掘り

### サンプルレートとWarp品質の関係

```
サンプルレート別の品質差:

44.1kHz（CD品質）:
  標準的なWarp品質
  ほとんどの用途で十分
  CPU負荷: 標準

48kHz（映像標準）:
  44.1kHzとほぼ同等
  映像制作との互換性

88.2kHz / 96kHz:
  Warpアルゴリズムの精度向上
  高域のアーティファクト減少
  CPU負荷: 約2倍

192kHz:
  理論上最高品質
  実用上はオーバースペック
  CPU負荷: 約4倍

推奨:
  制作: 48kHz（バランス良い）
  マスタリング品質: 96kHz
  DJ用途: 44.1kHz で十分

ビット深度:
  16bit: 最低限（MP3からの変換等）
  24bit: 推奨（Warp処理の精度向上）
  32bit float: 内部処理で使用

重要:
  Warp処理は内部で32bit float
  元素材が24bitあれば品質は十分
  16bitでも実用上の問題は少ない
```

### Warp処理のレイテンシーとライブパフォーマンス

```
Warpのレイテンシー:

Beats Mode:
  レイテンシー: 最小
  ライブパフォーマンス向き

Tones Mode:
  レイテンシー: 小
  Grain Size に依存

Texture Mode:
  レイテンシー: 中
  Grain Size + Flux に依存

Re-Pitch Mode:
  レイテンシー: ゼロ
  処理なしのため

Complex Mode:
  レイテンシー: 大（FFT処理のため）
  ライブでは注意が必要

Complex Pro Mode:
  レイテンシー: 最大
  ライブ使用時はFreeze推奨

ライブパフォーマンスでの対策:

1. Buffer Size:
   256 samples が推奨バランス
   128 samples: 低レイテンシーだがCPU厳しい
   512 samples: 安定だが遅延感あり

2. 事前Freeze:
   Complex / Complex Pro トラックは
   セット前にFreeze しておく

3. Mode選択:
   ライブ中のリアルタイム変更は
   Beats / Tones / Re-Pitch が安全

4. 同時再生トラック数:
   Warpトラックは10-15本を上限に
   それ以上はFreeze/Flattenで対応
