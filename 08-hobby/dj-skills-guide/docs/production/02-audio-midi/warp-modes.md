## Warpとは

**BPM同期の魔法:**

### 基本概念

```
Warpの仕組み:

元のサンプル:
140 BPM

プロジェクト:
128 BPM

Warp処理:
140 BPM → 128 BPM
タイムストレッチ

結果:
テンポは変わる
ピッチは変わらない (デフォルト)

従来の方法:

ピッチ変更:
BPM変わる = ピッチも変わる
140 BPM → 128 BPM = 音が低くなる

Warp:
BPM だけ変わる
ピッチは保持
```

### タイムストレッチの原理

```
グラニュラー合成:

原理:
音声を小さな粒（Grain）に分割
各粒を再配置
→ テンポだけ変更

テンポを遅くする場合:
粒と粒の間隔を広げる
→ 長い再生時間
→ ピッチは各粒が保持

テンポを速くする場合:
粒と粒の間隔を狭める
→ 短い再生時間
→ ピッチは各粒が保持

粒のサイズ (Grain Size):

大きい粒:
音質が良い
タイミング精度が低い
パッド・テクスチャ向き

小さい粒:
タイミング精度が高い
アーティファクトが出やすい
リズム素材向き

位相ボコーダー (Phase Vocoder):

Complex/Complex Proで使用:
周波数領域で処理
より高音質
CPU負荷高い

仕組み:
FFT (高速フーリエ変換)
周波数成分を分析
時間を変更
逆FFTで再合成
```

### Warp On/Off

```
Clip View > Warp:

☑ Warp (On)
プロジェクトBPMに同期

☐ Warp (Off)
元のテンポで再生

Auto-Warp:

Preferences > Record Warp Launch:
Auto-Warp Long Samples: ☑

効果:
長いサンプル (30秒以上) を
自動でWarp

短いサンプル (Kick等):
自動Warpなし
手動でOnにする
```

### Warpの基本パラメーター

```
Clip View の Warp関連パラメーター:

1. Warp On/Off:
   ☑ / ☐

2. Warp Mode:
   Beats / Tones / Texture /
   Re-Pitch / Complex / Complex Pro

3. Seg. BPM:
   サンプルの元BPM
   自動検出 or 手動入力

4. :2 / ×2 ボタン:
   BPMを半分 / 倍に
   間違った検出の修正に

5. Transpose:
   ピッチ変更 (半音単位)
   -48 〜 +48 st

6. Detune:
   微細ピッチ変更 (セント単位)
   -50 〜 +50 ct

7. Gain:
   Clip全体のゲイン
   -inf 〜 +6 dB

8. Start / End:
   再生範囲
   Warp Markerとは別

9. Loop:
   ループ On/Off
   ループ範囲
```

## Warp Mode 6種類

**音源別の使い分け:**

### 1. Beats Mode

```
用途:
ドラムループ
パーカッション
リズム重視

特徴:
トランジェント (アタック) を保持
グリッドに吸着

設定:

Preserve:
Transients (推奨)
または
Transients + Envelope

Granulation Resolution:
1/8 (デフォルト)

音質:
ドラムに最適
クリアなアタック

使用例:

Kick Loop: Beats
Hi-Hat Loop: Beats
Percussion: Beats
Drum Break: Beats

NG:

ボーカル: × (不自然)
ベース: △ (ケースバイケース)
```

### Beats Modeの詳細パラメーター

```
Preserve:

Transients:
トランジェントのみ保持
最も一般的
推奨設定

1/4:
4分音符ごとに保持
大きなグリッド

1/8:
8分音符ごとに保持
デフォルト

1/16:
16分音符ごとに保持
細かいリズム

1/32:
32分音符ごとに保持
非常に細かい

Transients + Loop:
テンポを遅くした場合
各セグメントをループ
→ リズムの隙間を埋める

Transients + Loop + Crossfade:
ループのつなぎ目をクロスフェード
より自然な音

テンポ変更時の動作:

テンポを速く:
セグメントが短くなる
→ 自然な音

テンポを遅く:
セグメントが長くなる
→ 隙間が発生

隙間の処理:
Off: 無音
Loop: セグメントをループ
Loop + CF: ループ + クロスフェード

大幅なテンポ変更 (例: 140→100 BPM):

Beats Modeの弱点:
アーティファクトが目立つ
→ Complexの方が良い場合あり

小さなテンポ変更 (±10 BPM):
Beats Modeで十分
最もクリアな音
```

### 2. Tones Mode

```
用途:
メロディ
ハーモニー
音程重視

特徴:
ピッチを保持
音楽的

設定:

Grain Size:
30-60 (デフォルト 44)

音質:
メロディに最適
自然な音程

使用例:

ボーカルメロディ: Tones
ストリングス: Tones
ブラス: Tones
ピアノ: Tones

NG:

ドラム: × (アタック鈍る)
```

### Tones Modeの詳細

```
Grain Sizeの影響:

小さい値 (10-30):
粒が小さい
タイミング精度高い
音質はやや劣る
速いフレーズに

大きい値 (50-100):
粒が大きい
音質が良い
タイミング精度低い
ゆっくりしたフレーズに

推奨値:

ボーカルメロディ: 40-60
ベースライン: 50-70
ストリングス: 60-80
ピアノ: 40-50
リード: 30-50

調整方法:

1. デフォルト値 (44) で聴く
2. アーティファクトがあれば:
   値を上下させて試す
3. ベストな値を見つける

Tones Mode vs Complex:

Tones:
CPU軽い
単純な音源に最適
モノフォニックに強い

Complex:
CPU重い
複雑な音源に最適
ポリフォニックに強い
```

### 3. Texture Mode

```
用途:
アンビエント
パッド
テクスチャ

特徴:
滑らかな変化
グラニュラー

設定:

Grain Size:
20-100

Flux:
0-100 (ランダム性)

音質:
柔らかい
エフェクト的

使用例:

Pad: Texture
アンビエント: Texture
SFX: Texture
ドローン: Texture

NG:

ドラム: × (ぼやける)
ボーカル: △ (特殊効果なら○)
```

### Texture Modeの詳細

```
Grain Size:

小さい値 (5-20):
細かい粒
テクスチャ感が強い
エフェクト的

中間値 (30-60):
バランス良い
一般的な使用

大きい値 (70-100):
大きい粒
元の音に近い
微細なテクスチャ

Flux (ランダム性):

0:
ランダム性なし
規則的なグラニュラー

50:
適度なランダム性
自然な変化

100:
最大のランダム性
予測不能な変化
クリエイティブ向け

クリエイティブな活用:

1. パッドの進化:
   Flux: 70-100
   → 常に変化するパッド
   → アンビエントに最適

2. テクスチャ生成:
   短い音源をTexture Mode
   テンポを大幅に遅く
   → 新しいテクスチャ

3. サウンドデザイン:
   Grain Size + Flux を自動化
   → 動的に変化するサウンド

4. グリッチエフェクト:
   Grain Size: 1-5
   → グリッチ・グラニュラー効果
```

### 4. Re-Pitch Mode

```
用途:
ターンテーブル的

特徴:
BPM変更 = ピッチ変更
Warpなし状態

設定:
なし (シンプル)

音質:
アナログ的
自然

使用例:

ヴィンテージ効果: Re-Pitch
DJ的なピッチ変更: Re-Pitch

NG:

通常の制作: × (BPM同期しない)
```

### Re-Pitch Modeの詳細

```
動作原理:

レコードプレーヤーと同じ:
速度を上げる → ピッチが上がる
速度を下げる → ピッチが下がる

計算:
元: 140 BPM
プロジェクト: 128 BPM
ピッチ変化: 128/140 = 0.914
→ 約 -1.5 半音

メリット:

1. CPU負荷ゼロ:
   タイムストレッチなし
   単純な速度変更

2. 音質劣化なし:
   アルゴリズム処理なし
   完全に自然な音

3. アナログ感:
   レコード/テープの質感
   温かみがある

デメリット:

1. ピッチが変わる:
   テンポ変更 = ピッチ変更
   キーが合わなくなる

2. 小さなテンポ変更のみ:
   ±5%程度が限界
   大幅な変更は不自然

活用シーン:

1. ヴァイナルDJ:
   レコードのピッチファインダーと同じ効果
   ±3%程度の微調整

2. テープエフェクト:
   テープの速度変化をシミュレート
   ヴィンテージ感

3. チョップドサンプル:
   短い単音サンプル
   ピッチ変更が味になる

4. Lo-Fi制作:
   意図的なピッチ変動
   テープワウフラッター風
```

### 5. Complex Mode

```
用途:
複雑な音源
マルチトラック
ミックスダウン

特徴:
高音質
CPU負荷高い

設定:

なし (自動)

音質:
最高品質
万能

使用例:

フルミックス: Complex
ステムファイル: Complex
複雑なループ: Complex

推奨:

迷ったら Complex:
高音質保証

CPU余裕:
Complexが最強
```

### Complex Modeの詳細

```
アルゴリズム:

位相ボコーダー (Phase Vocoder):
FFT分析
周波数成分の保持
高品質な再合成

特徴:

1. ポリフォニック対応:
   複数の音が混在してもOK
   和音・ミックスに最適

2. 高い音質:
   アーティファクトが少ない
   自然なサウンド

3. 幅広いテンポ変更:
   ±50%でも高品質
   大幅な変更に強い

CPU負荷:

Beats: 低
Tones: 低
Texture: 中
Re-Pitch: 最低
Complex: 高
Complex Pro: 最高

対策:
1. 必要なトラックのみComplex
2. 確定したらFreeze/Flatten
3. ドラムはBeatsで十分

使い分け:

Complex が必要な場面:
- フルミックスのWarp
- ステムファイル
- 複雑なハーモニー
- 大幅なテンポ変更

Complex が不要な場面:
- 単純なドラムループ → Beats
- 単一メロディ → Tones
- パッド → Texture
```

### 6. Complex Pro Mode

```
用途:
Complexの上位版
ピッチ変更に対応

特徴:
最高音質
Formant保持可能

設定:

Formants:
100 (デフォルト)
ボーカルの音色保持

Envelope:
100 (デフォルト)

音質:
最高峰

使用例:

CPU重い
必要な場合のみ
```

### Complex Pro Modeの詳細

```
Formants パラメーター:

Formant（フォルマント）とは:
声の音色を決める共鳴特性
母音の識別に関わる

「あ」「い」「う」「え」「お」:
→ 異なるフォルマント特性

ピッチ変更時の問題:

通常のピッチシフト:
ピッチ上げ → フォルマントも上がる
→ チップマンク効果（声が小さくなる）

ピッチ下げ → フォルマントも下がる
→ 声が太く不自然に

Complex Pro:
ピッチ変更してもフォルマント保持
→ 自然な声質を維持

Formants値:

0:
フォルマント補正なし
= 通常のピッチシフト

100 (デフォルト):
完全なフォルマント保持
ボーカルに最適

中間値 (30-70):
部分的な補正
クリエイティブな効果

Envelope パラメーター:

エンベロープの追従性:

低い値:
スペクトルエンベロープが平坦
→ ぼやけた音

高い値:
スペクトルエンベロープが忠実
→ 鮮明な音

推奨: 100-128

Transpose との組み合わせ:

例1: ボーカルを5半音上げ
Mode: Complex Pro
Formants: 100
Transpose: +5

結果:
ピッチは上がるが声質は保持
チップマンク効果なし

例2: ボーカルを3半音下げ
Mode: Complex Pro
Formants: 100
Transpose: -3

結果:
ピッチは下がるが声質は保持
自然な低い声

例3: 性別変換効果
Formants: 0 にして
Transpose: +12 → 女性的
Transpose: -12 → 男性的
```

## Warp Marker

**細かいタイミング調整:**

### Warp Markerとは

```
定義:
波形上の基準点

用途:
タイミング修正
手動グリッド合わせ

表示:

Clip View > Waveform:

     ↓ ← Warp Marker
~~~~~|~~~~~/\~~~~~~
     1.1.1

黄色い縦線:
グリッド位置

操作:

ダブルクリック:
Warp Marker 作成

ドラッグ:
位置調整

Cmd+クリック:
削除
```

### Warp Markerの種類

```
1. Start Marker (灰色):
   クリップの再生開始位置
   固定

2. Auto Warp Marker (灰色):
   自動生成されたマーカー
   Abletonが自動配置

3. User Warp Marker (黄色):
   ユーザーが手動配置
   最も重要

4. Transient Marker (灰色・小):
   トランジェント位置
   自動検出

マーカーの優先度:
User > Auto > Transient

User Markerを配置すると:
その位置が確定
周囲のAutoMarkerが調整される
```

### Warp Marker配置

```
基本フロー:

1. Warp: On

2. 最初のビートを探す

3. その位置で ダブルクリック
   → Warp Marker 作成

4. その Marker を 1.1.1 にドラッグ

5. 次のビート (1.2.1) を探す

6. ダブルクリック → ドラッグ

7. 繰り返し

自動配置:

多くの場合:
Abletonが自動で配置

手動調整:
ズレている箇所のみ修正

複数Marker:

ドラムループ:
4-8箇所

ボーカル:
フレーズごと
```

### 効率的なWarp Marker配置テクニック

```
テクニック1: アンカーポイント戦略

手順:
1. まず曲の先頭 (1.1.1) を固定
2. 曲の中間点を固定
3. 曲の終わり近くを固定
4. 各セクション間を細分化

理由:
大きな区間から固定
→ 全体のズレを防ぐ
→ 細部を詰める

テクニック2: ダウンビート優先

手順:
1. 各小節の1拍目にMarker
2. 再生して確認
3. ズレている拍にMarker追加

理由:
1拍目が合えば全体のグルーヴが保たれる
細かいズレは後から修正

テクニック3: 右クリックメニュー活用

Set 1.1.1 Here:
選択位置を1.1.1に設定
曲の先頭を正確に合わせる

Warp From Here:
選択位置から自動Warp
BPM再検出

Warp From Here (Straight):
選択位置から一定BPMでWarp
テンポ一定の素材に

Warp From Here (Start At...):
選択位置を指定拍に設定
```

### ボーカルWarpの詳細テクニック

```
フレーズ単位のWarp:

原則:
ボーカルは「フレーズ」単位で考える

例: "昨日の夜に / 見た夢の中で / 君と踊った"

フレーズ1: "昨日の夜に"
→ 1.1.1 - 2.1.1

フレーズ2: "見た夢の中で"
→ 3.1.1 - 4.1.1

フレーズ3: "君と踊った"
→ 5.1.1 - 6.1.1

各フレーズの先頭にWarp Marker配置
→ フレーズ全体が正しい位置に

母音と子音:

Warp Markerの位置:
子音の前に配置（子音は短い）
母音の部分をストレッチ

例: "か" = k + a
Markerは "k" の直前

理由:
子音を引き伸ばすと不自然
母音は引き伸ばしても自然

ブレス（息継ぎ）処理:

ブレスの位置:
フレーズとフレーズの間

選択肢:
1. そのまま残す → 自然
2. カットする → クリーン
3. 音量を下げる → 中間

Warp時:
ブレスの位置もMarkerで管理
フレーズ間のタイミングを調整

ビブラート・メリスマ:

Warp注意:
ビブラートの部分は
Warp Markerを避ける

理由:
Marker配置で
ビブラートが崩れる可能性

対策:
ビブラートの前後にMarker
ビブラート自体には配置しない
```

### ボーカルのタイミング修正

```
走り（Rush）の修正:

症状:
歌がビートより前にある

修正:
1. 早い部分を特定
2. Warp Marker配置
3. 後ろにドラッグ（グリッドに合わせる）

もたり（Drag）の修正:

症状:
歌がビートより後ろにある

修正:
1. 遅い部分を特定
2. Warp Marker配置
3. 前にドラッグ

注意:
完璧なグリッドに合わせすぎない
→ 機械的になる
→ 多少の揺れは人間味

プロのアプローチ:

1. まず全体を大まかに合わせる
2. 各フレーズの頭を合わせる
3. 明らかなズレのみ修正
4. グルーヴ感は残す
5. 聴いて自然か確認
```

## Warpの品質を最大化する

### 高品質Warpのための設定

```
Hi-Quality Mode:

設定:
Clip View > HiQ ボタン

効果:
より高品質なアルゴリズム
CPU負荷増加

推奨:
重要なサンプル: On
大量のサンプル: Off (必要な時だけ)

RAM Mode:

設定:
Clip View > RAM ボタン

効果:
サンプルをRAMに読み込み
ディスクアクセスなし
より安定した再生

推奨:
重要なサンプル: On
ディスク速度が遅い場合: On
```

### テンポ変更幅とWarp品質の関係

```
テンポ変更の目安:

±5% (例: 128 → 134 BPM):
どのモードでも高品質
違いがほぼわからない

±10% (例: 128 → 140 BPM):
適切なモード選択で高品質
Complex/Complex Proが安全

±20% (例: 128 → 154 BPM):
Complex/Complex Pro推奨
他モードはアーティファクト

±30%以上 (例: 128 → 170 BPM):
Complex Pro必須
それでもアーティファクトあり
再録音を検討

±50%以上:
極端な変更
サウンドデザイン的
Texture Modeも面白い

品質を保つためのTips:

1. 元サンプルの品質:
   24bit/48kHz以上が理想
   MP3は避ける

2. モノ vs ステレオ:
   モノの方がWarp品質高い
   不要なステレオ情報を排除

3. Normalize:
   Warp前に適切なレベル
   小さすぎる音はノイズが目立つ

4. ドライ音:
   リバーブ等が少ない方が
   Warp品質が高い
```
