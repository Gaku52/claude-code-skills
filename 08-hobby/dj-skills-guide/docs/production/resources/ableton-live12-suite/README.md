## 🎯 典型的な使い方

### シーン1: 「Techno Bassが欲しい！」

1. **[クイックリファレンス](./quick-reference.md)を開く**
2. **「Techno」セクションを見る**
3. **推奨場所**: `Synth Essentials/Analog/Bass/`（72個から選択）
4. **Ableton Liveでそこを開く**
5. **プリセットをオーディション**

#### 具体的なプリセット選びのコツ

Techno Bassを探す場合、単に「Bass」で検索するだけではなく、サブジャンルごとに最適なアプローチが異なる。

- **ハードテクノ**: Analog Bass プリセットが最適。ディストーション感のある太いベースが特徴。`Analog/Bass/` 内の「Acid」「Drive」と名前に含むものを優先的に試す
- **ディープテクノ**: Wavetable Bass プリセットが適している。`Wavetable/Bass/` 内の「Sub」「Deep」を含むプリセットを選択
- **ミニマルテクノ**: Operator Bass が有効。シンプルなサイン波ベースから FM 合成で倍音を加える。`Operator/Bass/` から開始
- **インダストリアルテクノ**: Collision のメタリック系プリセットを Bass レンジで使うテクニックも有効

#### プリセットのカスタマイズ手順

1. プリセットをロードしたら、まず **Filter Cutoff** を確認する
2. **Amp Envelope** の Decay/Release を曲のテンポに合わせて調整
3. **Drive/Saturation** を適度に加えてミックス内での存在感を出す
4. 必要に応じて **EQ Eight** で低域の不要な倍音をカット（30Hz以下）
5. **Glue Compressor** で軽くコンプレッションして安定させる

#### リバーブ選びの実践的なガイドライン

**ジャンル別推奨リバーブ設定**:

- **テクノ/ハウス**: Short Plate または Small Room。Decay は 0.5-1.5秒。Pre-Delay を 20-40ms に設定してビートの明瞭さを保つ
- **アンビエント**: Large Hall または Cathedral。Decay は 3-8秒。高域のダンピングを適度にかけてダークな質感に
- **ヒップホップ**: Medium Room または Plate。Decay は 1-2秒。EQ で低域をカットし、ボーカルの明瞭さを維持
- **ドラムンベース**: Non-Linear または Gated。短いテールでパーカッションのインパクトを強調
- **Lo-Fi**: Vintage カテゴリのプリセットが最適。テープエコー風のリバーブが雰囲気を作る

**リバーブのセンドエフェクト設定のコツ**:

1. **Return Track A** にメインリバーブ（Room系）を配置
2. **Return Track B** にセカンドリバーブ（Hall系）を配置
3. 各トラックからのセンド量で空間の奥行きをコントロール
4. Return Track にEQを挿して、低域（~200Hz以下）と高域（~8kHz以上）をカット
5. これにより、リバーブがミックスを濁らせずに空間感だけを付加できる

### 各シンセサイザーの詳細解説

#### Wavetable（100個）- Live 12 Suite の主力シンセ

Wavetable は Ableton Live 10 で導入されたウェーブテーブル方式のシンセサイザー。Live 12 Suite では最多の100個のプリセットが収録されている。

**アーキテクチャの特徴**:
- 2つのオシレーター、各々が独立したウェーブテーブルを持つ
- Sub Oscillator（サブオシレーター）でローエンドを補強
- 2つのフィルター（Serial/Parallel切替可能）
- 3つのLFO + 3つのエンベロープ
- Matrix モジュレーションシステム

**プリセットのカテゴリ別内訳（100個）**:

| カテゴリ | 数量 | 代表的なプリセット名の傾向 |
|---------|------|------------------------|
| Bass | 20個 | Sub, Growl, Reese, Wobble |
| Lead | 18個 | Saw, Square, Bright, Sharp |
| Pad | 15個 | Evolving, Warm, Lush, Ethereal |
| Keys | 12個 | Electric, Pluck, Bell, Glass |
| Rhythmic | 10個 | Sequence, Pulse, Gate, Stutter |
| Texture | 10個 | Noise, Grain, Atmosphere, Drift |
| FX | 8個 | Riser, Impact, Sweep, Transition |
| Misc | 7個 | Experimental, Hybrid |

**Wavetable の音作りワークフロー**:

1. **ウェーブテーブルの選択**: まず OSC 1 のウェーブテーブルを選ぶ。Basic Shapes から始めて、Complex やユーザーテーブルに進む
2. **ポジションの設定**: Wavetable Position ノブで音色の基本キャラクターを決定。LFO でモジュレーションすると動きが出る
3. **フィルターの設定**: LP12 (ローパス12dB) が最も汎用的。Cutoff と Resonance でキャラクターを微調整
4. **エンベロープの設定**: Amp Envelope で音量の時間変化を設定。Attack、Decay、Sustain、Release の4パラメーター
5. **モジュレーション**: Matrix で LFO やエンベロープを任意のパラメーターに接続

#### Operator（71個）- FM合成の強力なツール

Operator は FM（周波数変調）合成を基盤としたシンセサイザー。4つのオシレーター間の変調関係により、倍音豊かなサウンドを生成する。

**FM合成の基礎概念**:
- **キャリア**: 最終的に聴こえる音を生成するオシレーター
- **モジュレーター**: キャリアの周波数を変調するオシレーター
- **アルゴリズム**: オシレーター間の接続パターン（11種類）
- **Ratio**: オシレーター間の周波数比。整数比で調和的な音色、非整数比で金属的な音色

**プリセットのカテゴリ別内訳（71個）**:

| カテゴリ | 数量 | 特徴 |
|---------|------|------|
| Bass | 15個 | FM特有の倍音豊かなベース |
| Lead | 12個 | ブリリアントなリード音色 |
| Keys | 11個 | エレピ、ベル系 |
| Pad | 8個 | 複雑な倍音のパッド |
| Drums | 8個 | シンセドラム（キック、スネア） |
| FX | 7個 | エフェクト音、効果音 |
| Rhythmic | 5個 | アルペジオ系 |
| Misc | 5個 | 実験的サウンド |

**Operator でのドラム音色合成**:

Operator はシンセドラムの合成に非常に適している。

- **キック**: オシレーター A をサイン波に設定し、Pitch Envelope で急速なピッチ下降を設定。Decay は 200-500ms
- **スネア**: オシレーター A（ボディ用）+ オシレーター B（ノイズ用）。アルゴリズム 4 が最適
- **ハイハット**: 非整数比のオシレーター設定。Ratio を 1.00 / 2.31 / 4.57 / 7.13 のように設定
- **クラップ**: 短い Decay のノイズに少量の Reverb を追加

#### Analog（41個）- アナログサウンドのエミュレーション

Analog は減算合成方式のシンセサイザーで、クラシックなアナログシンセサイザーをモデリングしている。

**特徴**:
- 2つのオシレーター（Saw, Square/Pulse, Sine, Noise）
- 2つのマルチモードフィルター
- 2つのLFO
- AmpとFilterのエンベロープ
- Vibrato と Unison（最大8ボイス）

**プリセットのカテゴリ別内訳（41個）**:

| カテゴリ | 数量 | 特徴 |
|---------|------|------|
| Bass | 13個 | アナログ特有の温かみのあるベース |
| Lead | 9個 | クラシックなリード |
| Pad | 7個 | 温かいパッド |
| Keys | 5個 | キーボード系 |
| FX | 4個 | エフェクト系 |
| Misc | 3個 | その他 |

#### Collision（30個）- 物理モデリングシンセ

Collision は物理モデリング方式のシンセサイザーで、打楽器やマレット楽器のリアルなシミュレーションが得意。

**モデリング要素**:
- **Mallet**: マレット（バチ）の特性（硬さ、ノイズ量）
- **Noise**: ノイズコンポーネント（打撃音のアタック部分）
- **Resonator 1 & 2**: 共鳴体のシミュレーション（Beam, Marimba, String, Membrane, Plate, Pipe, Tube）
- **LFO**: モジュレーション用

**向いているサウンド**:
- マリンバ、ビブラフォンなどのマレット楽器
- ベル、チャイム系の金属音
- ゲームラン、カリンバなどの民族楽器
- 実験的なパーカッション

#### Tension（18個）- 弦楽器の物理モデリング

Tension は弦楽器に特化した物理モデリングシンセサイザー。

**モデリング要素**:
- **Excitator**: 弦の励起方法（Bow, Hammer, Pluck）
- **String**: 弦の物理特性（Decay, Damping, Inharmonicity）
- **Body**: ボディの共鳴特性
- **Filter**: サウンドシェイピング用フィルター
- **Vibrato**: ビブラートエフェクト

**プリセット用途**:
- リアルなギターシミュレーション
- バイオリン、チェロの表現
- ハープ、シタールなどの弦楽器
- 実験的な弦楽器サウンド

## 🎛️ Max for Live デバイス完全ガイド

### Max for Live とは

Max for Live（M4L）は、Cycling '74 社の Max/MSP プログラミング環境を Ableton Live に統合したもの。Live 12 Suite にはフルバージョンが含まれており、カスタムインストゥルメント、エフェクト、MIDIデバイスの作成・使用が可能。

### Suite 付属の Max for Live デバイス（137個）

|-----------|---------|------|
| **Granulator II** | グラニュラーシンセ | オーディオファイルを粒子（グレイン）に分解して再合成。テクスチャー生成の強力ツール |
| **Drift** | アナログモデリング | 温かみのあるアナログシンセサウンド。ドリフト（音程の微妙なゆらぎ）が特徴 |
| **PitchLoop89** | ピッチシフター | リアルタイムピッチシフト+ディレイ。実験的なサウンドデザインに最適 |
| **Bass** | ベースシンセ | ベースに特化したシンプルなシンセ。即座に太いベースを出力 |
| **Meld** | ウェーブフォールディング | 2つのオシレーターとウェーブフォールディング合成 |

#### Granulator II の詳細活用法

Granulator II はアンビエント、エクスペリメンタル、映像音楽の制作において非常に強力なツール。

**基本パラメーター**:

```
Grain Size: 10ms - 1000ms（グレインの長さ）
  - 小さい値: テクスチャー的、粒子的なサウンド
  - 大きい値: よりスムーズ、原音に近いサウンド

Position: 0% - 100%（ソースファイル内の再生位置）
  - LFOで変調すると、ファイル内をスキャンする効果

Spray: 0ms - 500ms（位置のランダム変動量）
  - 高い値でよりカオティックなテクスチャー

Pitch: -48st - +48st（ピッチシフト量）
  - 半音単位で±4オクターブ

Density: 1 - 100（同時発音グレイン数）
  - 高い値でより厚みのあるサウンド
```

**Granulator II を使ったアンビエント制作テクニック**:

1. フィールドレコーディング（鳥の声、水の音、街の環境音）をソースとして読み込む
2. Grain Size を 200-500ms に設定
3. Position を LFO でゆっくりスキャン（Rate 0.1-0.5Hz）
4. Spray を 100-200ms に設定してランダム性を追加
5. Pitch を -12st または +12st にシフトして非現実的なサウンドを生成
6. 出力に Convolution Reverb (Large Hall) を適用して壮大な空間感を付加
7. Echo デバイスで長いフィードバックのディレイを追加

6. 同一ネットワーク上の他の Link 対応アプリと自動接続
```

## 🎚️ ミキシング & マスタリング ワークフロー

### Ableton Live 12 Suite でのミキシング

Live 12 Suite は制作だけでなく、プロフェッショナルなミキシング環境としても十分に機能する。Session View と Arrangement View の両方でミキシングが可能であり、内蔵エフェクトの品質は年々向上している。

#### ミキシングの基本ワークフロー

**ステップ1: ゲインステージング**

```
1. 全トラックのフェーダーを 0dB に設定
2. 各トラックの先頭に Utility デバイスを挿入
3. Utility の Gain でトラックレベルを調整（ピーク -12dB 前後を目安）
4. マスターバスのピークが -6dB 程度になるよう全体バランスを調整
5. これにより、後段のエフェクトに適切なレベルで信号が入る
```

**ステップ2: 周波数バランスの調整**

```
低域整理の手順:
1. キックとベース以外の全トラックに HP Filter (30-100Hz)
2. キックの基音（50-60Hz）とベースの基音（80-120Hz）が被らないよう EQ で棲み分け
3. サイドチェーンコンプレッションでキックとベースの共存を確保

中域の処理:
1. 各楽器の基本周波数帯域を把握する
2. 不要なマスキングを EQ のカットで解消
3. 200-400Hz 帯域は「泥っぽさ」の原因になりやすいので注意

高域の処理:
1. ハイハット、シンバル系のピーク周波数を確認
2. 耳に痛い周波数（2-4kHz）を狭い Q でカット
3. Air Band（12kHz以上）のシェルビングで開放感を出す
```

**ステップ3: ダイナミクス処理**

```
個別トラックのコンプレッション:
  キック: Fast Attack (1-5ms), Medium Release (50-100ms), Ratio 4:1
  スネア: Medium Attack (5-15ms), Fast Release (30-50ms), Ratio 3:1
  ベース: Medium Attack (10-30ms), Auto Release, Ratio 3:1-4:1
  ボーカル: Slow Attack (20-40ms), Auto Release, Ratio 2:1-3:1
  パッド/シンセ: Gentle (Ratio 2:1), Slow settings

バスコンプレッション:
  ドラムバス: Glue Compressor, Ratio 2:1, Attack 0.3ms, Range -3dB
  楽器バス: Glue Compressor, Ratio 2:1, Attack 10ms, Range -2dB
  マスターバス: Glue Compressor, Ratio 2:1, Attack 30ms, Range -2dB
```

**ステップ4: 空間配置**

```
パンニングガイドライン:
  センター: キック、ベース、ボーカル、スネア（中心部）
  やや左右: ハイハット、パーカッション（L/R 20-40%）
  広め: パッド、アンビエンス（L/R 40-70%）
  極端: エフェクト、テクスチャー（L/R 70-100%）

奥行きの表現:
  前面: ドライ、コンプレッション強め、高域明瞭
  中面: 適度なリバーブ、自然なバランス
  背面: リバーブ多め、高域ロールオフ、ボリューム控えめ
```

#### Utility デバイスの活用テクニック

Utility は最もシンプルだが最も有用なデバイスの一つ。

```
主な使い方:
  1. Gain: ゲインステージング（エフェクト前後のレベル調整）
  2. Width: ステレオ幅の制御（0%=モノ、100%=ステレオ、200%=超ワイド）
  3. Bass Mono: 低域をモノラル化（クラブサウンドシステム対応）
     - Bass Mono Frequency: 120Hz（推奨）
  4. Phase Invert: 位相反転（マルチマイク録音時の位相問題解決）
  5. DC Offset: DCオフセットの除去
  6. Panorama: パン設定の確認用
```

### マスタリングチェーン

Ableton Live 12 Suite 内蔵エフェクトだけで構築するマスタリングチェーン。

```
推奨マスタリングチェーン（マスタートラック）:

1. EQ Eight (Corrective)
   - HP 20Hz, 48dB/oct
   - 問題周波数の小さなカット
   - Oversampling x4

2. Multiband Dynamics
   - Band 1 (Low): Below 120Hz, Ratio 2:1, Threshold -15dB
   - Band 2 (Mid): 120Hz-5kHz, Ratio 1.5:1, Threshold -12dB
   - Band 3 (High): Above 5kHz, Ratio 2:1, Threshold -18dB
   - 各バンドのゲインで音色バランスを微調整

3. EQ Eight (Tonal)
   - 音楽的なブースト/カット
   - Low Shelf 80Hz +1dB
   - High Shelf 12kHz +1.5dB
   - Oversampling x4

4. Glue Compressor
   - Ratio 2:1
   - Attack 30ms
   - Release Auto
   - Makeup Gain: 必要に応じて
   - Soft Clip: ON（アナログ的なサチュレーション）

5. Saturator (Optional)
   - Analog Clip or Soft Sine
   - Drive +1-3dB
   - Output -1-3dB（Drive分だけ下げる）
   - Dry/Wet 30-50%

6. Limiter
   - Ceiling: -0.3dB (Streaming用) or -1.0dB (CD用)
   - Lookahead: 1.5ms
   - Release: Auto

最終出力: -14 LUFS (Streaming) or -9 LUFS (Club)
```

推奨ラウドネス目標値:
  Spotify: -14 LUFS (integrated)
  Apple Music: -16 LUFS (integrated)
  YouTube: -14 LUFS (integrated)
  クラブプレイ: -8 to -6 LUFS (integrated)
  Beatport/Bandcamp: 制限なし（-8 to -10 LUFS が一般的）
```

### ウェーブテーブル合成によるサウンドデザイン

Wavetable シンセサイザーを使った実践的なサウンドデザインレシピ。

#### レシピ1: モーフィングパッド

```
OSC 1: Basic Shapes テーブル, Position を LFO1 でモジュレーション
  LFO1: Sine, Rate 0.2Hz, Amount 50%
OSC 2: Overtone テーブル, Position を LFO2 でモジュレーション
  LFO2: Triangle, Rate 0.15Hz, Amount 40%
Filter 1: LP 12, Cutoff 2kHz, Resonance 20%
  Cutoff → Envelope 2 (Attack 2s, Decay 3s, Sustain 60%, Release 4s)
Unison: 4 voices, Amount 30%
エフェクト: Chorus-Ensemble → Hybrid Reverb (Large Hall)
```

#### レシピ2: グロウルベース（ダブステップ/ニューロファンク）

```
OSC 1: Spectral テーブル, Position を LFO1 でモジュレーション
  LFO1: Saw Down, Rate 1/8 (Sync), Amount 100%
OSC 2: FM テーブル, Position を LFO2 でモジュレーション
  LFO2: Square, Rate 1/16 (Sync), Amount 80%
Filter 1: BP 12, Cutoff → Envelope 1 (short Decay)
Filter 2: LP 24, Cutoff 1.5kHz, Resonance 30%
  Cutoff → LFO3 (Sine, Rate 1/4, Amount 40%)
Matrix: Velocity → Filter 1 Cutoff, Aftertouch → OSC 1 Position
エフェクト: Saturator (Heavy) → EQ Eight (低域ブースト、高域カット)
```

#### レシピ3: アシッドベースライン

```
Operator を使用:
  Algorithm: 1 (全直列)
  OSC A: Saw, Level 100%, Coarse 1
  OSC B: Square, Level 60%, Coarse 1
  Filter: LP 24, Cutoff 500Hz, Resonance 65%
  Filter Envelope: Attack 0ms, Decay 200ms, Sustain 0%, Release 50ms
  Env Amount: 70%
  Glide: ON, Time 50ms
エフェクト:
  Saturator → Overdrive → Auto Filter (LFO Sine, Rate 1/8)
  ※ TB-303 の再現に近いサウンドが得られる
```

#### レシピ4: シネマティック・テクスチャー

```
Granulator II を使用:
  ソース: 環境音、楽器音、ノイズ等のオーディオファイル
  Grain Size: 50-200ms
  Position: LFO でスロースキャン (0.05Hz)
  Spray: 100ms
  Pitch: -12st to +12st（複数インスタンスでレイヤー）
  Density: 20-50

エフェクトチェーン:
  Spectral Blur (Residual モード、Halo 50%) →
  Hybrid Reverb (Convolution: Cathedral, Algorithm: Shimmer) →
  Echo (Feedback 60%, Character: Dark) →
  EQ Eight (HP 40Hz, LP 12kHz)
```

### FM合成によるサウンドデザイン

Operator を使った FM 合成の実践テクニック。

#### ベルサウンドの合成

```
Algorithm: 1 (A→B→C→D)
OSC D (Carrier): Sine, Level 100%
OSC C (Modulator 1): Sine, Level 40%, Coarse 3.0
OSC B (Modulator 2): Sine, Level 20%, Coarse 7.0
OSC A (Modulator 3): Sine, Level 10%, Coarse 11.0

各オシレーターのエンベロープ:
  OSC D: Attack 0ms, Decay 2000ms, Sustain 0%, Release 3000ms
  OSC C: Attack 0ms, Decay 800ms, Sustain 0%, Release 1000ms
  OSC B: Attack 0ms, Decay 400ms, Sustain 0%, Release 500ms
  OSC A: Attack 0ms, Decay 200ms, Sustain 0%, Release 200ms

ポイント: モジュレーターほど短い Decay にすることで、
  アタック時に倍音が豊かで徐々に純音に近づく
  → ベル/マリンバ/ビブラフォンのような自然な減衰
```

#### メタリックパーカッションの合成

```
Algorithm: 4 (A+B+C+D, 全パラレル)
全オシレーターのRatio を非整数に設定:
  OSC A: Coarse 1.00, Fine 0
  OSC B: Coarse 2.76, Fine 0
  OSC C: Coarse 5.43, Fine 0
  OSC D: Coarse 8.21, Fine 0

エンベロープ: 非常に短い Decay (50-200ms)
フィルター: HP 2kHz で低域をカット

非整数比の組み合わせにより金属的な音色が得られる。
Ratio の微調整で、ゴング、シンバル、ベル等の異なるキャラクターを作れる。
```

## ⚡ Live 12 の新機能と改善点

### Live 12 で追加された主要機能

#### 1. MIDI ジェネレーターとトランスフォーマー

Live 12 最大の新機能。MIDI クリップ内でジェネレーティブな音楽制作が可能。

```
MIDI Generators:
  - Rhythm: リズムパターンの自動生成
  - Melody: スケールに基づくメロディ生成
  - Euclidean: ユークリッドリズムの生成
  - Stacks: コードスタッキング
  - Seed: シード値によるパターン生成

MIDI Transformations:
  - Quantize: クオンタイズ（グリッド吸着）
  - Humanize: ヒューマナイズ（タイミング/ベロシティの微妙なゆらぎ）
  - Recombine: ノートの再配置・並び替え
  - Time Warp: タイミングの伸縮
  - Ornament: 装飾音の追加
  - Connect: ノート間のレガート接続
  - Span: ノートの長さ変更
  - Arpeggiate: アルペジオ化

スタック可能: 複数のジェネレーター/トランスフォーマーを重ねて適用
```

**活用例: ジェネラティブテクノ**

1. Drum Rack にドラムキットをロード
2. MIDI クリップを作成し、Rhythm Generator を適用
3. Euclidean Generator でハイハットパターンを生成
4. Humanize Transformer でグルーヴを追加
5. シーンを複数作成し、各シーンで異なるシード値を設定
6. Session View でシーンを切り替えながらパフォーマンス

#### 2. 改善されたブラウザ

```
新機能:
  - サウンド類似検索: プリセットの音色的な類似性で検索
  - タグフィルタリング: 複数タグの AND/OR 検索
  - コレクション改善: より柔軟なラベリングと整理
  - プレビュー強化: ブラウザ内でのオーディションが高速化
  - ドラッグ&ドロップ改善: より直感的な操作
```

#### 3. 新しいドリフトシンセの改善

Live 12 では Drift シンセが大幅にアップデートされ、新しいオシレーターモードやモジュレーションオプションが追加された。

#### 4. Tuning System（チューニングシステム）

```
対応チューニング:
  - 12-TET（12平均律）: 標準
  - Just Intonation（純正律）: より調和的な響き
  - Pythagorean（ピタゴラス音律）: 完全5度を重視
  - Meantone: 中全音律
  - Custom Scala files (.scl): 任意のマイクロチューニング

設定方法:
  1. トラックの「Tuning」ボタンをクリック
  2. チューニングシステムを選択
  3. 基準周波数を設定（デフォルト A=440Hz）
  4. 全てのインストゥルメントに自動適用

#### 5. MPE（MIDI Polyphonic Expression）の強化

```

  - 全ての内蔵シンセが MPE に完全対応
  - Wavetable: Per-note ピッチベンド、スライド、プレッシャー

  - Analog: Per-note フィルター制御

対応コントローラー:

  - Linnstrument

  - Expressive E Osmose

## 🛠️ トラブルシューティング & パフォーマンス最適化

### CPU負荷の軽減テクニック

```
1. バッファサイズの調整:
   制作時: 256-512 samples
   ミキシング時: 1024-2048 samples
   ライブパフォーマンス: 128-256 samples

2. フリーズ機能の活用:
   CPU負荷の高いトラックを右クリック → 「Freeze Track」
   トラックがオーディオにレンダリングされ、CPU負荷が激減
   編集が必要になったら「Unfreeze Track」

3. オーバーサンプリングの管理:
   EQ Eight, Saturator 等の Hi-Quality モードは CPU 負荷が高い
   制作中は OFF、最終ミックス/マスタリング時のみ ON

4. Max for Live デバイスの最適化:
   不要な M4L デバイスは削除またはフリーズ
   特に Granulator II はCPU負荷が高い

5. 未使用のリターントラック削除:
   信号が送られていないリターントラックでもCPUを消費する

6. 複数のリバーブインスタンスを統合:
   個別トラックのリバーブを削除し、センドエフェクトに集約
```

### オーディオ設定の最適化

```
推奨設定 (macOS):
  Audio Device: 外部オーディオインターフェース（内蔵は非推奨）
  Sample Rate: 44100Hz（最終配信用）or 96000Hz（高品質制作用）
  Buffer Size: 用途に応じて調整（上記参照）
  Driver Type: Core Audio

推奨設定 (Windows):
  Audio Device: ASIO ドライバー対応インターフェース
  Driver Type: ASIO
  Buffer Size: 256-512 samples
  ※ ASIO4ALL はレイテンシーが大きいため、専用ドライバー推奨
```

### よくある問題と解決策

| 問題 | 原因 | 解決策 |
|------|------|--------|
| 音が出ない | オーディオ設定 | 環境設定でOutput Device確認 |
| レイテンシーが大きい | バッファサイズ | バッファサイズを小さくする |
| CPU過負荷 | 重いプラグイン | Freeze Track、バッファサイズ増加 |
| クラッシュ | プラグイン互換性 | 問題プラグインの特定と削除 |
| ワープが合わない | ワープマーカー | 手動でワープマーカーを修正 |
| MIDIが反応しない | MIDI設定 | 環境設定のMIDI Ports確認 |
| プリセットが見つからない | Pack未インストール | ブラウザのPacks→利用可能を確認 |
