## Rekordbox 録音設定

### 初期設定

**一度設定すれば OK:**

```
Step 1: 環境設定を開く

Rekordbox メニュー:
[Rekordbox] > [Preferences]（Mac）
[File] > [Preferences]（Windows）

または:
[Cmd + ,]（Mac）
[Ctrl + ,]（Windows）

Step 2: 録音タブを選択

左メニュー:
[Controller] > [Rec]

設定画面が開く

Step 3: 保存先設定

Rec Folder:
[Browse] をクリック

推奨フォルダ構造:
/Users/あなたの名前/DJ/
  ├── Recordings/
  │   ├── Practice/（練習用）
  │   ├── Demos/（デモ用）
  │   └── Live/（ライブ録音）
  └── Music/

作成して選択:
[DJ] > [Recordings] > [Practice]

Step 4: ファイル形式設定

Rec Format:
WAV or MP3 を選択

WAV 設定:
- Sample Rate: 44100 Hz
- Bit Depth: 16 bit
- 非圧縮、最高音質
- ファイルサイズ大（60分 約600MB）

MP3 設定:
- Bit Rate: 320 kbps（最高品質）
- ファイルサイズ小（60分 約140MB）
- 圧縮、音質劣化わずか

推奨:
最初は WAV で録音
→ 後で MP3 に変換
（音質の選択肢を残す）

Step 5: ファイル名設定

Auto Naming:
チェックを入れる

Format:
%Y-%m-%d_%H%M_Recording

例:
2025-12-07_2200_Recording.wav

カスタマイズも可能:
%Y: 年（4桁）
%m: 月（2桁）
%d: 日（2桁）
%H: 時（24時間）
%M: 分（2桁）

Step 6: その他設定

Split Rec:
OFF（通常は不要）

Rec Level:
0 dB（デフォルト）

完了:
[OK] をクリック

設定完了！
```

## DDJ-FLX4 での録音手順

### 準備

**録音前のチェック:**

```
30分前:

□ Rekordbox 起動
□ DDJ-FLX4 接続確認
□ プレイリスト準備
□ Hot Cue 設定完了
□ ヘッドフォン動作確認

15分前:

□ リハーサル（軽く通す）
□ ミスしそうな箇所確認
□ トイレ、水分補給
□ 部屋を静かに（通知OFF）

5分前:

□ 深呼吸
□ 最初の2曲を確認
□ リラックス
```

### 録音開始

**ステップバイステップ:**

```
Step 1: 録音ボタン準備

DDJ-FLX4:
右上の [REC] ボタン確認

または

Rekordbox PC画面:
上部メニューの [●] ボタン

Step 2: レベル確認

Master Output:
メーター確認

理想的なレベル:
-6 dB〜 0 dB（黄色ゾーン）

赤ゾーン（クリップ）:
絶対に避ける
→ 歪む、修復不可能

TRIM/GAIN:
適切に調整済みか確認

Step 3: 録音開始

[REC] ボタンを押す:
LED が赤く点灯
→ 録音中

Rekordbox画面:
「Recording...」表示

または

ショートカットキー:
[R] キー（PC）

タイミング:
最初の曲のビート1から
または
数秒の無音 → 曲開始

Step 4: プレイ開始

通常通りDJプレイ:
□ リラックス
□ ミスを恐れない
□ ミスしても続ける
□ 楽しむ

録音中の注意:
□ マイク音声も録音される
□ PC通知音も録音される
□ 部屋の雑音も録音される

→ 静かな環境推奨

レベル監視:
定期的にメーターチェック
常に黄色ゾーン維持

Step 5: 録音終了

最後の曲が終わったら:
[REC] ボタンをもう一度押す

LED 消灯:
録音停止

または

[R] キー（PC）

自動保存:
設定フォルダに保存される

確認:
Rekordbox ブラウザで
Recordings フォルダ確認
```

## 録音中のベストプラクティス

### レベル管理

**歪みを防ぐ:**

```
ゲインステージング:

各チャンネル:
TRIM を適切に
→ チャンネルメーター -6〜0 dB

Master:
MASTER LEVEL で微調整
→ マスターメーター -6〜0 dB

常にモニター:
録音中は目を離さない

赤点灯したら:
すぐに MASTER LEVEL を下げる

理想:
終始 -6〜-3 dB
→ 余裕を持つ
→ 後処理で調整可能
```

### タイムスタンプメモ

**後で振り返る:**

```
録音中:

良い瞬間:
「25:30 - 完璧なトランジション」

ミス:
「42:15 - ビートズレ」

メモ方法:

1. 紙に書く（アナログ）
   時間: 内容

2. スマホメモ（デジタル）
   タイマー見ながら記録

3. 音声メモ（録音後確認時）

後で:
改善点の分析に役立つ
次回に活かす
```

## ファイル管理

### 整理術

**見つけやすく:**

```
フォルダ構造:

/DJ/Recordings/
  ├── 2025/
  │   ├── 01-January/
  │   │   ├── 2025-01-05_Practice_Techno.wav
  │   │   └── 2025-01-12_Demo_House.wav
  │   ├── 02-February/
  │   └── 03-March/
  └── Archive/
      └── 2024/

命名規則:

YYYY-MM-DD_Type_Genre.wav

例:
2025-12-07_Practice_Techno.wav
2025-12-10_Demo_House.wav
2025-12-15_Live_Festival.wav

Type:
- Practice: 練習
- Demo: デモ
- Live: ライブ録音
- Podcast: ポッドキャスト用

Genre:
- Techno
- House
- HipHop
- Mix（複数ジャンル）

一目で分かる:
効率的
```

### バックアップ戦略

**大切な録音を守る:**

```
3-2-1 ルール:

3: 3箇所に保存
2: 2種類のメディア
1: 1箇所は外部（クラウド等）

実践例:

コピー1: PC内蔵ストレージ
/Users/あなた/DJ/Recordings/

コピー2: 外付けHDD/SSD
/Volumes/Backup/DJ/Recordings/

コピー3: クラウド
Google Drive / Dropbox / iCloud

頻度:
録音後すぐにバックアップ

自動化:
Time Machine（Mac）
File History（Windows）
→ 設定しておく

失うと:
取り返しがつかない
必ずバックアップ
```

## WAV から MP3 への変換

### いつ変換する？

**用途別に:**

```
WAV のまま:
□ マスタリング前
□ 編集予定
□ 最高音質保存

MP3 に変換:
□ SoundCloud アップロード
□ デモとして送付
□ スマホで聴く
□ ファイルサイズ削減

両方保存:
推奨（ストレージに余裕あれば）
```

### Rekordbox で変換

**簡単3ステップ:**

```
Step 1: ファイル選択

Rekordbox ブラウザ:
Recordings フォルダ

WAV ファイル:
右クリック

Step 2: 変換実行

メニュー:
[Convert to MP3]

設定:
Bit Rate: 320 kbps
Quality: High

保存先:
元のフォルダ

[Convert] クリック

Step 3: 確認

同じフォルダに:
.mp3 ファイルが生成

例:
2025-12-07_2200_Recording.wav
2025-12-07_2200_Recording.mp3

両方保存されている:
用途別に使い分け
```

## トラブルシューティング

### 録音されない

**チェックリスト:**

```
原因1: REC ボタンが押されていない
解決: [REC] ボタン確認、LED点灯確認

原因2: 保存先フォルダが存在しない
解決: Preferences で保存先確認、フォルダ作成

原因3: ディスク容量不足
解決: ストレージ確認、不要ファイル削除

原因4: Rekordbox が録音モードでない
解決: Performance モードか確認
```

### 音が歪む

**対処法:**

```
原因: ゲインが高すぎる

解決:

録音中:
1. MASTER LEVEL を下げる
2. 各チャンネルの TRIM を下げる
3. EQ を控えめに

次回から:
録音前にレベル確認
リハーサルで調整
常に -6〜-3 dB を目標
```

### ファイルが見つからない

**探し方:**

```
Step 1: 保存先確認

Preferences:
[Controller] > [Rec]
Rec Folder を確認

Step 2: Finder/エクスプローラーで検索

検索キーワード:
「Recording」「.wav」

日付で絞り込み:
今日、昨日

Step 3: Rekordbox ブラウザ

左メニュー:
[Recordings]

表示されない場合:
フォルダを右クリック
[Reload]
```

## よくある質問

### Q1: WAV と MP3 どちらで録音？

**A:** WAV 推奨、後で変換

```
理由:
- 最高音質で保存
- 後で選択肢が広がる
- 編集・マスタリングに有利

ストレージ節約:
古い録音は MP3 に変換
WAV は削除

最初は:
WAV で録音する習慣を
```

### Q2: 録音中にミスしたら？

**A:** 止めずに続ける

```
ミスの種類:

小さいミス:
そのまま続ける
→ 誰も気づかない

大きいミス:
続けて最後まで録音
→ 後で編集 or 再録音

録音を止めるのは:
- 機材トラブル
- 大きな外部ノイズ
のみ

完璧主義:
禁物、続けることが重要
```

### Q3: どれくらいの長さを録音？

**A:** 用途による

```
練習:
30-60分

デモミックス:
30-60分（60分推奨）

ポッドキャスト:
60-90分

ライブセット:
実際の長さ（2-4時間）

初心者:
30分から始める
→ 徐々に伸ばす
```

## 録音機材の選び方と詳細ガイド

DJ録音のクオリティは使用する機材に大きく依存します。ここでは、さまざまな録音シナリオに対応するための機材選択と設定について詳しく解説します。

### 録音機材の種類と特徴

**DJミックス録音に使用できる機材カテゴリ:**

```
カテゴリ1: DJコントローラー内蔵録音

対象機材:
- DDJ-FLX4（USB経由でPCに録音）
- DDJ-1000（USB経由でPCに録音）
- DDJ-REV7（USB経由でPCに録音）
- XDJ-RX3（USB メモリに直接録音可能）

メリット:
- 追加機材不要
- 設定が簡単
- Rekordboxから直接操作
- 遅延なし

デメリット:
- 録音品質はコントローラー依存
- マルチトラック録音不可
- 外部エフェクトが録音されない場合がある

推奨度: ★★★★★（初心者〜中級者）

カテゴリ2: オーディオインターフェース

対象機材:
- Focusrite Scarlett 2i2（エントリー）
- Universal Audio Apollo Twin（ミドル）
- RME Babyface Pro（ハイエンド）
- Audient iD4 mkII（コスパ良好）

メリット:
- 高品質なAD/DA変換
- 低レイテンシー
- マイク入力も使用可能
- 複数入力でマルチトラック録音

デメリット:
- 追加投資が必要
- 接続設定が必要
- ドライバーインストールが必要

推奨度: ★★★★☆（中級者〜上級者）

カテゴリ3: ポータブルレコーダー

対象機材:
- ZOOM H6 Black（6トラック対応）
- ZOOM H5（コンパクト高品質）
- TASCAM DR-40X（ステレオ+2入力）
- SONY PCM-D10（高音質ステレオ）

メリット:
- PC不要で録音可能
- ライブ現場で威力を発揮
- バッテリー駆動
- 内蔵マイクでアンビエント録音

デメリット:
- 録音レベルの監視が小画面
- ストレージ容量制限
- 後でPCに転送が必要

推奨度: ★★★★☆（ライブ録音向け）

カテゴリ4: ミキサー内蔵レコーダー

対象機材:
- Allen & Heath Xone:96（USB録音対応）
- Pioneer DJM-A9（USB録音対応）
- Pioneer DJM-900NXS2（USB録音対応）

メリット:
- ミキサー出力を直接録音
- 最も正確な出力音声を取得
- プロ現場の標準

デメリット:
- 高価
- クラブ以外では大きすぎる

推奨度: ★★★★★（プロ向け）
```

### DDJ-FLX4 録音の詳細スペック

**DDJ-FLX4のオーディオ仕様を理解する:**

```
DDJ-FLX4 オーディオスペック:

AD変換:
- ビット深度: 24bit
- サンプリングレート: 44.1kHz
- ダイナミックレンジ: 104dB以上

DA変換:
- ビット深度: 24bit
- サンプリングレート: 44.1kHz
- ダイナミックレンジ: 104dB以上

USB接続:
- USB-C（デバイス側）
- USB-A（PC側）
- バスパワー対応（電源不要）

録音時の信号経路:

DJ楽曲（PC内）
  ↓
Rekordbox オーディオエンジン
  ↓
DDJ-FLX4 内部ミキサー
  ↓
マスター出力（スピーカー/ヘッドフォン）
  ↓（同時に）
Rekordbox 録音エンジン
  ↓
WAV/MP3 ファイル

ポイント:
DDJ-FLX4の場合、録音はPC内部で完結
→ アナログ→デジタル変換が不要
→ 理論上最高品質で録音可能
→ 外部オーディオインターフェース不要
```

### 外部機器を接続した場合の録音

**ターンテーブルやCDJを追加する場合:**

```
DDJ-FLX4 + 外部入力の録音:

注意: DDJ-FLX4には外部入力がない
→ 外部機器の音を録音する場合は
  別途オーディオインターフェースが必要

構成例1: ターンテーブル追加
ターンテーブル → フォノプリアンプ → オーディオI/F → PC
DDJ-FLX4 → USB → PC（同一PC）
→ DAW（Ableton等）で両方をミックス・録音

構成例2: CDJ追加
CDJ → RCAケーブル → オーディオI/F → PC
DDJ-FLX4 → USB → PC
→ DAWで合流・録音

構成例3: マイク追加（MC用）
マイク → オーディオI/F → PC
DDJ-FLX4 → USB → PC
→ DAWでミックス・録音

各構成のメリット・デメリット:

構成例1:
メリット: アナログレコードの温かい音を録音可能
デメリット: フォノプリアンプの品質が音に直結

構成例2:
メリット: CDJのジョグホイール操作が活きる
デメリット: BPM同期は手動対応が必要

構成例3:
メリット: MCプレイの臨場感を録音
デメリット: マイクブリードに注意が必要
```

## オーディオインターフェースの詳細設定

### オーディオインターフェースとは

**デジタル録音の心臓部:**

```
役割:
アナログ音声信号 ←→ デジタル音声信号の変換

入力（アナログ→デジタル: AD変換）:
マイク/楽器 → インターフェース → PC

出力（デジタル→アナログ: DA変換）:
PC → インターフェース → スピーカー/ヘッドフォン

DJでの主な用途:
1. ミキサー出力の高品質録音
2. マイク入力の録音（MC、ポッドキャスト）
3. 外部エフェクターの接続
4. マルチトラック録音
5. モニタリング環境の改善
```

### バッファサイズの最適化

**安定した録音のための設定:**

```
バッファサイズとは:
PCが音声データを処理する単位
小さい = 低遅延、CPU負荷高
大きい = 高遅延、CPU負荷低

DJ録音時の推奨:

リアルタイムモニタリングが必要な場合:
128〜256 サンプル
→ 遅延: 約3〜6ms（44.1kHz時）
→ CPU負荷: やや高い
→ 用途: ライブ配信、リアルタイムエフェクト

録音のみ（モニタリングはミキサーから）:
512〜1024 サンプル
→ 遅延: 約12〜23ms（44.1kHz時）
→ CPU負荷: 低い
→ 用途: ミックス録音（遅延は関係なし）

バッファサイズ別の遅延（44.1kHz）:
32:   0.7ms（超低遅延、不安定リスク）
64:   1.5ms（低遅延、高性能PC向け）
128:  2.9ms（低遅延、推奨最小値）
256:  5.8ms（標準、多くの用途に適合）
512:  11.6ms（安定重視）
1024: 23.2ms（最も安定、録音専用向け）
2048: 46.4ms（高遅延、特殊用途のみ）

PC性能別推奨:
高性能（M1 Pro以上/Core i7以上）: 128
中性能（M1/Core i5）: 256
低性能（Intel Core i3等）: 512

ドロップアウトが発生する場合:
1. バッファサイズを上げる（256→512）
2. 不要なアプリを閉じる
3. Wi-Fiを切断（バックグラウンド通信停止）
4. USBハブを使わず直接接続
5. USBポートを変更（USB3.0推奨）
```

## ファイル形式の技術的詳細

### 音声ファイル形式の完全比較

**それぞれの特性を理解する:**

```
非圧縮形式:

WAV（Waveform Audio File Format）:
- 開発: Microsoft / IBM
- 圧縮: なし
- ビット深度: 16/24/32bit
- サンプルレート: 44.1/48/96/192kHz
- ファイルサイズ: 約10MB/分（16bit/44.1kHz）
- 特徴: 最も互換性が高い非圧縮形式
- DJ録音推奨度: ★★★★★

AIFF（Audio Interchange File Format）:
- 開発: Apple
- 圧縮: なし
- ビット深度: 16/24/32bit
- サンプルレート: 44.1/48/96/192kHz
- ファイルサイズ: WAVとほぼ同じ
- 特徴: Macでの標準、メタデータ対応が良い
- DJ録音推奨度: ★★★★☆

可逆圧縮形式（ロスレス）:

FLAC（Free Lossless Audio Codec）:
- 開発: Xiph.org
- 圧縮: 可逆（50〜70%に圧縮）
- ビット深度: 最大32bit
- サンプルレート: 最大655.35kHz
- ファイルサイズ: WAVの約50〜70%
- 特徴: 圧縮しても音質劣化なし
- DJ録音推奨度: ★★★★☆

ALAC（Apple Lossless Audio Codec）:
- 開発: Apple
- 圧縮: 可逆（FLACと同程度）
- 特徴: Apple製品との互換性
- DJ録音推奨度: ★★★☆☆

非可逆圧縮形式（ロッシー）:

MP3（MPEG Audio Layer-3）:
- 開発: Fraunhofer
- 圧縮: 非可逆
- ビットレート: 128〜320kbps
- ファイルサイズ: WAVの約10〜20%
- 特徴: 最も普及している圧縮形式
- 320kbpsなら聴感上ほぼ無劣化
- DJ録音推奨度: ★★★☆☆

AAC（Advanced Audio Coding）:
- 開発: MPEG
- 圧縮: 非可逆
- ビットレート: 128〜320kbps
- ファイルサイズ: MP3と同程度
- 特徴: MP3より効率的な圧縮
- 同ビットレートでMP3より高音質
- DJ録音推奨度: ★★★☆☆

OGG Vorbis:
- 開発: Xiph.org
- 圧縮: 非可逆
- 特徴: 完全無料・オープン
- DJ録音推奨度: ★★☆☆☆
```

### ファイルサイズ計算

**録音前にストレージを確認:**

```
WAV ファイルサイズの計算式:

サイズ（byte）= サンプルレート × ビット深度/8 × チャンネル数 × 秒数

16bit / 44.1kHz / ステレオの場合:
44100 × 16/8 × 2 = 176,400 byte/秒
= 約10.6 MB/分
= 約635 MB/60分

24bit / 44.1kHz / ステレオの場合:
44100 × 24/8 × 2 = 264,600 byte/秒
= 約15.9 MB/分
= 約953 MB/60分

24bit / 48kHz / ステレオの場合:
48000 × 24/8 × 2 = 288,000 byte/秒
= 約17.3 MB/分
= 約1,037 MB/60分

24bit / 96kHz / ステレオの場合:
96000 × 24/8 × 2 = 576,000 byte/秒
= 約34.6 MB/分
= 約2,074 MB/60分

MP3 ファイルサイズの目安:
128kbps: 約1 MB/分（60分 = 60MB）
192kbps: 約1.5 MB/分（60分 = 90MB）
256kbps: 約2 MB/分（60分 = 120MB）
320kbps: 約2.4 MB/分（60分 = 144MB）

FLAC ファイルサイズの目安:
WAVの約50〜70%
16bit/44.1kHz: 約5〜7 MB/分
24bit/44.1kHz: 約8〜11 MB/分

ストレージ必要量の目安（1ヶ月分）:

週2回、60分の練習録音:
WAV (16bit): 635MB × 8 = 約5GB
WAV (24bit): 953MB × 8 = 約7.6GB
FLAC: 約3.5〜5.3GB
MP3 (320k): 144MB × 8 = 約1.2GB

推奨: 録音専用に最低50GBを確保
→ 余裕を持って100GB推奨
```

## ビットレートとサンプルレートの完全ガイド

### サンプルレートとは

**音声のデジタル化の精度:**

```
アナログ → デジタル変換:

音声は連続的なアナログ波形
→ 一定間隔でサンプリング（測定）
→ デジタルデータに変換

サンプルレート = 1秒間のサンプリング回数

44.1kHz = 1秒間に44,100回サンプリング
48kHz = 1秒間に48,000回サンプリング
96kHz = 1秒間に96,000回サンプリング
192kHz = 1秒間に192,000回サンプリング

ナイキストの定理:
サンプルレートの半分の周波数まで正確に再現可能

44.1kHz → 最大22.05kHz（人間の聴覚上限: 約20kHz）
48kHz → 最大24kHz
96kHz → 最大48kHz

つまり:
44.1kHzで人間が聴ける全ての周波数をカバー
→ CDクオリティとして採用されている理由

DJ録音での推奨:

44.1kHz:
- CD品質、業界標準
- ファイルサイズ最小
- Rekordboxの標準設定
- SoundCloud/Mixcloud推奨
- 推奨度: ★★★★★

48kHz:
- 映像制作の標準（YouTube等）
- 映像と組み合わせる場合はこちら
- 44.1kHzとの音質差はほぼ不明
- 推奨度: ★★★★☆

96kHz:
- ハイレゾ音源
- オーバーサンプリングの恩恵
- ファイルサイズ2倍
- 一般的なDJ用途では過剰
- 推奨度: ★★☆☆☆

192kHz:
- 超ハイレゾ
- マスタリングスタジオ向け
- DJ録音には不要
- 推奨度: ★☆☆☆☆
```

### ビット深度とは

**音量の解像度:**

```
ビット深度 = 音量の段階数

16bit = 65,536段階（ダイナミックレンジ約96dB）
24bit = 16,777,216段階（ダイナミックレンジ約144dB）
32bit float = 約40億段階（理論上無限のダイナミックレンジ）

分かりやすく例えると:

16bit = 定規の1mm刻み
24bit = 定規の0.004mm刻み
32bit float = 定規の0.0000002mm刻み

DJ録音での推奨:

16bit:
- CDクオリティ
- ダイナミックレンジ96dB（十分）
- ファイルサイズ最小
- 最終配信用としては十分
- 推奨度: ★★★★☆

24bit:
- プロスタジオ標準
- ダイナミックレンジ144dB
- 録音時のヘッドルームが広い
- ポスプロ処理に余裕がある
- 推奨度: ★★★★★

32bit float:
- 理論上クリップしない
- 一部のDAWと録音ソフトが対応
- ファイルサイズ大
- 推奨度: ★★★☆☆（対応環境があれば）

実践的アドバイス:

録音時: 24bit推奨
→ ヘッドルームに余裕があり安全

最終ファイル: 16bit で十分
→ 配信プラットフォームは16bitが標準

ワークフロー:
24bitで録音 → 編集・マスタリング → 16bitで書き出し
```

### ビットレート（MP3/AAC）

**圧縮音声の品質指標:**

```
ビットレート = 1秒間のデータ量

CBR（固定ビットレート）:
全区間が同じビットレート
320kbps = 常に320kbps

VBR（可変ビットレート）:
音の複雑さに応じて変動
複雑な音 → 高ビットレート
シンプルな音 → 低ビットレート

MP3 ビットレート別の品質:

128kbps:
- 明らかに音質劣化あり
- 高域が曇る
- ステレオ感が弱い
- 推奨しない

192kbps:
- 注意深く聴けば劣化を感じる
- カジュアルリスニングには十分
- デモ配布用

256kbps:
- ほとんどの人が劣化を感じない
- コスパの良い選択肢
- ストリーミング配信用

320kbps:
- 聴感上ほぼ無劣化
- ABXテストで判別困難
- 最高品質MP3
- SoundCloud/Mixcloud投稿用

DJ録音での推奨:
MP3を使う場合は必ず320kbps CBR

理由:
- DJミックスは周波数帯域が広い
- トランジション部分は音が複雑
- 低ビットレートだとアーティファクトが目立つ
- 320kbpsならほぼ問題なし
```

## DJミックス録音の実践ワークフロー

録音機材と設定の基本を理解したら、次は実際のDJミックス録音の流れを詳しく見ていきましょう。ここでは、練習録音からプロ品質のデモ作成まで、段階別のワークフローを解説します。

### 練習録音ワークフロー

**日常的な練習を効率よく録音:**

```
フェーズ1: 事前準備（録音30分前）

環境整備:
□ 部屋を静かにする
  - エアコン → 録音中は停止推奨（騒音源）
  - 窓を閉める
  - スマホ → サイレントモード
  - PC通知 → おやすみモード（Mac）/ 集中モード（Windows）

機材確認:
□ DDJ-FLX4 → USBケーブル接続確認
□ ヘッドフォン → 両耳動作確認
□ スピーカー → 適切な音量に設定
□ Rekordbox → 最新バージョン確認

プレイリスト:
□ 使用する曲を選択（12〜20曲/60分の場合）
□ 曲順のラフプランを作成
□ キーの互換性を確認（Camelot Wheel）
□ BPM範囲を確認（変動幅±5BPM推奨）

フェーズ2: サウンドチェック（録音10分前）

レベル設定:
1. 最もラウドな曲をロード
2. 再生してマスターレベルを確認
3. ピーク: -6dB〜-3dB に調整
4. 最も静かな曲でも確認
5. 曲間の音量差を把握

ゲインマッチング:
1. デッキAに曲1をロード
2. デッキBに曲2をロード
3. 両方再生してTRIMで音量合わせ
4. メーターが同じレベルになるよう調整

テスト録音:
1. [REC] ボタンで録音開始
2. 30秒間再生
3. [REC] で停止
4. 録音ファイルを再生して確認
5. 問題なければ本番へ

フェーズ3: 本番録音

開始手順:
1. 深呼吸、リラックス
2. [REC] ボタン → LED点灯確認
3. 2〜3秒の無音
4. 最初の曲をフェードイン
5. 楽しむ！

録音中の心得:
- ミスを恐れない → 誰でもミスする
- ミスしても止めない → 続行が鉄則
- レベルメーターを定期的にチェック
- タイムスタンプメモを取る
- 水分補給を忘れずに

録音終了:
1. 最後の曲を自然にフェードアウト
2. 3〜5秒の無音を残す
3. [REC] ボタン → LED消灯確認
4. 保存確認

フェーズ4: 録音後の確認

即時確認:
□ ファイルが正しく保存されたか
□ ファイルサイズが妥当か（60分WAV ≒ 600MB）
□ 冒頭と末尾を再生して確認
□ 大きな問題（無音、ノイズ）がないか

翌日確認:
□ 全体を通して聴く
□ タイムスタンプメモと照合
□ 良かった点・改善点を記録
□ 次回の練習計画に反映
```

## マルチトラック録音

### マルチトラック録音とは

**プロレベルの録音手法:**

```
シングルトラック録音（通常）:
DJミキサーの出力 → 1つのステレオファイル
= 全てが1つにミックスされた状態
= 後から個別調整不可

マルチトラック録音:
DJミキサーの各チャンネル → 個別のステレオファイル
= 各チャンネルが独立して録音される
= 後から個別に調整可能

メリット:
- トランジション部分を後から微調整できる
- 各チャンネルのEQ/エフェクトを個別に後処理
- ミスしたトランジションを修正可能
- マスタリング時の柔軟性が高い

デメリット:
- 録音設備が複雑になる
- ファイルサイズが大幅に増加
- 後処理の手間が増える
- セットアップに時間がかかる

必要な機材:
1. マルチ出力対応DJミキサー
   （DJM-A9、DJM-900NXS2、Xone:96等）
2. マルチ入力対応オーディオインターフェース
   （4ch以上の入力）
3. DAWソフトウェア
   （Ableton Live、Logic Pro、Pro Tools等）

推奨対象:
- プロDJ
- レコードレーベル提出用
- 映像作品と合わせる場合
- 完璧な音質を求める場合
```

### マルチトラック録音のセットアップ

**具体的な接続と設定:**

```
構成例: DJM-900NXS2 + RME Babyface Pro

物理接続:
DJM-900NXS2 MASTER OUT → Babyface Pro INPUT 1-2（ステレオ）
DJM-900NXS2 BOOTH OUT → Babyface Pro INPUT 3-4（ステレオ）
（または各チャンネルのSEND/RETURNを活用）

USB接続の場合（DJM-900NXS2のUSB録音機能）:
DJM-900NXS2 → USB → PC
→ 最大4ch（マスター + 各チャンネル）を独立録音可能

DAW設定（Ableton Live の場合）:

Step 1: オーディオ設定
- 環境設定 > Audio
- Audio Device: RME Babyface Pro
- サンプルレート: 44100Hz
- バッファサイズ: 512

Step 2: トラック作成
- Track 1: マスター出力（Input 1-2）
- Track 2: デッキA / チャンネル1（Input 3-4 等）
- Track 3: デッキB / チャンネル2
- Track 4: マイク入力（使用時）

Step 3: 録音アーム
- 各トラックの録音ボタンをON
- 入力モニタリングを設定
- レベル確認

Step 4: 同時録音開始
- Arrangement View で録音
- 全トラックが同期して録音される
- セッション全体が一つのプロジェクトに

後処理の例:
- Track 1（マスター）: リファレンスとして使用
- Track 2（デッキA）: 個別にEQ補正
- Track 3（デッキB）: 個別にEQ補正
- ミックスダウン: 各トラックを再バランスして書き出し
```

録音が完了したら、ポストプロダクション（後処理）で音質をさらに向上させることができます。ここでは、DJミックスに特化したポスプロ処理を解説します。

### ポスプロの基本ステップ

**録音後の処理フロー:**

```
Step 1: 素材確認
↓
Step 2: トリミング（冒頭/末尾の不要部分除去）
↓
Step 3: ノイズ除去（必要な場合）
↓
Step 4: レベル調整（ノーマライズ / コンプレッション）
↓
Step 5: EQ補正（必要な場合）
↓
Step 6: フェードイン / フェードアウト
↓
Step 7: マスタリング
↓
Step 8: 書き出し（WAV / MP3）
↓
Step 9: メタデータ追加
↓
Step 10: 品質チェック（異なる再生環境で確認）

所要時間の目安:
シンプルな処理: 15〜30分
丁寧な処理: 1〜2時間
プロ品質のマスタリング: 3〜4時間

どこまで処理するかの判断:

練習録音:

SoundCloud/Mixcloud投稿:
```

```
4. 楽曲に合わせて調整
```

## ノイズ除去テクニック

### ノイズの種類と対処法

**クリーンな録音を実現する:**

```
ノイズの種類:

1. ホワイトノイズ / ヒスノイズ:
   原因: 機材のアナログ回路、ケーブル
   特徴: 「シャーッ」という持続的な音
   対処: ノイズゲート、ノイズリダクション

2. ハムノイズ（50Hz/60Hz）:
   原因: 電源のグランドループ
   特徴: 「ブーン」という低い持続音
   対処: ノッチフィルター、DI ボックス

3. クリックノイズ / ポップノイズ:
   原因: 接続不良、静電気
   特徴: 「パチッ」という断続的な音
   対処: ケーブル交換、接点クリーニング

4. デジタルノイズ:
   原因: USB干渉、Wi-Fi干渉
   特徴: 「ジジジ」という高周波ノイズ
   対処: フェライトコア、USBアイソレーター

5. 環境ノイズ:
   原因: エアコン、交通騒音、隣人
   特徴: 背景の持続的な騒音
   対処: 録音環境の改善、ノイズリダクション
```

### Audacity でのノイズ除去

**無料で効果的なノイズリダクション:**

```
Step 1: ノイズプロファイルの取得
1. ノイズのみの部分を選択（無音だが背景ノイズがある箇所）
   - 曲間の無音部分が最適
   - 最低0.5秒以上の長さが必要
2. Effect > Noise Reduction
3. 「Get Noise Profile」をクリック
   → ソフトがノイズの特徴を学習

Step 2: ノイズ除去の適用
1. 全体を選択（Ctrl+A / Cmd+A）
2. Effect > Noise Reduction
3. パラメータ設定:
   - Noise Reduction (dB): 6〜12（控えめから始める）
   - Sensitivity: 3〜6
   - Frequency Smoothing: 3
4. 「Preview」で確認
5. 問題なければ「OK」で適用

パラメータの目安:

軽いノイズ:
- Noise Reduction: 6dB
- Sensitivity: 3
- 音質への影響: 最小

中程度のノイズ:
- Noise Reduction: 12dB
- Sensitivity: 6
- 音質への影響: わずかにあり

重いノイズ:
- Noise Reduction: 18dB以上
- Sensitivity: 8以上
- 音質への影響: 明らかにあり（要注意）

注意点:
- やりすぎると音が不自然になる（ウォーターフォール効果）
- 段階的に適用（6dBずつ）
- 「元に戻す」で比較しながら調整
- 音楽が鳴っている部分には影響が少ない
```

### iZotope RX によるプロ品質のノイズ除去

**業界標準のノイズ除去ツール:**

```
iZotope RX の特徴:
- 音声修復の業界標準ソフト
- スペクトログラム表示で視覚的にノイズを確認
- AIベースのノイズ除去
- 個別のノイズタイプに特化したモジュール

主要モジュール:

Spectral De-noise:
- スペクトル分析に基づくノイズ除去

- リアルタイムプレビュー対応

Voice De-noise:
- マイク録音のノイズに特化
- MCパート/ポッドキャスト録音に最適

De-hum:
- ハムノイズ（電源由来）専用

De-click:
- クリック/ポップノイズ専用
- レコードのスクラッチノイズにも有効

De-clip:

- 録音レベルが高すぎた場合の救済

エディション:

DJ録音推奨: RX Elements で十分
```

## レベル調整とダイナミクス処理

### ノーマライズ

**録音レベルの最適化:**

```
ノーマライズとは:
音声全体を分析し、最大ピークが指定レベルになるよう
全体の音量を均一に上げ下げする処理

ピークノーマライズ:
最大ピークを0dB（または指定値）に設定
→ 全体が同じ比率で調整される
→ ダイナミクスは保持される

ラウドネスノーマライズ:
全体の平均ラウドネス（LUFS）を指定値に設定
→ プラットフォームの基準に合わせる
→ 異なる録音間の音量感を統一

Audacity でのノーマライズ:
1. 全体を選択（Ctrl+A）
2. Effect > Normalize
3. 設定:
   - Normalize peak amplitude to: -1.0 dB
   - Remove DC offset: チェック
4. OK

推奨設定:
- ピーク: -1.0 dB（ヘッドルーム確保）
- -0.3 dB〜-1.0 dB が一般的
- 0 dB にしない（一部のプレーヤーでクリップする可能性）

配信プラットフォーム別のラウドネス基準:
- SoundCloud: 特に規定なし（-14 LUFS推奨）
- Mixcloud: 特に規定なし（-14 LUFS推奨）
- YouTube: -14 LUFS
- Spotify: -14 LUFS
- Apple Music: -16 LUFS
- Podcast: -16〜-14 LUFS
```

### コンプレッション

**ダイナミクスの制御:**

```
コンプレッサーとは:
音量の大きい部分を抑えて、
全体の音量差（ダイナミックレンジ）を縮める処理

DJ録音でのコンプレッション:

注意: DJミックスは既にある程度コンプレッションされている
→ 追加のコンプレッションは控えめに
→ やりすぎると「つぶれた」音になる

推奨設定（軽いコンプレッション）:
- Threshold: -12dB〜-8dB
- Ratio: 2:1〜3:1（穏やかな圧縮）
- Attack: 10〜30ms（トランジェントを残す）
- Release: 100〜300ms（自然なリリース）
- Make-up Gain: 圧縮分を補正
- Knee: Soft（なめらかな圧縮開始）

やってはいけないこと:
✗ Ratio 10:1以上（リミッターに近い）
✗ Threshold -20dB以下（圧縮しすぎ）
✗ Attack 0ms（パンチがなくなる）
✗ Release 10ms以下（ポンピング現象）

Audacity でのコンプレッション:
1. 全体を選択
2. Effect > Compressor
3. パラメータ設定（上記参照）
4. Preview で確認
5. OK で適用

プロのアプローチ:
→ マスタリング段階でリミッターを使用
→ コンプレッションは最小限に
→ 「音圧を上げたい」だけならリミッターの方が適切
```

### リミッター

**ピーク制御と音圧確保:**

```
リミッターとは:
設定した閾値を超える音量を即座にカットする処理
= 非常に速いアタックのコンプレッサー

DJミックスでの使用目的:
1. クリッピング防止
2. 全体の音圧を上げる
3. 配信プラットフォーム用に最適化

推奨設定:
- Ceiling: -1.0 dB（True Peak）
- Threshold / Input Gain: +2〜+6 dB
  → 音圧を上げつつ、ピークを制限

注意:
- Input Gainを上げすぎると歪む
- 波形が「壁」のようになったらやりすぎ
- AB比較（処理前/後）で常に確認

無料のリミッターVSTプラグイン:
- Youlean Loudness Meter（測定用）
- TDR Limiter 6 GE（高品質リミッター）
- Frontier（シンプルで使いやすい）

有料のリミッターVSTプラグイン:
- FabFilter Pro-L 2（業界標準）
- iZotope Ozone Maximizer
- Waves L2 Ultramaximizer
```

## ライブ録音テクニック

クラブやイベントでのライブ録音は、自宅録音とは異なる課題があります。ここでは、ライブ環境で最高品質の録音を得るための実践的なテクニックを解説します。

### クラブ・イベントでのライブ録音

**現場録音の基本戦略:**

```
録音方法の選択肢:

方法1: DJミキサーからの直接録音（推奨）
接続: ミキサー REC OUT → レコーダー / オーディオI/F
品質: ★★★★★（最高品質）
```

## 配信用マスタリング

### DJミックスのマスタリング

**配信プラットフォームに最適化:**

```
マスタリングとは:
録音した音源を配信に適した状態に仕上げる最終工程
= 音質・音量を最適化する「磨き」の作業

DJ録音のマスタリングチェーン:

信号の流れ:
録音ファイル
  ↓
EQ（周波数バランス調整）
  ↓
コンプレッション（ダイナミクス調整）
  ↓
ステレオイメージング（広がり調整）
  ↓
リミッター（音圧・ピーク制御）
  ↓
ディザリング（ビット深度変換時）
  ↓
最終ファイル書き出し

各ステップの詳細:

EQ:
目的: 周波数バランスの微調整
設定例:
- ハイパスフィルター: 30Hz以下をカット（サブベース以下のノイズ除去）
- 低域 (60-100Hz): ±1dB（重低音のバランス）
- 中域 (1-5kHz): ±0.5dB（明瞭さの調整）
- 高域 (10kHz+): ±1dB（空気感の調整）
注意: 大きな補正が必要な場合は録音をやり直すべき

コンプレッション:
目的: 全体のダイナミクスを整える
設定: （前述のコンプレッション設定参照）
注意: DJミックスは元々圧縮されているため控えめに

ステレオイメージング:
目的: ステレオの広がりを調整
設定:
- 低域（300Hz以下）: モノラルに寄せる（位相問題回避）
- 高域: やや広げる（空間感向上）
- 過度な処理は避ける

リミッター:
目的: ピーク制御と音圧確保
設定:
- Ceiling: -1.0 dB True Peak
- ターゲットラウドネス: -14 LUFS
  （SoundCloud/YouTube基準）

ディザリング:
目的: 24bit → 16bit変換時のノイズ低減
設定:
- Type: TPDF（最も自然）
- 24bitで配信する場合は不要
```

### プラットフォーム別の書き出し設定

**各サービスに最適な形式:**

```
SoundCloud:
- 推奨形式: WAV 16bit/44.1kHz
- 代替: FLAC, AIFF
- 最大ファイルサイズ: 5GB（Pro Unlimited）
- ラウドネス: 自動ノーマライズあり
- 備考: WAVアップロードが最高品質
       SoundCloudが自動的にストリーミング用に変換

Mixcloud:
- 推奨形式: MP3 320kbps / WAV
- 最大ファイルサイズ: 1GB
- 最大長さ: 4時間
- ラウドネス: 特に処理なし
- 備考: DJミックス向けプラットフォーム
       著作権クリアランス対応

YouTube:
- 推奨形式: WAV 16bit/48kHz（映像と合わせる場合）
- 音声のみ: WAV + 静止画 で動画化
- ラウドネス: -14 LUFS にノーマライズ
- 備考: サンプルレート48kHzが映像標準

Podcast配信:
- 推奨形式: MP3 192〜320kbps
- ラウドネス: -16 LUFS（Podcast基準）
- 備考: モノラルの場合はビットレート半分でOK

デモ提出（プロモーター/レーベル）:
- 推奨形式: WAV 16bit/44.1kHz + MP3 320kbps
- ラウドネス: -14〜-10 LUFS
- 備考: 両方の形式を用意するのがベスト
```

## 録音トラブルシューティング詳細ガイド

基本的なトラブルシューティングに加え、より詳細な問題と解決策を網羅します。

### 音切れ・ドロップアウト

**録音中に音が途切れる:**

```
原因と対策:

原因1: CPU過負荷
症状: 不定期に音が途切れる、プチプチとした音
対策:
1. 不要なアプリケーションを全て終了
2. ブラウザ（Chrome等）は閉じる（メモリ消費大）
3. バッファサイズを上げる（256→512→1024）
4. Rekordboxの不要な機能をOFF
   - ビデオ機能OFF
   - ライブラリのリアルタイムスキャンOFF
5. PC省電力モードをOFF → 高パフォーマンスモードに

原因2: USB帯域不足
症状: 定期的に音が途切れる
対策:
1. USBハブを使わず直接接続
2. 他のUSBデバイスを取り外す
3. USB 3.0ポートを使用（2.0より帯域広い）
4. ケーブルを交換（劣化している可能性）

原因3: ストレージ速度不足
症状: 長時間録音で途中から途切れ始める
対策:
1. SSDに録音する（HDDより高速）
2. 外付けストレージの場合はUSB 3.0以上で接続
3. ストレージの空き容量を確保（最低20%以上）
4. デフラグ実行（HDDの場合、Windowsのみ）

原因4: ドライバーの問題
症状: 特定の操作で音が途切れる
対策:
1. オーディオドライバーを最新版に更新
2. ASIO4ALL（Windows）の使用を検討
3. ドライバーの再インストール
4. OS のサウンド設定で排他モードを確認
```

### 左右のバランス問題

**ステレオバランスの異常:**

```
症状: 左右の音量が違う / 片方しか音が出ない

原因1: ケーブル接続不良
対策:
- RCAケーブルのL/R確認
- 接点のクリーニング
- ケーブル交換

原因2: ミキサーのパン設定
対策:
- 各チャンネルのPAN/BALが中央にあるか確認
- マスターのバランスも確認

原因3: オーディオI/Fの入力設定
対策:
- ステレオ入力として認識されているか確認
- 入力チャンネルの割り当て確認
- モノラル入力×2になっていないか確認

原因4: 位相反転
症状: 音が薄い、中抜けした音になる
対策:
- ケーブルの配線確認
- オーディオI/Fの位相反転ボタン確認
- モノラルで再生してテスト（音が消えたら位相反転）

確認方法:
1. テストトーン（ステレオ）を再生
2. 左右のメーターが均等に振れるか確認
3. ヘッドフォンで左右を個別に確認
4. モノラル切り替えで位相チェック
```

### 録音品質が想定より低い

**音質問題の診断と対策:**

```
症状: 録音がこもっている / 薄い / 解像度が低い

チェックポイント:

1. 録音フォーマットの確認
- WAV 44.1kHz/24bit になっているか
- 誤ってMP3 128kbpsで録音していないか
- サンプルレートが意図通りか

2. ゲイン設定
- 録音レベルが低すぎないか（-20dB以下は問題）
- 低すぎる録音は後からゲインを上げるとノイズも増幅
- 適正レベル: -6dB〜-3dBピーク

3. ケーブル品質
- 安価なケーブルは高域が落ちる場合がある
- バランス接続を使用しているか
- ケーブル長が長すぎないか（信号劣化）

4. サンプルレートの不一致
- ミキサー/I/F/DAWのサンプルレートが統一されているか
- 不一致は品質低下とクリックノイズの原因

5. モニタリング環境
- 実は録音品質は問題なく、モニター環境の問題の場合も
- 異なるスピーカー/ヘッドフォンで確認
- 車、スマホ、ノートPCスピーカーなど複数環境で確認
```

### 録音ファイルの破損

**ファイルが再生できない / 途中で切れる:**

```
原因:
- 録音中の強制終了（PC クラッシュ、電源断）
- ストレージの故障
- ファイル転送中のエラー

対策:

WAVファイルの修復:
1. Audacity で開く → 破損部分を自動スキップする場合がある
2. WAV Repair Tool（無料）で修復を試みる
3. FFmpeg でコンテナの再構築:
   ffmpeg -i broken.wav -c copy repaired.wav

予防策:
- UPS（無停電電源装置）の使用
- 録音中はPCの電源管理を無効に
- 長時間録音は途中で分割保存
- 定期的に自動保存する録音ソフトを使用

バックアップからの復旧:
- Time Machine / File History からの復元
- クラウド同期サービスのバージョン履歴
- 録音中もリアルタイムでクラウド同期していれば安心
```

## 録音環境の最適化

### ホームスタジオの防音・吸音

**録音品質を左右する部屋の音響:**

```
自宅DJ録音の音響対策:

DDJ-FLX4 + Rekordbox の場合:
→ 基本的にPC内部録音なので部屋の音響は録音に影響しない
→ ただし、スピーカーモニタリングの精度に影響

外部ミキサー + マイク録音の場合:
→ 部屋の反響が録音に入る可能性あり
→ 吸音対策が重要

簡易吸音対策:
1. 厚手のカーテン: 窓からの反射を低減
2. 本棚（本がたくさん）: 拡散効果
3. ラグ/カーペット: 床からの反射を低減
4. クッション/ソファ: 中低域の吸音
5. 吸音パネル: 壁の一次反射点に設置

マイク録音時の対策:
1. リフレクションフィルター使用
2. マイクと壁の距離を確保
3. 部屋の角を避ける（低域が溜まる）
4. ポップフィルター使用（破裂音対策）

騒音対策:
1. 録音時間帯の選択（深夜は避ける/選ぶ）
2. エアコン停止（ノイズ源）
3. PC冷却ファンの静音化
4. スマートフォンの機内モード
5. 同居人への事前通知
```
