## 著作権の基礎

### 著作権とは

**創作物を守る権利:**

```
著作権:
音楽、絵画、文章など
創作物を作った人の権利

音楽の場合:

1. 著作権（楽曲）
   作詞家、作曲家の権利
   例: 歌詞、メロディ

2. 著作隣接権（音源）
   実演家、レコード会社の権利
   例: ボーカル、演奏、録音

2つの権利が存在
両方をクリアする必要
```

### 保護期間

**いつまで保護される？**

```
著作権:
作者の死後70年
（2018年12月30日以降）

著作隣接権:
発行後70年

パブリックドメイン:
保護期間が切れた作品
→ 自由に使える

例:
ベートーヴェン、モーツァルト
→ 楽曲は自由

でも:
現代の録音（演奏）
→ まだ保護されている

注意必要
```

## 著作権の歴史と発展

### 著作権制度の起源

**印刷技術が著作権を生んだ:**

```
歴史的経緯:

1455年: グーテンベルク活版印刷術
  → 書物の大量複製が可能に
  → 「コピー」の概念が社会問題化

1710年: アン法（イギリス）
  → 世界初の著作権法
  → 著者に14年間の独占権を付与
  → 更新で最大28年に延長可能

1790年: アメリカ著作権法
  → 合衆国憲法に著作権条項
  → 14年間の保護（更新で28年）

1886年: ベルヌ条約
  → 国際的な著作権保護の枠組み
  → 「無方式主義」を採用
  → 登録なしで著作権発生

1899年: 日本がベルヌ条約に加盟
  → 国際的な著作権保護に参加

1970年: 日本の現行著作権法
  → 全面改正により現在の形に

2018年: TPP11発効
  → 日本の保護期間が死後50年→70年に延長
```

### 音楽著作権の発展

**録音技術が著作権を複雑にした:**

```
音楽著作権の歴史:

1877年: エジソン蓄音機発明
  → 音楽を「固定」できるように
  → 著作権の概念が拡張

1898年: 日本初の蓄音機輸入
  → 音楽ビジネスの始まり

1910年: 日本著作権法（旧法）
  → 音楽の著作権保護開始

1939年: JASRAC設立
  → 大日本音楽著作権協会として
  → 音楽著作権の集中管理開始

1971年: レコード保護条約
  → 海賊版対策の国際枠組み

1996年: WIPO著作権条約
  → デジタル時代の著作権
  → インターネット配信に対応

2001年: iTunes Store登場
  → デジタル音楽配信の幕開け
  → 著作権管理がさらに複雑に

2016年: NexTone設立
  → JASRAC独占体制の変化
  → 著作権管理の選択肢が増加

現在:
ストリーミング時代
→ 著作権管理はますます複雑
→ AIの登場でさらに新たな課題
```

## 著作権の基本概念を深く理解する

### 著作者人格権

**著作者の人格的利益を守る権利:**

```
著作者人格権の3つの権柱:

1. 公表権
   未発表の著作物を公表するかどうか
   いつ、どのように公表するか
   著作者が決定する権利

   DJとの関連:
   - 未発表トラックのリーク禁止
   - プロデューサーの意思を尊重
   - プレビュー音源の取り扱い注意

2. 氏名表示権
   著作物に著作者名を表示するか
   本名/ペンネームどちらで表示するか
   著作者が決定する権利

   DJとの関連:
   - トラックリストでの正しいクレジット
   - リミックスでの原作者表示
   - プレイリスト公開時の注意

3. 同一性保持権
   著作物の内容を勝手に変更されない権利
   意に反する改変の禁止

   DJとの関連:
   - リミックス/エディットの問題
   - マッシュアップへの影響
   - ピッチ変更、カット、ループ

重要:
著作者人格権は譲渡不可
著作者の死亡で消滅
（遺族が遺志を守る権利あり）
```

### 著作財産権の種類

**お金に関わる権利の全体像:**

```
著作財産権（経済的権利）:

1. 複製権（第21条）
   著作物をコピーする権利
   - CD のリッピング
   - 楽曲のダウンロード
   - DJ用にファイル変換

2. 上演権・演奏権（第22条）
   公に演奏する権利
   - クラブでの DJ プレイ
   - フェスでのパフォーマンス
   - BGMとしての使用

3. 公衆送信権（第23条）
   インターネット等で送信する権利
   - ライブ配信
   - ポッドキャスト
   - ミックスのアップロード

4. 送信可能化権
   サーバーにアップロードする権利
   - SoundCloud へのアップ
   - YouTube へのアップ
   - 自分のWebサイト

5. 譲渡権（第26条の2）
   著作物の複製物を譲渡する権利
   - CD の販売
   - デジタルファイルの販売

6. 貸与権（第26条の3）
   著作物の複製物を貸す権利
   - レコードの貸し出し
   - CD レンタル

7. 翻案権（第27条）
   著作物を改変して新しい作品を作る権利
   - リミックス
   - サンプリング
   - マッシュアップ

DJに特に重要:
演奏権 → クラブプレイ
公衆送信権 → 配信
翻案権 → リミックス制作
```

### 著作隣接権の詳細

**実演家・レコード会社の権利:**

```
著作隣接権の権利者:

1. 実演家（アーティスト/ミュージシャン）
   - 録音権・録画権
   - 放送権・有線放送権
   - 送信可能化権
   - 譲渡権
   - 貸与権
   - 二次使用料請求権

   具体例:
   ボーカリストの歌唱
   ギタリストの演奏
   DJ のミックスパフォーマンス（録音物）

2. レコード製作者（レーベル）
   - 複製権
   - 送信可能化権
   - 譲渡権
   - 貸与権
   - 二次使用料請求権

   具体例:
   レコーディングスタジオでの録音
   マスタリング済みの音源
   リリースされた楽曲の音源

3. 放送事業者
   - 複製権
   - 再放送権
   - テレビジョン放送の伝達権

4. 有線放送事業者
   - 複製権
   - 放送権
   - 再有線放送権

DJに重要な実務ポイント:

著作権（楽曲）の許可 ≠ 著作隣接権（音源）の許可

例:
JASRACで楽曲の許可を得ても
レコード会社の原盤使用許可は別途必要

ライブ配信:
→ 著作権（JASRAC/NexTone）
→ 著作隣接権（レコード会社/アーティスト）
両方の処理が必要
```

## クリエイティブ・コモンズ（CC）

### CCライセンスとは

**著作権の柔軟な運用:**

```
クリエイティブ・コモンズ:
著作権者が自ら条件を設定して
作品の利用を許可する仕組み

利点:
- 個別に許可を取る必要なし
- 条件を守れば自由に利用可能
- 国際的に統一された仕組み

基本要素（4つ）:

BY（表示）:
  著作者のクレジットを表示すること
  ほぼ全てのCCライセンスに含まれる

SA（継承）:
  改変した場合、同じライセンスで公開すること
  「コピーレフト」的な考え方

NC（非営利）:
  営利目的での利用を禁止
  個人使用、教育目的はOK

ND（改変禁止）:
  作品を改変してはいけない
  そのままの形で利用すること
```

### CCライセンスの組み合わせ

**6種類のライセンス:**

```
1. CC BY
   表示のみ
   最も自由度が高い
   クレジット表示すれば何でもOK
   DJ利用: ◎ リミックスも商用もOK

2. CC BY-SA
   表示 + 継承
   改変OK、同じライセンスで公開
   DJ利用: ○ リミックスOK（同じCCで公開）

3. CC BY-NC
   表示 + 非営利
   非営利目的なら自由に利用
   DJ利用: △ 無料イベントならOK、有料ギグは×

4. CC BY-NC-SA
   表示 + 非営利 + 継承
   非営利で改変OK、同じライセンス
   DJ利用: △ 限定的

5. CC BY-ND
   表示 + 改変禁止
   そのまま利用のみ
   DJ利用: × リミックス不可、プレイのみ

6. CC BY-NC-ND
   表示 + 非営利 + 改変禁止
   最も制限的
   DJ利用: × ほぼ使えない

CC0（パブリックドメイン）:
  全ての権利を放棄
  完全に自由に利用可能
  DJ利用: ◎ 何でもOK
```

### DJにとってのCC音源

**CC音源の入手先と活用法:**

```
CC音源の入手先:

1. Free Music Archive（FMA）
   URL: freemusicarchive.org
   特徴:
   - 大量のCC音源
   - ジャンル別に検索可能
   - ダウンロード無料
   DJ向け: エレクトロニカ、アンビエント豊富

2. ccMixter
   URL: ccmixter.org
   特徴:
   - リミックス素材特化
   - ステム（パーツ）も入手可能
   - コミュニティベース
   DJ向け: サンプル素材として優秀

3. Jamendo
   URL: jamendo.com
   特徴:
   - 商用ライセンスも対応
   - 高品質な楽曲多数
   - ジャンル豊富
   DJ向け: BGM用途に最適

4. Bandcamp（一部CC）
   URL: bandcamp.com
   特徴:
   - 一部アーティストがCCで公開
   - 高音質
   - 直接アーティスト支援
   DJ向け: インディー楽曲発掘

5. SoundCloud（一部CC）
   一部楽曲がCCライセンス
   ダウンロード可能な楽曲を探す
   フィルターで「To use commercially」

注意点:
- ライセンス条件を必ず確認
- クレジット表示を忘れない
- NC（非営利）の楽曲に注意
- 「著作権フリー」と「CC」は違う
```

## DJプレイと著作権

### クラブ・バーでのプレイ

**会場が使用料を払っている:**

```
通常:

クラブ/バー/イベント会場:
JASRAC、NexTone と包括契約
→ 使用料を年間/月間で支払い

DJ は:
自由にプレイ OK

ただし:

小さいバー、カフェ:
契約していない場合も

確認:
「著作権の手続きは？」
→ プロモーター/オーナーに

DJ の責任:
通常なし
（会場が契約してれば）

でも:
録音・配信は別
```

### プライベートパーティ

**注意が必要:**

```
公の場:
著作権使用料必要

例:
- 企業パーティ
- ウェディング（披露宴）
- 大規模な誕生日パーティ

手続き:
主催者が JASRAC に申請
→ 使用料支払い

プライベート（真の意味で）:
- 自宅パーティ（友人のみ）
- 家族の集まり

→ 使用料不要

DJ として:
「著作権の手続きは？」
→ 事前確認必須
```

### DJ活動における著作権の実務的判断

**場面ごとの法的整理:**

```
場面別・著作権処理の必要性:

【クラブ/ライブハウスでのDJプレイ】
著作権（演奏権）: 会場が包括契約で処理
著作隣接権: 処理不要（生演奏扱い）
結論: DJは特に手続き不要

【野外フェスティバル】
著作権: 主催者がJASRACに申請
著作隣接権: 処理不要（生演奏扱い）
結論: 主催者側で処理

【ウェディングDJ】
著作権: 式場が包括契約している場合が多い
著作隣接権: 処理不要
注意: 式場に確認必須
  - 契約していない式場もある
  - 持ち込みDJの場合は特に注意

【企業イベント】
著作権: 主催企業が申請すべき
著作隣接権: 処理不要（通常）
注意:
  - 録画・配信がある場合は別途処理
  - 社内イベントでも必要

【ストリートパフォーマンス】
著作権: 道路使用許可とは別に必要
著作隣接権: 処理不要
注意:
  - 路上ライブと同じ扱い
  - 投げ銭は「営利」と見なされる可能性

【オンラインDJイベント】
著作権: 配信者が処理（包括契約 or 個別）
著作隣接権: レコード会社の許可必要
注意:
  - 最もハードルが高い
  - 著作権と著作隣接権の両方が必要
  - Mixcloud Live等の利用を推奨

判断に迷ったら:
→ 著作権の専門家に相談
→ 弁護士費用: 相談30分 ¥5,000〜
→ 日本弁護士連合会の法律相談窓口
```

### DJミックスの法的位置づけ

**DJミックスは著作物か？**

```
DJミックスの法的性質:

1. 選曲の著作物性
   - 楽曲の選択と配列に創作性があれば
     → 「編集著作物」になりうる
   - ただし、個々の楽曲の著作権は別

2. ミキシングの著作物性
   - EQ、エフェクト、トランジション
   - 創作的な表現と認められる可能性
   - ただし判例は少ない

3. DJ自身の権利
   - DJプレイが「実演」と認められれば
     → 著作隣接権が発生
   - 録音されたDJミックス
     → DJに実演家の権利

4. 他者の権利との関係
   - 使用楽曲の著作権は残る
   - レコード会社の原盤権も残る
   - DJミックスの配布には全ての許可が必要

実務上のポイント:
- DJミックスは「二次的著作物」に該当する可能性
- 原著作物の著作権者の許可なく作成・配布は違法
- しかし業界慣行として黙認されているケースが多い
- プロモーション効果を認めるレーベルも増加

グレーゾーンの現実:
多くのDJミックスは法的にはグレー
↓
レーベル/アーティストの対応
- 積極的に取り締まる ← メジャーレーベルに多い
- 黙認する ← インディーレーベルに多い
- 推奨する ← プロモーション重視のレーベル
```

## JASRAC と NexTone

### 2つの団体

**音楽著作権を管理:**

```
JASRAC（ジャスラック）:
日本音楽著作権協会
設立: 1939年
管理楽曲: 約200万曲以上
シェア: 約95%

NexTone（ネクストーン）:
設立: 2016年
管理楽曲: 約10万曲
シェア: 約5%

役割:
- 著作権者の代理
- 使用料徴収
- 著作権者に分配

DJ/会場:
どちらかと契約
→ 両方の楽曲使用可能
```

### 使用料の仕組み

**会場が支払う:**

```
クラブ・ライブハウス:

包括契約:
年間 or 月間定額

金額:
会場の広さ、収容人数による

例:
小規模クラブ（50人）:
月額 ¥10,000-30,000

中規模クラブ（200人）:
月額 ¥50,000-100,000

大規模クラブ（500人+）:
月額 ¥100,000-300,000

イベント単位:

都度申請:
1回のイベントごと

金額:
入場料 × 観客数 × 数%

例:
入場料 ¥3,000 × 100人 × 5%
= ¥15,000

DJ:
通常支払い不要
会場/主催者が負担
```

### JASRACの許可申請手順

**実際にどう申請するか:**

```
JASRACへの利用許可申請:

1. オンライン申請（J-OPUS）
   URL: j-opus.jasrac.or.jp
   24時間受付
   最も簡単な方法

   手順:
   ① アカウント作成
   ② 利用形態を選択
   ③ 楽曲情報入力
   ④ 使用料確認
   ⑤ 申請送信
   ⑥ 使用料支払い

2. 窓口申請
   全国の支部で対応
   東京本部: 渋谷区上原

3. 郵送申請
   申請書をダウンロード
   記入して郵送

利用形態別の手続き:

【演奏（ライブ・イベント）】
申請先: JASRAC演奏部
必要情報:
- 開催日時、場所
- 入場料、定員
- 使用楽曲リスト（分かる範囲）
申請期限: 開催日の2週間前まで

【配信・アップロード】
申請先: JASRAC複製部
必要情報:
- 配信プラットフォーム
- 使用楽曲リスト
- 配信期間
注意: 著作隣接権は別途処理

【BGM利用】
申請先: JASRAC演奏部
包括契約が便利
月額: 面積に応じた定額

使用料の支払い方法:
- 銀行振込
- クレジットカード
- コンビニ払い

領収書:
確定申告に使用可能
経費として計上できる
```

## 録音・配信の著作権

### 録音の注意点

**個人使用 vs 公開:**

```
個人使用:
自分で聴くだけ
→ OK、問題なし

公開:
SoundCloud、YouTube 等
→ 著作権処理必要

問題:
DJ ミックスは他人の楽曲使用
→ 許可が必要

## リミックス・ブートレグの法的問題

### リミックスの法的位置づけ

**原曲の著作権者の許可が必要:**

```
リミックスの種類と法的リスク:

1. オフィシャルリミックス
   レーベル/アーティストから正式に依頼
   → 契約書で権利関係を明確化
   → 完全に合法
   → 収益分配も契約で決定

   プロセス:
   ① レーベルからステムを受領
   ② リミックス制作
   ③ レーベルに納品
   ④ レーベルがリリース
   ⑤ 収益は契約に基づき分配

2. アンオフィシャルリミックス
   アーティスト/レーベルの許可なし
   → 原則として著作権侵害
   → ただし黙認されるケースも多い

   リスク要因:
   - 原曲の知名度（高いほどリスク大）
   - レーベルのポリシー（厳格/寛容）
   - 配信方法（無料/有料）
   - 利益の有無

3. ブートレグリミックス
   非公式、無許可の改変
   → 最もグレーなカテゴリ
   → SoundCloudで多数存在

   典型的なブートレグ:
   - A曲のアカペラ + B曲のインスト
   - 有名曲のダンスフロア向けエディット
   - メジャー曲のジャンル変換
```

### ブートレグの現状と対策

**業界の暗黙のルール:**

```
ブートレグの実態:

文化的背景:
- DJ文化の伝統として存在
- ディスコ時代からの歴史
- 多くの有名DJもブートレグ出身

業界の対応:

寛容なケース:
- 無料配布のみ
- プロモーション効果あり
- アーティストが認知
- インディーレーベル

厳格なケース:
- 商用利用
- Spotify等での配信
- メジャーレーベルの楽曲
- 広告収入の獲得

リスク最小化のガイドライン:

□ 絶対に販売しない
□ ストリーミングサービスに配信しない
□ クレジットを明記する
  例: 「アーティスト名 - 曲名 (DJ名 Bootleg)」
□ 著作権者から削除要請があれば即対応
□ SoundCloudは「Free Download」タグで
□ 商用利用の問い合わせが来たら断る

安全策:
- 自作の要素を多く含める
- 元曲が判別しにくい変換
- CC音源のみでリミックス
- 許可を求めるDMを送る（返答なしでも記録に）
```

### マッシュアップの著作権

**複数楽曲の組み合わせ:**

```
マッシュアップ:
2曲以上の楽曲を組み合わせた作品

法的状況:
- 複数の著作権が関係
- 全ての原曲の許可が理論上必要
- 実務上は許可取得がほぼ不可能

マッシュアップの種類:

1. ボーカル + インスト
   A曲のボーカル + B曲の伴奏
   → 2つの著作権が関係

2. 複数曲のメドレー
   3曲以上を連続的に組み合わせ
   → 全曲の著作権が関係

3. サウンドコラージュ
   多数の音源の断片的使用
   → 著作権の範囲が曖昧

判例の傾向:
- 日本では明確な判例少ない
- 米国ではフェアユースの議論
- 商用でなければ訴えられにくい

DJプレイでの扱い:
- ライブでのマッシュアップ → 会場の包括契約内
- 録音・配信 → グレーゾーン
- 販売 → 確実にNG
```

## サンプリングクリアランス

### サンプリングとは

**他の楽曲の一部を使用すること:**

```
サンプリングの定義:
既存の楽曲から音の一部を取り出し
新しい楽曲に使用すること

歴史:
- 1970年代: ヒップホップの誕生と共に
- 1980年代: サンプラーの普及
- 1990年代: 訴訟の増加
- 現在: クリアランスが一般的に

有名なサンプリング訴訟:

1. Grand Upright Music v. Warner Bros. (1991)
   Biz Markie vs Gilbert O'Sullivan
   → サンプリングには許可が必要と明確化

2. Bridgeport Music v. Dimension Films (2005)
   → 「2秒のギターリフでも侵害」
   → de minimis（些少）の原則を否定

3. VMG Salsoul v. Ciccone (2016)
   Madonna "Vogue" のサンプリング
   → 0.23秒のホーンサンプルで訴訟
   → 第9巡回区はde minimisを認定

教訓:
- どんなに短くても許可を取るのが安全
- 「バレなければ」は通用しない
- テクノロジーで検出精度が向上
```

### クリアランスの手順

**正式にサンプリング許可を得る方法:**

```
サンプリングクリアランスのプロセス:

Step 1: 権利者の特定
  - 楽曲の著作権者（作詞家/作曲家）
  - 音源の原盤権者（レコード会社）
  - 両方の許可が必要

  調査方法:
  - JASRAC/NexTone の楽曲データベース
  - レコード会社のWebサイト
  - AllMusic、Discogs等のデータベース

Step 2: 許可の申請
  申請書に含める情報:
  - サンプリング元の楽曲情報
  - サンプリング箇所（秒数、パート）
  - 使用する新曲の情報
  - 配信方法、販売計画
  - 希望する条件

Step 3: 条件交渉
  一般的な条件:
  - 一括払いの使用料（$500〜$50,000+）
  - ロイヤリティ（売上の1〜5%）
  - 共同著作権のクレジット
  - 使用期間の限定
  - 地域の限定

Step 4: 契約締結
  契約書に明記すべき事項:
  - サンプリングの範囲
  - 使用料と支払い条件
  - クレジット表示方法
  - 権利の帰属
  - 契約解除条件

Step 5: クレジット表示
  リリース時に正確に表記:
  "Contains a sample from [曲名] by [アーティスト名]
   Used by permission of [権利者名]"

費用の目安:
  インディーアーティスト: $500〜$5,000
  中堅アーティスト: $5,000〜$25,000
  メジャーアーティスト: $25,000〜$100,000+
  超有名曲: 交渉次第（拒否される場合も）

注意:
許可が得られない場合もある
代替案: 似た音を自分で演奏/打ち込み
→ 「リプレイ」と呼ばれる手法
```

### サンプリングの代替手法

**リスクなしでサンプリング的表現を実現:**

```
合法的な代替手法:

1. リプレイ（Replay/Interpolation）
   原曲を参考に自分で演奏し直す
   → 原盤権の問題を回避
   → ただし楽曲の著作権は別途処理

   利点:
   - 原盤使用料不要
   - 音質のコントロールが可能
   - アレンジの自由度が高い

   注意:
   - メロディが同じなら著作権処理必要
   - コード進行だけなら著作権なし（通常）

2. ロイヤリティフリーサンプル
   Splice、Loopmasters等のサービス
   → 商用利用OK
   → 月額制で大量のサンプル利用可能

   主要サービス:
   - Splice: 月額$7.99〜
   - Loopmasters: 個別購入
   - LANDR: 月額制
   - Tracklib: サンプリング特化

3. パブリックドメイン音源
   著作権が切れた楽曲の音源
   → 古い録音を利用
   → ジャズ、クラシックに多い

4. 自分でフィールドレコーディング
   環境音、自然音を録音
   → 完全にオリジナル
   → ユニークなサウンド
```

## 国際的な著作権制度

### 主要な国際条約

**国境を越える著作権保護:**

```
音楽に関連する主要な国際条約:

1. ベルヌ条約（1886年）
   加盟国: 181か国
   原則:
   - 無方式主義（登録不要で保護）
   - 内国民待遇（自国民と同等の保護）
   - 遡及効（条約加入前の作品も保護）

   DJとの関連:
   - 海外楽曲も日本で保護される
   - 日本の楽曲も海外で保護される

2. WIPO著作権条約（1996年）
   デジタル環境に対応
   原則:
   - 公衆送信権の国際基準
   - 技術的保護手段の保護
   - 権利管理情報の保護

   DJとの関連:
   - インターネット配信の国際ルール
   - DRM回避の禁止

3. TRIPS協定（1995年）
   WTO加盟国に適用
   原則:
   - 知的財産権の最低基準
   - 紛争解決メカニズム

4. ローマ条約（1961年）
   実演家・レコード製作者の保護
   DJとの関連:
   - DJの実演家としての権利
   - レコード会社の国際的保護
```

### 各国の著作権法の違い

**海外活動時の注意点:**

```
国別の著作権制度比較:

【アメリカ】
保護期間: 著作者の死後70年（法人著作95年）
特徴:
- フェアユース制度あり
- 著作権登録制度（訴訟の前提条件）
- 法定損害賠償（最大$150,000/件）
DJ注意点:
- DMCAテイクダウン制度
- フェアユースの主張が可能（条件付き）

【イギリス】
保護期間: 著作者の死後70年
特徴:
- フェアディーリング制度
- 限定的な例外規定
DJ注意点:
- PRS for Music が管理
- クラブ文化が盛んだが規制も厳格

【ドイツ】
保護期間: 著作者の死後70年
特徴:
- GEMA（著作権管理団体）が強力
- クラブへの課金が高額
DJ注意点:
- GEMA料金がイベント費用に大きく影響
- テクノ/電子音楽の中心地だが規制も厳しい

【オランダ】
保護期間: 著作者の死後70年
特徴:
- Buma/Stemra が管理
- 比較的DJ活動に寛容
DJ注意点:
- ADE（Amsterdam Dance Event）の慣行
- 電子音楽産業に理解あり

【韓国】
保護期間: 著作者の死後70年
特徴:
- KOMCA が管理
- K-POP関連の規制が厳格
DJ注意点:
- K-POPのリミックスは要注意

海外でDJする場合:
1. 現地の著作権法を確認
2. 会場/主催者に著作権処理を確認
3. 必要に応じて現地の管理団体に連絡
4. 録音・配信は現地法に従う
```

### デジタル時代の国際著作権課題

**ボーダーレスな配信時代の問題:**

```
デジタル配信特有の課題:

1. 属地主義の限界
   著作権法は国ごとに異なる
   しかしインターネットには国境がない
   → どの国の法律が適用される？

   例:
   日本のDJがSoundCloudにアップ
   → アメリカのサーバー
   → 世界中からアクセス
   → どの国の著作権法が適用？

   原則: アップロード元の国の法律
   + アクセスされる国の法律
   → 非常に複雑

2. プラットフォームの責任
   YouTube、SoundCloud等の責任
   - セーフハーバー条項（DMCA）
   - DSA（EU デジタルサービス法）
   - プロ責法（日本）

3. AI生成音楽の著作権
   AIが作成した楽曲に著作権はあるか？
   - 日本: AI生成物に著作権なし（現時点）
   - アメリカ: 議論中
   - EU: AI Act による規制

   DJ活動への影響:
   - AI生成楽曲は自由に使える可能性
   - ただし学習データの著作権問題
   - 今後の法改正に要注目

4. NFTと音楽著作権
   NFT化された音楽の権利
   - 所有権 ≠ 著作権
   - NFT購入者の利用範囲は限定的
   - DJ利用の許可は別途必要
```

### 楽曲購入と所有権

**購入した楽曲で何ができるか:**

```
デジタル楽曲購入の権利範囲:

Beatport/Traxsource等で購入:
- DJプレイ: ○
- 個人鑑賞: ○
- バックアップコピー: ○
- 友人への譲渡: ×
- 配信/アップロード: ×
- リミックス素材として: ×（許可必要）
- CD-Rに焼いて販売: ×

iTunes/Amazon Music等で購入:
- DJプレイ: ○（DRM無しの場合）
- 個人鑑賞: ○
- 配信: ×
- リミックス: ×

ストリーミング（Spotify等）:
- DJプレイ: △（個人利用の範囲で）

- 配信: ×

重要な区別:

所有していないもの:
- 楽曲の著作権

## レーベル契約とロイヤリティ

### レーベル契約の基本

**DJプロデューサーが知るべき契約知識:**

```
レーベル契約の種類:

1. シングルリリース契約
   1曲単位の契約
   最も一般的な入門形態
   条件:
   - 楽曲の独占的使用許諾
   - 期間: 通常3〜5年
   - ロイヤリティ: 売上の15〜30%
   - 原盤権: レーベルが保有

2. EP/アルバム契約
   複数曲のリリース
   条件:
   - アドバンス（前払い金）がある場合も
   - ロイヤリティ: 売上の20〜40%
   - マーケティング費用の分担
   - 原盤権の帰属が重要

3. ライセンス契約
   楽曲の使用権のみ許諾
   原盤権はアーティストが保持
   条件:
   - 期間限定（通常2〜3年）
   - ロイヤリティが高め（40〜60%）
   - 期間終了後に権利が戻る
   - アーティストに有利

4. 360度契約
   音楽以外の収入も含む
   DJ/プロデューサーの全活動
   条件:
   - ギグ収入の一部もレーベルに
   - グッズ収入も対象
   - 包括的なサポート提供
   - 大手レーベルに多い

契約時の注意点:
□ 契約期間と延長条件
□ ロイヤリティ率と計算方法
□ 原盤権の帰属
□ 独占/非独占の範囲
□ テリトリー（地域）の範囲
□ 解除条件
□ 監査権（会計チェックの権利）
```

### ロイヤリティの仕組み

**どうやって収入を得るか:**

```
音楽ロイヤリティの種類:

1. 機械的ロイヤリティ
   楽曲が複製されるごとに発生
   - CD製造
   - デジタルダウンロード
   - ストリーミング再生

   率: 日本では JASRAC が管理
   ストリーミング1再生: 約¥0.3〜1.0

2. 演奏ロイヤリティ
   楽曲が公に演奏されるごとに発生
   - ラジオ放送
   - クラブでの使用
   - ライブ演奏

   管理: JASRAC/NexTone が徴収・分配

3. シンクロナイゼーション（シンク）ロイヤリティ
   映像と楽曲を同期使用する場合
   - CM
   - 映画
   - テレビ番組
   - ゲーム

   交渉: 個別に権利者と
   金額: 数万円〜数千万円

4. マスター使用ロイヤリティ
   原盤（マスター録音）の使用
   - サンプリング
   - コンピレーション収録
   - 映像使用

   権利者: レコード会社（通常）

DJプロデューサーの収入例:

Beatport売上:
  販売価格 ¥250
  - プラットフォーム手数料: 約40%（¥100）
  - レーベル取り分: 約35%（¥87）
  - アーティスト取り分: 約25%（¥63）

Spotify再生:
  1再生あたり約¥0.5
  - レーベル取り分: 約60%
  - アーティスト取り分: 約40%
  → 1再生 ≒ ¥0.2（アーティスト）
  → 100万再生 ≒ ¥200,000

年間収入モデル（中堅DJ/プロデューサー）:
  Beatport/Traxsource: ¥200,000〜500,000
  ストリーミング: ¥100,000〜300,000
  ギグ収入: ¥1,000,000〜3,000,000
  シンクライセンス: 不定期
```

### 原盤権（マスター権）

**最も重要な権利の一つ:**

```
原盤権とは:
録音された音源そのものに対する権利
楽曲の著作権とは別物

原盤権の保有者:
- 通常: レコード会社/レーベル
- インディー: アーティスト自身
- 共同制作: 契約で決定

原盤権が関係する場面:

使用される場面 → 原盤権者の許可必要:
- CM使用
- 映画/ドラマ使用
- コンピレーション収録
- サンプリング
- カバー曲（原盤は不要、楽曲のみ）
- ストリーミング配信

原盤権の価値:
- 大ヒット曲: 数億円〜数十億円
- 有名アーティストのカタログ: 数百億円

DJプロデューサーへのアドバイス:

□ 可能な限り原盤権を自分で保持する
□ レーベルに譲渡する場合は条件を精査
□ ライセンス契約を優先的に検討
□ 期間限定の権利譲渡を交渉
□ 共同制作時は事前に権利配分を決める

セルフリリースの場合:
- DistroKid、TuneCore等を利用
- 原盤権は100%自分で保持
- ロイヤリティも自分で管理
- レーベルのプロモーション力は得られない
```

## 著作権侵害への対処

### 自分の楽曲が侵害された場合

**権利を守るための行動:**

```
侵害発見時の対応手順:

Step 1: 証拠の保全
  - スクリーンショットを保存
  - URLを記録
  - 日時を記録
  - 可能なら録音/録画

Step 2: プラットフォームへの申し立て
  各プラットフォームの通報窓口:

  YouTube: 著作権侵害の申し立て
  → youtube.com/copyright_complaint_form

  SoundCloud: Copyright Infringement
  → soundcloud.com/pages/copyright

  Spotify: Content Abuse Report
  → artists.spotify.com

  Instagram: IP Report Form
  → help.instagram.com

Step 3: 警告書の送付（任意）
  侵害者に直接連絡
  - 削除を要求
  - 期限を設定（通常7〜14日）
  - 応じなければ法的手段を示唆

Step 4: 法的手段（必要な場合）
  - 弁護士に相談
  - 内容証明郵便
  - 訴訟（最終手段）

  費用の目安:
  弁護士相談: ¥5,000〜/30分
  内容証明: ¥10,000〜30,000
  訴訟: ¥300,000〜（着手金）
```

### 著作権侵害の申し立てを受けた場合

**冷静に対応する:**

```
申し立てを受けた時の対応:

1. 内容を確認
   - 何が侵害とされているか
   - 申し立て者は誰か
   - 正当な申し立てか

2. 正当な申し立ての場合
   - 即座にコンテンツを削除
   - 謝罪の連絡（必要に応じて）
   - 再発防止

3. 不当な申し立ての場合
   - 異議申し立て（Counter Notification）
   - 証拠を添えて反論
   - 必要に応じて弁護士に相談

4. Content ID の誤検出
   YouTube等で誤って検出される場合
   - 異議申し立てフォームを使用
   - オリジナルである証拠を提示
   - 制作データ（DAWプロジェクト等）を保持

予防策:
□ オリジナル楽曲の制作日時を記録
□ DAWプロジェクトファイルを保存
□ リリース前にContent IDに登録
□ 著作権登録を検討（特に重要な楽曲）
```

## フェアユースとフェアディーリング

### フェアユース（米国）

**著作権の例外規定:**

```
フェアユース（Fair Use）:
米国著作権法107条に基づく例外
著作権者の許可なく使用できる場合がある

4つの判断要素:

1. 使用の目的と性質
   - 商用利用か非商用か
   - 変容的使用（transformative use）か
   - 教育/批評/パロディ目的か

2. 著作物の性質
   - 事実的か創作的か
   - 公表済みか未公表か

3. 使用された部分の量と重要性
   - 全体のどの程度を使用したか
   - 楽曲の「心臓部」を使用したか

4. 市場への影響
   - 原著作物の市場を損なうか
   - 代替品になりうるか

DJとフェアユース:
- DJミックスはフェアユースに該当しにくい
  → 楽曲全体を使用
  → 市場への代替効果
- ただし短いサンプルは可能性あり
- 教育目的のデモは主張可能

注意:
- フェアユースは米国の法律
- 日本には適用されない
- 訴訟になるまで確定しない
- 「安全」ではなく「抗弁」の一種
```

### 日本の権利制限規定

**日本著作権法における例外:**

```
日本の権利制限規定（著作権法30条〜50条）:

DJに関連する主な規定:

1. 私的使用のための複製（第30条）
   個人的に楽しむための複製はOK
   - CDをリッピングしてDJ用に変換: ○
   - 友人にコピーを渡す: ×
   - ネットにアップロード: ×

2. 引用（第32条）
   正当な範囲での引用はOK
   条件:
   - 引用の必要性がある
   - 主従関係が明確
   - 出典を明記
   DJとの関連:
   - レビュー動画での楽曲使用（短い断片）
   - DJ講座での説明用再生

3. 営利を目的としない上演等（第38条）
   非営利・無料・無報酬なら演奏OK
   条件:
   - 入場料なし
   - 演奏者に報酬なし
   - 営利目的でない
   DJとの関連:
   - 完全無料のチャリティイベント
   - 学校の文化祭
   - 非営利団体のイベント

4. 時事の事件の報道のための利用（第41条）
   報道目的なら利用OK
   DJとの関連:
   - ニュース番組でのフェス映像
   - イベントレポート記事

注意:
日本にはフェアユースのような
包括的な例外規定はない
個別の制限規定に当てはまるかが重要
```

## まとめ

### 著作権 5つのポイント

```
1. 知識を持つ
   └─ 無知は言い訳にならない

2. 会場の契約確認
   └─ プレイ前に必ず

3. Mixcloud 活用
   └─ 合法的な配信

4. リスクを理解
   └─ グレーゾーンを認識

5. 商用利用は慎重に
   └─ 許可なしは絶対NG
```

### 著作権管理の10原則

```
DJ/プロデューサーのための10原則:

1. 正規の楽曲を使う
   └─ 海賊版、不正ダウンロードは絶対NG

2. 権利の二重構造を理解する
   └─ 著作権（楽曲）+ 著作隣接権（音源）

3. 会場の契約を確認する
   └─ JASRAC/NexToneとの包括契約

4. 配信はプラットフォームを選ぶ
   └─ Mixcloudが最も安全

5. オリジナル楽曲の権利を守る
   └─ 原盤権は可能な限り自分で保持

6. 契約書は必ず読む
   └─ レーベル契約、ライセンス契約

7. クレジットを正確に表示する
   └─ トラックリスト、リミックスクレジット

8. 商用利用は許可を取る
   └─ 無許可の商用利用は高リスク

9. 記録を残す
   └─ ギグ、配信、収入の記録

10. 最新情報をキャッチアップする
    └─ 法改正、判例、業界動向
```
